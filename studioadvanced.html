21ModuleScript 고급

코드가 길어지면 스크립트 하나로는 감당이 안 됩니다. 기능별로 쪼개서 재사용하는 도구.

모듈 만들기

-- ReplicatedStorage > MathUtil (ModuleScript)
local MathUtil = {}

function MathUtil.clamp(value, min, max)
    return math.max(min, math.min(max, value))
end

function MathUtil.round(value, decimals)
    local mult = 10 ^ (decimals or 0)
    return math.floor(value * mult + 0.5) / mult
end

return MathUtil  -- 모듈은 반드시 무언가를 return

모듈 사용하기

-- 어느 스크립트에서든
local MathUtil = require(game.ReplicatedStorage.MathUtil)

print(MathUtil.clamp(150, 0, 100))  -- 100
print(MathUtil.round(3.14159, 2))   -- 3.14

모듈 배치 기준

위치접근 가능용도
ReplicatedStorage서버 + 클라이언트설정값, 유틸 함수, 공용 데이터
ServerScriptService / ServerStorage서버만게임 로직, 보상 계산, 데이터 처리
TIP

require된 모듈은 캐싱됩니다. 여러 스크립트에서 require해도 같은 테이블 하나를 공유합니다. 이 특성으로 스크립트 간 데이터 공유도 가능합니다.

22OOP와 메타테이블 고급

같은 종류의 오브젝트를 클래스로 찍어내는 객체지향 패턴. 대형 게임 코드의 표준 구조.

클래스 패턴

-- Enemy (ModuleScript)
local Enemy = {}
Enemy.__index = Enemy

-- 생성자
function Enemy.new(name, health)
    local self = setmetatable({}, Enemy)
    self.Name = name
    self.Health = health
    self.MaxHealth = health
    return self
end

-- 메서드
function Enemy:TakeDamage(amount)
    self.Health = math.max(0, self.Health - amount)
    if self.Health <= 0 then
        self:Die()
    end
end

function Enemy:Die()
    print(self.Name .. " 사망")
end

return Enemy

사용

local Enemy = require(game.ServerStorage.Enemy)

local goblin = Enemy.new("고블린", 50)
local orc = Enemy.new("오크", 120)

goblin:TakeDamage(30)  -- 체력 20 남음
goblin:TakeDamage(30)  -- "고블린 사망"

핵심 개념 풀이

  • 메타테이블: 테이블에 "없는 키를 찾으면 어디서 찾을지" 같은 특수 동작을 부여하는 테이블.
  • __index: goblin에 TakeDamage가 없으면 Enemy 테이블에서 찾아라, 라는 규칙. 이것이 상속의 원리.
  • 콜론(:) 문법: goblin:TakeDamage(30)은 Enemy.TakeDamage(goblin, 30)의 줄임. self가 자동으로 전달됩니다.

23DataStore 저장 고급

플레이어가 나갔다 들어와도 데이터가 유지되게. 레벨, 돈, 인벤토리 영구 저장.

사전 설정

Game Settings > Security > Enable Studio Access to API Services 켜기. 그리고 게임이 퍼블리시되어 있어야 작동합니다.

기본 저장/불러오기

local DataStoreService = game:GetService("DataStoreService")
local Players = game:GetService("Players")
local store = DataStoreService:GetDataStore("PlayerData_v1")

-- 입장 시 불러오기
Players.PlayerAdded:Connect(function(player)
    local key = "player_" .. player.UserId

    local ok, data = pcall(function()
        return store:GetAsync(key)
    end)

    if ok then
        data = data or {coins = 0, level = 1}  -- 신규 유저 기본값
        print(player.Name, "코인:", data.coins)
    else
        warn("데이터 로드 실패: " .. player.Name)
    end
end)

-- 퇴장 시 저장
Players.PlayerRemoving:Connect(function(player)
    local key = "player_" .. player.UserId
    local data = {coins = 120, level = 3}  -- 실제로는 현재 값

    local ok = pcall(function()
        store:SetAsync(key, data)
    end)
    if not ok then warn("저장 실패") end
end)

반드시 지킬 것

  • pcall 필수: DataStore는 네트워크 요청이라 실패할 수 있습니다. pcall 없이 에러가 나면 스크립트가 죽습니다.
  • game:BindToClose: 서버가 닫힐 때(마지막 플레이어 퇴장 등)도 저장되도록 처리하세요. 안 하면 데이터 유실이 생깁니다.
  • UpdateAsync 권장: 단순 SetAsync보다 기존 값을 받아 수정하는 UpdateAsync가 데이터 충돌에 안전합니다.
  • 요청 제한: DataStore는 분당 요청 횟수 제한이 있습니다. 매초 저장하지 말고 퇴장 시 + 주기적(1~2분) 저장.
-- 서버 종료 시 저장 보장
game:BindToClose(function()
    for _, player in ipairs(Players:GetPlayers()) do
        saveData(player)  -- 저장 함수 호출
    end
    task.wait(3)  -- 저장 완료 시간 확보
end)

24Attribute와 태그 고급

인스턴스에 커스텀 데이터를 붙이는 현대적인 방법. Value 오브젝트보다 깔끔합니다.

Attribute

-- 값 설정 (Properties 창 하단 Attributes에서 수동으로도 가능)
part:SetAttribute("Damage", 25)
part:SetAttribute("ElementType", "Fire")

-- 값 읽기
local dmg = part:GetAttribute("Damage")

-- 값이 바뀔 때 감지
part:GetAttributeChangedSignal("Damage"):Connect(function()
    print("데미지 변경:", part:GetAttribute("Damage"))
end)

CollectionService 태그

"이 종류의 파트 전부"를 다룰 때 씁니다. 예: 모든 용암 파트에 데미지 적용.

local CS = game:GetService("CollectionService")

-- 태그 붙이기 (에디터의 Tags 속성에서도 가능)
CS:AddTag(part, "Lava")

-- 태그 달린 모든 파트에 동작 연결
for _, lava in ipairs(CS:GetTagged("Lava")) do
    lava.Touched:Connect(function(hit)
        local h = hit.Parent:FindFirstChildOfClass("Humanoid")
        if h then h:TakeDamage(10) end
    end)
end

-- 나중에 추가되는 것도 자동 처리
CS:GetInstanceAddedSignal("Lava"):Connect(function(newLava)
    -- 위와 같은 연결
end)
TIP

용암 100개에 스크립트 100개를 넣는 대신, 태그 하나와 스크립트 하나로 전부 처리할 수 있습니다. 유지보수가 압도적으로 쉬워집니다.

25CFrame 정복 고급

위치 + 회전을 하나로 담는 좌표계. 카메라, 텔레포트, 발사체, 모든 공간 계산의 기초.

Position vs CFrame

  • Position: 위치만 (Vector3). 회전 정보 없음.
  • CFrame: 위치 + 회전. 파트가 "어디에, 어느 방향으로" 있는지 전부.

기본 사용

-- 위치 지정
part.CFrame = CFrame.new(0, 10, 0)

-- 특정 지점을 바라보게
part.CFrame = CFrame.lookAt(part.Position, target.Position)

-- 회전 (각도는 라디안: math.rad로 변환)
part.CFrame = part.CFrame * CFrame.Angles(0, math.rad(90), 0)

상대 이동 — 곱셈의 의미

-- 파트가 바라보는 방향으로 5만큼 앞으로
part.CFrame = part.CFrame * CFrame.new(0, 0, -5)
-- Roblox에서 -Z가 "앞"입니다

-- 방향 벡터 활용
local forward = part.CFrame.LookVector   -- 앞 방향
local up = part.CFrame.UpVector          -- 위 방향
local right = part.CFrame.RightVector    -- 오른쪽

-- 캐릭터 앞 10 studs 위치에 소환
local spawnPos = root.CFrame * CFrame.new(0, 0, -10)
!!

CFrame 곱셈은 순서가 중요합니다. A * B와 B * A는 다른 결과입니다. "A의 좌표계 안에서 B만큼 이동/회전"으로 읽으세요.

실전 예제 — 회전하는 발판

local RunService = game:GetService("RunService")
local platform = workspace.SpinPlatform

RunService.Heartbeat:Connect(function(dt)
    platform.CFrame = platform.CFrame
        * CFrame.Angles(0, math.rad(45) * dt, 0)
    -- dt를 곱해 프레임 속도와 무관하게 초당 45도 회전
end)

26Raycast 고급

한 점에서 광선을 쏴서 무엇에 맞는지 알아내는 기술. 총알 판정, 시야 확인, 지면 감지의 핵심.

기본 Raycast

local origin = gun.Position
local direction = gun.CFrame.LookVector * 100  -- 앞으로 100 studs

-- 필터: 발사자 자신은 무시
local params = RaycastParams.new()
params.FilterType = Enum.RaycastFilterType.Exclude
params.FilterDescendantsInstances = {shooter.Character}

local result = workspace:Raycast(origin, direction, params)

if result then
    print("맞은 것:", result.Instance.Name)
    print("맞은 위치:", result.Position)
    print("거리:", result.Distance)

    local humanoid = result.Instance.Parent
        :FindFirstChildOfClass("Humanoid")
    if humanoid then
        humanoid:TakeDamage(20)
    end
end

활용처

용도방법
총/레이저 무기총구에서 LookVector 방향으로 발사
지면 감지캐릭터 아래로(-Y) 짧게 쏴서 바닥 재질 확인
시야 확인A에서 B로 쏴서 중간에 벽이 있는지 검사
마우스 클릭 위치카메라에서 마우스 방향으로 발사
TIP

result.Normal은 맞은 면의 수직 방향입니다. 총알 자국 데칼을 표면에 정확히 붙이거나, 반사각을 계산할 때 사용합니다.

27물리와 Constraint 고급

힘, 경첩, 용수철, 모터. Roblox 물리 엔진을 활용한 동적인 구조물.

주요 Constraint

Constraint동작예시
HingeConstraint경첩. 한 축으로 회전문, 레버, 바퀴
SpringConstraint용수철차량 서스펜션, 탄성 다리
RopeConstraint늘어나지 않는 줄그네, 매달린 램프
WeldConstraint두 파트를 단단히 붙임움직이는 모델 조립
PrismaticConstraint레일 위 직선 이동엘리베이터, 슬라이딩 도어

힘 적용 — Mover

-- 파트를 위로 띄우는 힘 (Attachment 필요)
local att = Instance.new("Attachment", part)

local force = Instance.new("VectorForce")
force.Attachment0 = att
force.Force = Vector3.new(0, part.AssemblyMass * workspace.Gravity, 0)
-- 중력과 같은 크기 = 공중에 둥둥
force.Parent = part

자동으로 도는 문 (HingeConstraint Motor)

  1. 문 파트와 문틀 파트에 각각 Attachment 추가 (경첩 위치에).
  2. HingeConstraint 추가, Attachment0/1 연결.
  3. ActuatorType을 Motor로, AngularVelocity에 회전 속도 입력.

순간 속도 — AssemblyLinearVelocity

-- 점프 패드: 밟으면 위로 발사
pad.Touched:Connect(function(hit)
    local root = hit.Parent:FindFirstChild("HumanoidRootPart")
    if root then
        root.AssemblyLinearVelocity = Vector3.new(0, 120, 0)
    end
end)

28애니메이션 고급

캐릭터와 NPC의 동작. 애니메이션 에디터로 만들고 코드로 재생합니다.

애니메이션 만들기

  1. Avatar 탭 > Animation Editor.
  2. 리그(캐릭터 모델) 선택. 없으면 Avatar 탭 > Rig Builder로 더미 생성.
  3. 타임라인에서 시간을 옮기고, 관절을 회전시켜 포즈를 잡으면 키프레임이 생성됩니다.
  4. 포즈 사이는 자동으로 부드럽게 보간됩니다.
  5. 완성 후 에디터 메뉴 > Publish to Roblox. 발급된 Animation ID를 기록.

코드로 재생

local humanoid = npc:WaitForChild("Humanoid")
local animator = humanoid:WaitForChild("Animator")

local anim = Instance.new("Animation")
anim.AnimationId = "rbxassetid://애니메이션ID"

local track = animator:LoadAnimation(anim)
track:Play()

-- 제어
track.Looped = true
track:AdjustSpeed(1.5)   -- 1.5배속
track:Stop()

Priority — 애니메이션 우선순위

걷기 중에 공격 애니메이션을 겹치려면 우선순위를 다르게:

  • Core: 가장 낮음. 기본 idle.
  • Movement: 걷기, 달리기.
  • Action: 공격, 스킬. Movement 위에 겹쳐 재생됨.
RBX

기본 걷기/점프 애니메이션을 교체하려면 캐릭터 안의 Animate(LocalScript)에 들어있는 애니메이션 ID 값을 내 것으로 바꾸면 됩니다.

29보안과 안티 익스플로잇 고급

익스플로잇 유저는 클라이언트의 모든 것을 조작할 수 있습니다. 방어는 서버 설계에서 시작합니다.

대원칙: 클라이언트는 절대 신뢰하지 않는다

  • 익스플로잇 유저는 LocalScript를 읽고, 수정하고, RemoteEvent를 마음대로 호출할 수 있습니다.
  • 클라이언트가 보내는 모든 값은 "조작됐을 수 있다"고 가정하세요.
  • 중요한 계산(돈, 데미지, 보상)은 전부 서버에서.

나쁜 예 vs 좋은 예

-- 나쁜 예: 클라이언트가 보낸 금액을 그대로 믿음
buyEvent.OnServerEvent:Connect(function(player, price)
    coins[player] = coins[player] - price  -- price를 0이나 음수로 보내면?
end)

-- 좋은 예: 서버가 가격을 직접 결정
local PRICES = {Sword = 100, Shield = 150}

buyEvent.OnServerEvent:Connect(function(player, itemName)
    local price = PRICES[itemName]
    if not price then return end            -- 존재하는 아이템인지
    if coins[player] < price then return end -- 돈이 충분한지

    coins[player] = coins[player] - price
    giveItem(player, itemName)
end)

서버 검증 체크리스트

  • 타입 검사: 받은 값이 기대한 타입인지 (typeof(x) == "string").
  • 범위 검사: 숫자가 정상 범위인지 (음수 금액, 비정상 거리 등).
  • 거리 검사: "상자 열기" 요청 시 플레이어가 실제로 상자 근처에 있는지.
  • 쿨다운 검사: 스킬 요청이 서버 기준 쿨다운을 지켰는지. 클라이언트 쿨다운은 장식입니다.
  • 요청 빈도 제한: 같은 RemoteEvent를 초당 수백 번 호출하는 스팸 차단.
!!

중요한 값(돈, 인벤토리)을 클라이언트가 읽을 수 있는 Value 오브젝트나 ReplicatedStorage에 원본으로 두지 마세요. 원본은 서버 스크립트 내부 테이블에 두고, 표시용 사본만 클라이언트에 보냅니다.

30최적화와 수익화 고급

완성한 게임을 빠르게 만들고, 수익 구조를 붙여 출시하는 마지막 단계.

성능 측정

  • MicroProfiler: Play 중 Ctrl+F6. 프레임별 CPU/GPU 비용 분석.
  • Script Performance: View > Script Performance. 어떤 스크립트가 CPU를 먹는지 순위로 표시.
  • Stats: Shift+F5(인게임 F9 콘솔)로 메모리, 네트워크 확인.

최적화 핵심 기법

영역기법
렌더링파트 수 줄이기(Union, 대형 파트), 작은 오브젝트 CastShadow 끄기, MeshPart RenderFidelity를 Automatic으로
스크립트while wait() 루프 대신 이벤트 사용, 매 프레임 도는 코드 최소화, 무거운 작업은 task.spawn으로 분산
네트워크RemoteEvent 호출 횟수와 데이터 크기 최소화. 매 프레임 Fire 금지
대형 맵Workspace.StreamingEnabled 켜기 — 플레이어 주변만 로드
메모리안 쓰는 connection은 Disconnect, Destroy한 오브젝트 참조 정리

수익화 수단

수단특징예시
게임패스 (Game Pass)한 번 구매, 영구 소유VIP, 2배 경험치, 특수 도구
개발자 제품 (Developer Product)반복 구매 가능코인 1000개, 즉시 부활
구독 (Subscription)월간 결제월간 혜택 패키지
몰입형 광고월드 내 광고판패시브 수익

local GAMEPASS_ID = 12345678

-- 보유 확인
local ok, owns = pcall(function()
    return MarketplaceService:UserOwnsGamePassAsync(
        player.UserId, GAMEPASS_ID)
end)
if ok and owns then
    -- VIP 혜택 지급

31검기 VFX 만들기 고급

킹 레가시, 블록스 프루츠 스타일의 검기(슬래시) 이펙트. 핵심은 호(arc) 모양 메시 + UV 그라데이션 텍스처입니다.

검기의 원리

검기는 파티클이 아니라 메시입니다. 칼이 지나간 궤적 모양의 얇은 호(arc) 메시에, 한쪽 끝은 진하고 반대쪽은 투명해지는 그라데이션 텍스처를 입힌 것. 이 메시를 코드로 순간적으로 키우면서 회전시키고 투명하게 사라지게 하면 검기가 됩니다.

질문하신 UV가 맞습니다 — 그라데이션이 메시의 어느 방향으로 흐를지를 결정하는 게 UV입니다. UV를 잘못 펼치면 그라데이션이 엉뚱한 방향으로 흐릅니다.

1. 메시 (블렌더)

초승달 모양의 얇은 호. 칼 궤적의 형태를 결정.

2. 텍스처 (UV)

흰색에서 투명으로 빠지는 그라데이션. 검기의 꼬리 느낌을 결정.

3. 코드 (Studio)

Tween으로 확대 + 회전 + 페이드. 검기의 속도감을 결정.

1단계 — 블렌더에서 검기 메시 만들기

  1. Shift+A > Mesh > Torus 추가. Major Segments 48, Minor Segments 3, Minor Radius 0.05 정도로 얇게.
  2. 에디트 모드(Tab) > 토러스의 2/3를 박스 선택해서 삭제. 남은 1/3이 초승달 호가 됩니다.
  3. 양 끝 부분을 비례 편집(O)으로 가늘게 모아 뾰족하게. 검기 끝이 날카로워집니다.
  4. 위에서 봤을 때 살짝 납작하게 S > Z로 눌러줍니다.
  5. 오브젝트 모드에서 Ctrl+A > All Transforms 적용.
TIP

더 간단한 방법: Plane 하나를 추가하고 Simple Deform 모디파이어(Bend, 120도)로 휘어도 검기 호가 됩니다. 폴리곤이 적어 게임에 더 가볍습니다.

2단계 — UV 펼치기와 그라데이션 텍스처

  1. 에디트 모드 > A 전체 선택 > U > Unwrap.
  2. UV Editing 워크스페이스로 이동. 호가 일자로 펴진 UV가 보입니다.
  3. UV 아일랜드를 회전/정렬해서 호의 길이 방향이 U축(가로)이 되게 합니다. 이게 핵심입니다 — 그라데이션이 가로로 흐르는 텍스처를 쓸 것이기 때문.
  4. 그라데이션 이미지 제작: 256x64 픽셀, 왼쪽은 완전 흰색(불투명), 오른쪽으로 갈수록 알파 0(투명). 블렌더 텍스처 페인트나 외부 이미지 툴 어느 쪽이든 가능합니다.
  5. 가운데가 가장 밝고 위아래 가장자리가 투명해지는 세로 그라데이션도 겹치면 더 부드러운 검기가 됩니다.
!!

텍스처는 반드시 PNG로 저장해야 알파(투명도)가 유지됩니다. JPG는 알파 채널이 없습니다.

3단계 — Studio로 가져오기

  1. FBX로 내보내고 Studio에서 Import 3D로 임포트.
  2. 그라데이션 PNG를 Creator Hub에서 Decal/Image로 업로드하고 Asset ID를 받습니다.
  3. MeshPart의 TextureID에 rbxassetid://ID 입력.
  4. MeshPart 속성: Material: Neon CanCollide: false Anchored: true CastShadow: false
  5. Color를 검기 색으로 (하늘색, 보라, 빨강 등). Neon + 밝은 색이면 자체발광처럼 보입니다.
  6. 완성된 검기 메시를 ReplicatedStorage에 템플릿으로 보관.

4단계 — 코드로 휘두르기

-- SlashVFX (ModuleScript) : 검기 발사
local TweenService = game:GetService("TweenService")
local Debris = game:GetService("Debris")
local template = game.ReplicatedStorage.VFX.SlashMesh

local SlashVFX = {}

function SlashVFX.play(character, slashColor)
    local root = character.HumanoidRootPart
    local slash = template:Clone()

    -- 캐릭터 앞 2 studs, 칼 휘두르는 각도로 기울이기
    slash.CFrame = root.CFrame
        * CFrame.new(0, 0, -2)
        * CFrame.Angles(0, 0, math.rad(25))
    slash.Size = slash.Size * 0.3   -- 작게 시작
    slash.Transparency = 0.1
    slash.Color = slashColor or Color3.fromRGB(120, 200, 255)
    slash.Parent = workspace

    -- 0.25초 동안: 커지면서 + 앞으로 나가면서 + 사라짐
    local goal = {
        Size = slash.Size * 4,
        CFrame = slash.CFrame * CFrame.new(0, 0, -8),
        Transparency = 1,
    }
    local info = TweenInfo.new(0.25,
        Enum.EasingStyle.Quart, Enum.EasingDirection.Out)
    TweenService:Create(slash, info, goal):Play()

    Debris:AddItem(slash, 0.3)  -- 0.3초 뒤 자동 삭제
end

return SlashVFX
RBX

연속 베기는 이 함수를 0.1초 간격으로 좌우 각도(Z축 +25도 / -25도)를 번갈아 호출하면 됩니다. 검기가 X자로 교차하면서 나가는 블록스 프루츠 느낌이 납니다.

보조 이펙트 — Trail과 ParticleEmitter

  • Trail (칼 잔상): 칼 모델의 칼끝과 손잡이에 Attachment 2개 추가 > Trail 인스턴스 연결 > Lifetime 0.15, Transparency NumberSequence(0 → 1). 칼을 휘두를 때만 Trail.Enabled = true로 켭니다. 검기 메시와 별개로 칼 자체에 남는 빛 줄기입니다.
  • ParticleEmitter (타격 불꽃): 검기가 적에 맞는 순간 적 위치에 파티클을 0.1초만 방출(Emit(15)). Texture는 작은 빛 점, Speed 8~15, Lifetime 0.2~0.4.
  • 사운드: 휘두를 때 swoosh, 맞을 때 hit 사운드. VFX는 소리가 절반입니다.

TIP

검기가 밋밋하면 같은 메시를 크기와 투명도만 다르게 2~3겹 겹쳐서 동시에 발사해보세요. 안쪽은 흰색(코어), 바깥쪽은 색깔(글로우)로 하면 입체감이 확 살아납니다. 에너지 구체, 빔, 충격파, 마법진 등 다른 스킬 부품 메시 만들기는 블렌더 고급 강좌의 "게임 스킬 이펙트 메시" 챕터에서 다룹니다.
