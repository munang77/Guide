21파티클 시스템 고급

오라, 빛 입자, 연기, 불꽃을 표현하는 파티클 시스템. 이펙트의 핵심 기술입니다.

파티클이란?

파티클은 메시 표면에서 자동으로 생성되고 움직이는 작은 오브젝트들입니다. 불꽃 하나하나, 오라에서 날리는 입자들, 눈발이 모두 파티클로 만들어집니다.

파티클 시스템 추가

오브젝트 선택 > Properties > 파티클 아이콘(점들) > + > New.

파티클 타입

타입특징이펙트 활용
Emitter (방출기)표면에서 입자를 계속 방출. 일정 시간 후 사라짐불꽃, 오라 입자, 마법 가루, 상승 입자
Hair (헤어)표면에서 가닥이 자람. 정적풀, 털, 가시처럼 방사형으로 뻗는 오라 스파이크

Emitter 주요 파라미터

항목위치설명오라 이펙트 권장값
Number (개수)Emission총 파티클 개수200~600
Lifetime (생존 시간)Emission파티클이 살아있는 프레임 수30~60 (짧으면 반짝, 길면 부유)
Normal (수직 속도)Velocity표면에서 수직으로 튀어나가는 속도상승 이펙트: 1~3
Gravity (중력)Field Weights중력 영향도. 0이면 무중력부유 오라: 0
Size (크기)Render파티클 표시 크기먼지: 0.02~0.05, 구슬: 0.1~0.2
Random (크기 랜덤)Render크기를 랜덤하게0.3~0.5로 자연스럽게

포스 필드 (Force Field)

파티클에 힘을 주는 오브젝트입니다. Shift+A > Force Field:

종류효과이펙트 활용
Vortex (소용돌이)축 중심으로 회전회전하는 오라, 토네이도
Wind (바람)한 방향으로 밀기상승 기류, 먼지 날리기
Turbulence (난류)무작위 방향으로 흔들기불규칙하게 떠다니는 오라 입자
Charge (끌어당기기/밀어내기)가운데로 끌어당기거나 밀어냄에너지 집중(양수) 또는 폭발(음수)
Force (힘)지정한 방향으로 힘방향성 있는 이펙트

파티클 재질

파티클 오브젝트에 Emission(발광) 재질을 적용하면 빛나는 파티클이 됩니다. Emission Strength를 5~20으로 설정하고, EEVEE에서 Bloom을 켜면 실제로 빛이 번지는 효과가 납니다.

뷰포트 성능

  • Display Percentage: 파티클 설정에서 Display로 내려가면 있습니다. 10~30%로 낮춰 작업 시 버벅임 방지.
  • Render Percentage: 렌더링 시에는 100%로.

22물리 시뮬레이션 고급

천, 강체 파편, 유체, 연기. 시뮬레이션 결과를 정적 메시로 굳혀 게임 에셋으로 활용합니다.

Cloth (천 시뮬레이션)

Properties > Physics > Cloth. 깃발, 망토, 커튼에 활용합니다.

  • Vertex Mass: 천의 무게. 실크처럼 가볍게 = 0.1, 가죽처럼 무겁게 = 1.0.
  • Stiffness: 뻣뻣한 정도. 높을수록 잘 안 구부러짐.
  • Pinning: 특정 버텍스를 고정점으로 지정. 깃발 끝은 고정하고 나머지는 펄럭이게.
  • 충돌면 오브젝트에도 Physics > Collision을 추가해야 천이 그 표면에 닿습니다.
  • 결과 고정: 원하는 형태가 됐을 때 해당 프레임에서 모디파이어를 Apply(적용).

Rigid Body (강체 - 물리 파편)

  • Active: 물리 영향을 받는 오브젝트. 파편, 도미노처럼 쓰러지는 오브젝트.
  • Passive: 고정된 충돌면. 바닥, 벽.
  • Collision Shape: Box나 Sphere(빠름), Mesh(정확하지만 느림). 단순한 형태는 Box를 쓰세요.

Cell Fracture (파편 분열)

메시를 세포 분열하듯 여러 파편으로 나눠주는 애드온입니다. Preferences에서 Cell Fracture를 활성화한 뒤:

  1. 분열시킬 메시 선택.
  2. F3 검색창 > Cell Fracture 실행.
  3. Source Limit: 파편 개수 (50~150 권장).
  4. 생성된 파편들 전체 선택 > Physics > Rigid Body (Active).
  5. Force Field (Charge, 음수값)로 폭발 효과.

Volume (볼륨 - 연기/안개)

볼륨 재질로 3D 공간 안에 연기나 안개를 채울 수 있습니다:

  1. Cube 추가 > 연기 영역 크기로 조절.
  2. 재질 추가 > Shader Editor에서 Surface 셰이더를 지우고 Volume Scatter 노드 추가.
  3. Volume Scatter의 Color로 연기 색상, Density로 진하기 조절.
  4. Noise Texture를 Density에 연결하면 불균일한 자연스러운 연기.
RBX

블렌더 물리 시뮬레이션은 결과를 정적 형태로 굳혀서 FBX로 내보내는 용도로 씁니다. 천 시뮬로 자연스러운 망토 형태를 잡고 Apply한 뒤 내보내기. 실시간 물리는 Studio에서 별도 처리합니다.

23EEVEE 이펙트 세팅 고급

이펙트 프리뷰에 최적인 EEVEE 엔진을 최대한 활용하는 설정.

이펙트 필수 설정

Properties > Render > EEVEE (렌더 엔진이 EEVEE일 때):

설정권장값효과
Bloom켬, Threshold 0.8, Intensity 0.05Emission 재질이 실제로 빛나는 효과. 이펙트 필수
Ambient Occlusion켬, Distance 0.2메시 틈새에 음영이 생겨 입체감 증가
Screen Space Reflections켬표면 반사 효과
Volumetrics켬, Tile Size 2px안개, 빛기둥(God Ray) 표현
Shadows > Cube Size4096그림자 품질 최대화

Bloom 활용법

Emission 재질의 Strength를 3 이상으로 올리고 Bloom을 켜면 재질에서 빛이 번져 글로우 효과가 납니다. Threshold는 이 밝기 이상에서만 Bloom이 발생하는 기준값입니다. 너무 낮으면 전체가 번져 지저분해지고, 너무 높으면 발광 효과가 안 나옵니다. 0.8이 적절합니다.

Volumetric 빛기둥 만들기 (God Ray)

  1. Render > Volumetrics 켜기.
  2. World 설정에서 Shader Editor를 World로 전환 > Use Nodes.
  3. Volume Scatter 노드 추가. Density 0.02~0.05.
  4. Spot 조명을 배치하고 Shadow 옵션 켜기.
  5. 조명 방향으로 빛기둥이 보입니다.

EEVEE vs Cycles 비교

항목EEVEECycles
속도매우 빠름 (초~분)느림 (분~시간)
Bloom내장, 즉시 사용컴포지터에서 별도 처리
이펙트 프리뷰최적정확하지만 느림

24솔스RNG 이펙트 해부 고급

솔스RNG 같은 스타일의 이펙트가 어떤 요소로 이루어져 있는지 분해합니다.

이펙트의 구성 요소

요소별 블렌더 구현 방법

요소블렌더 방법핵심 포인트
A. 오라 (전신 발광)캐릭터 메시 복제 → Scale 1.05 → Emission+Fresnel 재질Layer Weight의 Fresnel 출력을 Mix Shader Fac에 연결. 외곽만 빛남
B. 빔 (수직 빛기둥)길쭉한 Cylinder → Emission 재질 → Scale Y 애니메이션Scale Y를 0에서 1로 키프레임. Bloom 필수
C. 링 (회전 고리)Torus + Wireframe 모디파이어 → Z축 회전 키프레임Wireframe 두께 0.01~0.02. Relative ON
D. 파티클 (부유 입자)Emitter → Gravity 0 → Turbulence Force FieldTurbulence Strength 2~5. Lifetime 40~80
E. 트레일 (이동 잔상)Emitter → Gravity 0.1 → Normal 속도 낮게Lifetime 40~80. Size 0.05~0.1

이펙트 프리뷰 영상 만들기

  1. EEVEE 엔진 선택 + Bloom + AO 켜기.
  2. 검은 배경 캐릭터(또는 단순 블록) 배치.
  3. A~E 요소 순서대로 추가. 각각 Emission 재질 + 원하는 색.
  4. Timeline 1~120 프레임 루프 설정.
  5. Space로 재생 확인.
  6. Ctrl+F12로 영상(mp4) 렌더링.

25이펙트 레시피 7종 고급

블렌더에서 바로 만들 수 있는 이펙트 레시피입니다. 단계별로 따라하면 됩니다.

레시피 1 — 회오리 오라

  1. 캐릭터 메시 선택 > Shift+D로 복제.
  2. 복제한 메시에 Simple Deform 모디파이어 추가. Mode: Twist, Angle: 45도.
  3. Solidify 모디파이어 추가. Thickness: 0.015.
  4. 재질: Emission + Layer Weight(Fresnel)로 외곽만 발광. Strength 5~10.
  5. Simple Deform의 Angle 값에 키프레임. 0도 → 360도 → Loop. 비틀리며 회전하는 오라.

레시피 2 — 에너지 구체 (Orb)

  1. UV Sphere 추가. Segments 64, Rings 32.
  2. 재질: Noise Texture(Scale 3, Distortion 2) → ColorRamp(어두운 파랑~밝은 하늘색~흰색) → Emission(Strength 12~18).
  3. Displace 모디파이어 추가. Texture: 같은 Noise, Strength: 0.06. 표면이 살아서 꿈틀거리는 느낌.
  4. Noise Texture의 W 파라미터에 키프레임(0 → 100). 텍스처가 시간에 따라 변화해 구체가 살아있는 것처럼 보임.

레시피 3 — 번개/전기

  1. Bezier Curve 추가. 시작점과 끝점 두 개 배치.
  2. 에디트 모드 > 전체 선택(A) > 우클릭 > Subdivide 5~7번 반복. 중간 점이 많이 생깁니다.
  3. 중간 점들을 선택하고 G로 조금씩 이동. 지그재그 형태 만들기.
  4. Curve 설정 > Geometry > Bevel > Round. Depth: 0.015 (두께).
  5. 재질: Emission, 밝은 파랑/흰색, Strength: 25~40.
  6. 위 과정을 3~5개 복제하고 각각 살짝 다른 경로로 조정 → 번개 다발.

레시피 4 — 마법진

  1. Plane(평면) 추가 > UV 언랩.
  2. 외부 툴(Figma, Photoshop, 또는 무료 온라인 툴)에서 룬 문자/원진 PNG 제작. 흰색 선, 투명 배경. 2048x2048 권장.
  3. 재질: Image Texture(룬 PNG) → Emission. Alpha 출력을 재질 Alpha에 연결.
  4. 재질 설정: Blend Mode > Alpha Blend. Backface Culling 끄기.
  5. Z축으로 납작하게(Scale Z 0.01) → 바닥에 누워있는 마법진.
  6. 회전 키프레임으로 천천히 회전. 여러 개 크기를 다르게 해서 동심원으로 쌓기.

레시피 5 — 파편/분열 이펙트

  1. Preferences에서 Cell Fracture 애드온 활성화.
  2. 분열할 오브젝트 선택 > F3에서 Cell Fracture 검색 > 실행.
  3. Source Limit: 80~150. OK.
  4. 생성된 파편들 전체 선택 > Physics > Rigid Body (Active).
  5. Force Field(Charge, 음수 세기)를 중앙에 배치 > 일정 프레임부터 활성화 → 파편 폭발.
  6. 컴포지터에서 Glare 노드 추가로 파편에 빛 번짐.

레시피 6 — 연기/안개 볼륨

  1. Cube 추가 > 연기 영역 크기로 조절.
  2. 재질 추가 > Shader Editor에서 Surface는 삭제, Volume Scatter 추가.
  3. Volume Scatter: Color 연기 색상, Density 0.15~0.4.
  4. Noise Texture(Scale 2) → Density에 연결. 자연스러운 불균일 연기.
  5. Noise의 W 파라미터에 키프레임(0 → 50). 연기가 흐르는 애니메이션.

레시피 7 — 홀로그램

  1. 메시에 재질 추가.
  2. Emission: 색상 #00FFEE (청록), Strength: 3.
  3. Wave Texture(Rings, Scale 50) → ColorRamp(밝음/어둠 반복) → Alpha에 연결.
  4. 재질 설정: Blend Mode > Alpha Blend. Shadow Mode > None.
  5. Wireframe 모디파이어: Relative Thickness, 두께 0.002. 선 격자 오버레이 추가.
  6. 두 재질(홀로그램 + 선 격자)을 Mix Shader로 합성. 비율은 0.5~0.7.
ADV

레시피들을 조합하면 더 다양한 이펙트가 됩니다. 오라(레시피1) + 에너지 구체(레시피2) + 파티클 = 솔스RNG "신 오라" 느낌. 번개(레시피3) + 마법진(레시피4) = 소환 이펙트.

26Studio 이펙트 구현 고급

블렌더에서 만든 이펙트 컨셉을 Roblox Studio에서 실제로 구현하는 방법.

Beam (빔) 속성

  • Attachment0 / Attachment1: 빔의 시작점과 끝점. Part에 Attachment를 두 개 만들어야 합니다.
  • Width0 / Width1: 양 끝 폭. 0은 뾰족, 1~3은 균일한 두께.
  • LightEmission: 1로 설정하면 빔이 배경을 통과해 빛납니다.
  • CurveSize0 / CurveSize1: 빔 휘어짐. 번개처럼 구불구불하게 할 때.
  • FaceCamera: 켜면 항상 카메라를 향합니다.

Trail (트레일) 속성

  • Attachment0 / Attachment1: 추적할 두 점. 오브젝트 앞뒤에 배치.
  • Lifetime: 트레일이 유지되는 시간(초). 0.3~1.5.
  • MinLength: 이 거리 이상 이동해야 트레일 생성.
  • LightEmission: 트레일 발광 강도.

-- 오라 ON/OFF 모듈 예시
local AuraModule = {}

function AuraModule.Enable(character, color)
    local fx = character:FindFirstChild("AuraEffects")
    if not fx then return end

    for _, obj in ipairs(fx:GetDescendants()) do
        if obj:IsA("ParticleEmitter") then
            obj.Enabled = true
            obj.Color = ColorSequence.new(color)
        elseif obj:IsA("Beam") or obj:IsA("Trail") then
            obj.Enabled = true
        end
    end
end

function AuraModule.Disable(character)
    local fx = character:FindFirstChild("AuraEffects")
    if not fx then return end

    for _, obj in ipairs(fx:GetDescendants()) do
        if obj:IsA("ParticleEmitter")
        or obj:IsA("Beam")
        or obj:IsA("Trail") then
            obj.Enabled = false
        end
    end
end

return AuraModule

27Geometry Nodes 고급

노드로 지오메트리(형태)를 자동으로 생성하고 변형하는 시스템. 반복 패턴, 자동 배치에 강력합니다.

Geometry Nodes란?

Geometry Nodes는 노드(블록)를 연결해서 메시의 형태를 프로그래밍 방식으로 만드는 기능입니다. 예를 들어 "표면의 모든 점에 나무를 배치하라"처럼 규칙 기반으로 복잡한 형태를 만들 수 있습니다.

Geometry Nodes 추가

오브젝트 선택 > Properties > Modifier > Add Modifier > Geometry Nodes. 노드 에디터가 열립니다.

자주 쓰는 노드

노드하는 일활용
Instance on Points각 점에 오브젝트 배치표면에 돌, 나무, 스파이크 배치
Mesh to Points메시의 버텍스를 점 클라우드로 변환Instance on Points와 함께
Set Position각 점의 위치 변경Noise로 울퉁불퉁하게
Curve to Mesh커브에 단면을 씌워 메시 생성파이프, 케이블 자동 생성
Resample Curve커브의 점 개수 조절균일한 간격으로 나누기
Random Value랜덤 숫자 생성크기, 회전, 위치를 랜덤하게
Scene Time현재 프레임/시간 값시간 기반 자동 애니메이션
Realize Instances인스턴스를 실제 메시로 변환FBX 내보내기 전 필수

간단한 예시: 타워 표면에 스파이크 배치

  1. 타워 메시 선택 > Geometry Nodes 모디파이어 추가.
  2. Mesh to Points 노드 → Instance on Points 노드 연결.
  3. Instance on Points의 Instance 입력에 스파이크 메시 오브젝트 연결 (Object Info 노드 사용).
  4. Random Value 노드로 각 스파이크의 회전/크기 랜덤화.
!!

Geometry Nodes로 만든 인스턴스는 FBX로 내보내기 전에 Realize Instances 노드를 통과시키거나, 모디파이어를 Apply해야 합니다. 안 하면 빈 메시로 내보내집니다.

28프로시저럴 텍스처링 고급

이미지 파일 없이 노드만으로 텍스처를 만드는 방법. 에너지 패턴, 금속 스크래치, 마법 재질에 적합합니다.

프로시저럴 텍스처란?

이미지 파일을 쓰는 대신, 수학적으로 생성된 패턴을 텍스처로 쓰는 방법입니다. 해상도에 상관없이 깔끔하고, 파라미터 하나로 패턴 전체를 바꿀 수 있습니다.

주요 텍스처 노드

노드패턴이펙트 활용
Noise Texture구름/안개 같은 불규칙 패턴에너지 구체 표면, 거친 금속, 연기
Voronoi Texture세포 분열 같은 패턴에너지 균열, 세포 조각, 마법 문양
Wave Texture물결 또는 링 패턴마법 에너지 링, 홀로그램 스캔 라인
Gradient Texture그라데이션거리 기반 색상 전환
Magic Texture사이키델릭 패턴마법 인쇄, 차원 이동 이펙트

레시피: 에너지 오라 재질 (완전 프로시저럴)

  1. Layer Weight 노드: Fresnel 출력. 시선 각도에 따라 외곽이 밝아짐.
  2. Noise Texture(Scale 8, Detail 12): Fac 출력.
  3. ColorRamp: 검정~원하는 오라 색~흰색. 날카롭게 설정.
  4. Math(Multiply): Fresnel 값 * ColorRamp 출력.
  5. Emission: Strength 5~10.
  6. Mix Shader: Transparent BSDF + Emission. Fac = 위 결과.
  7. 재질 설정: Blend Mode > Alpha Blend.

29Python 자동화 고급

반복 작업을 자동화하는 Python 스크립팅. FBX 일괄 내보내기, 오브젝트 자동 배치 등.

Python 스크립팅이란?

블렌더는 Python 언어로 거의 모든 기능을 제어할 수 있습니다. 상단 Scripting 워크스페이스에서 코드를 작성하고 실행합니다. 같은 작업을 100번 반복해야 할 때, 또는 복잡한 패턴을 자동으로 만들 때 씁니다.

TIP

Info 에디터(상단 메뉴바를 넓혀서 에디터 타입을 Info로 변경)에서 블렌더에서 한 모든 작업이 Python 코드로 기록됩니다. "이 버튼 눌렀을 때 Python 코드가 뭐야?" 를 확인하는 가장 쉬운 방법입니다.

기본 문법 예시

import bpy

# 현재 선택된 오브젝트 이름 출력
obj = bpy.context.active_object
if obj:
    print(obj.name)

# 씬의 모든 오브젝트 목록 출력
for o in bpy.data.objects:
    print(o.name, o.type)

# 큐브 추가
bpy.ops.mesh.primitive_cube_add(location=(0, 0, 2))

실용 예시 1: 오브젝트 원형으로 배열

import bpy, math

obj = bpy.context.active_object  # 선택한 오브젝트
count = 12       # 배열 개수
radius = 5.0    # 원 반지름

for i in range(count):
    angle = (2 * math.pi / count) * i
    x = radius * math.cos(angle)
    y = radius * math.sin(angle)
    new_obj = obj.copy()
    new_obj.data = obj.data.copy()
    new_obj.location = (x, y, 0)
    bpy.context.scene.collection.objects.link(new_obj)

실용 예시 2: FBX 일괄 내보내기

import bpy, os

output_dir = "C:/MyExports/"  # 저장 경로
os.makedirs(output_dir, exist_ok=True)

for obj in bpy.data.objects:
    if obj.type != 'MESH':
        continue
    bpy.ops.object.select_all(action='DESELECT')
    obj.select_set(True)
    bpy.context.view_layer.objects.active = obj
    path = os.path.join(output_dir, obj.name + ".fbx")
    bpy.ops.export_scene.fbx(
        filepath=path,
        use_selection=True,
        apply_scale_options='FBX_SCALE_NONE',
        mesh_smooth_type='FACE'
    )
    print(f"저장됨: {path}")

30최적화와 성능 고급

게임 에셋을 최적화해서 로블록스 게임 성능을 유지하는 방법.

왜 최적화가 필요한가?

폴리곤(면)이 너무 많거나, 텍스처 해상도가 너무 높으면 게임이 느려집니다. 특히 타워디펜스 게임은 같은 타워가 여러 개 배치되므로 하나의 에셋이 가벼울수록 전체 게임이 쾌적합니다.

폴리곤 예산 (TD 타워 기준)

타입트라이앵글 목표텍스처 해상도비고
작은 소품 (장식용)200~800256x256많이 배치됨
기본 타워800~2,000512x512여러 개 배치됨
특수 타워 (이펙트 있음)2,000~5,000512~1024화면에 제한적
보스 몬스터3,000~8,0001024x1024화면에 하나씩
배경 지형1,000~4,000512x512 타일이동 없는 배경

블렌더에서 폴리곤 줄이기

  • Decimate 모디파이어: Ratio를 0.5로 설정하면 폴리곤 수가 절반으로 줄어듭니다. Planar 모드는 평평한 면을 자동으로 통합.
  • Merge by Distance: 에디트 모드 > Mesh > Merge by Distance. 겹쳐있는 버텍스 자동 제거.
  • Remove Doubles: F3에서 검색. 중복 버텍스 제거.
  • Dissolve Edges: 불필요한 내부 엣지 선택 후 Ctrl+X로 제거. 폴리곤 수 감소.

텍스처 최적화

  • 256x256 또는 512x512가 대부분의 에셋에 충분합니다.
  • PNG는 투명도가 필요할 때, JPEG는 불필요할 때. JPEG가 용량이 훨씬 작습니다.
  • 여러 오브젝트를 같은 텍스처 하나로 처리하는 텍스처 아틀라스(Atlas)를 쓰면 draw call이 줄어 성능 향상.

Studio 이펙트 최적화

  • ParticleEmitter Rate: 너무 높으면 프레임 드랍. 타워당 Rate 합계 100~200 이하 유지.
  • Enabled 관리: 화면 밖 타워의 이펙트는 Enabled = false로.
  • LightInfluence 0: 1이면 조명 계산이 추가됨. 이펙트 파티클은 0으로 설정.
  • 텍스처 공유: 같은 이미지를 여러 이펙트가 공유하면 메모리 절약.
RBX

Studio의 MicroProfiler(Ctrl+F6)로 성능을 측정할 수 있습니다. 이펙트가 많은 씬에서 GPU 시간이 높으면 ParticleEmitter Rate를 줄이거나 Enabled 관리를 해야 합니다.
