21파티클 시스템 심화 고급

Emitter와 Hair 파티클의 세밀한 제어. 포스 필드와 충돌 설정까지.

파티클 시스템 추가

오브젝트 선택 > Properties > 파티클 아이콘(점 세 개) > + > New.

Emitter vs Hair

Hair표면에서 자라나는 가닥. 정적머리카락, 풀, 털, 가시

포스 필드 (Force Field)

Shift+A > Force Field 로 추가. 파티클에 힘을 줍니다.

종류효과활용

파티클 충돌 (Collision)

파티클이 다른 오브젝트와 충돌하게 하려면 해당 오브젝트에 Physics > Collision을 추가합니다. Particle Damping으로 충돌 후 감속 정도를 조절합니다.

Hair 파티클 + 커브 편집

  1. Hair 파티클 추가 후 Children(자식 가닥)을 설정하면 가닥 수가 늘어납니다. Simple: 빠름, Interpolated: 자연스러움.
  2. Particle Edit 모드(Tab 전환)에서 Comb, Cut, Add, Smooth 도구로 각 가닥을 직접 다듬습니다.
  3. Clump, Roughness 탭에서 뭉치거나 불규칙하게 만들 수 있습니다.
TIP

작업 중 파티클이 너무 많아 뷰포트가 느리면, 파티클 설정 > Viewport Display > Display As를 Dot으로, Amount를 10~20%로 낮추세요. 렌더 시엔 자동으로 원래대로 됩니다.

22물리 시뮬레이션 고급

Cloth, Rigid Body, Soft Body. 시뮬레이션 결과를 베이크하고 메시로 굳히는 방법까지.

Cloth (천 시뮬레이션)

Properties > Physics > Cloth. 깃발, 망토, 천, 커튼에 사용합니다.

설정설명재질별 추천
Vertex Mass천의 무게 (kg)실크 0.05 / 면 0.3 / 가죽 1.0
Tension Stiffness늘어남 저항높을수록 탄탄. 가죽: 50+
Bending Stiffness구부러짐 저항낮을수록 부드럽게 흐름
Pinning고정 버텍스 그룹 지정깃발 한쪽 끝, 옷 어깨 부위
Self Collision천이 자기 자신과 충돌두꺼운 옷, 쌓이는 처마 치마
!!

충돌면 오브젝트에도 Physics > Collision을 반드시 추가해야 천이 그 표면에서 멈춥니다. 없으면 통과합니다.

Rigid Body (강체)

  • Active: 중력과 충돌 영향을 받는 오브젝트. 파편, 쓰러지는 오브젝트.
  • Passive: 움직이지 않는 충돌면. 바닥, 벽.
  • Collision Shape: Convex Hull(빠름, 대부분 OK) / Mesh(정확, 느림) / Box,Sphere(단순 형태).
  • Scene > Rigid Body World > Bake: 시뮬레이션 결과를 미리 계산해서 저장. 편집 중 재생이 빠릅니다.

Soft Body

물렁물렁하게 변형되는 물체를 표현합니다. 젤리, 쿠션, 물풍선 등. Physics > Soft Body. Goal로 원래 형태 유지 강도, Edges로 가장자리 탄성을 조절합니다.

Cell Fracture — 파편 분열

  1. Preferences > Add-ons에서 Cell Fracture 활성화.
  2. 분열시킬 메시 선택 > F3 > Cell Fracture 검색 > 실행.
  3. Source Limit: 파편 수(50~200). 높을수록 촘촘하게 분열.
  4. 생성된 파편 전체 선택 > Physics > Rigid Body (Active).
  5. 중앙에 Force Field(Magnetic, 음수값)를 배치 > 일정 프레임에서 켜기 → 폭발.

시뮬레이션 베이크와 메시 굳히기

원하는 형태가 됐을 때 오브젝트의 모디파이어를 Apply(적용)하면 시뮬레이션 결과가 정적 메시가 됩니다. FBX 내보내기 전에 이 작업이 필요합니다.

23유체 시뮬레이션 고급

Mantaflow 엔진 기반 연기, 불꽃, 액체 시뮬레이션.

유체 시뮬레이션 구조

유체 시뮬레이션은 반드시 Domain(영역) + Flow(소스) 구조로 만들어야 합니다.

  • Domain: 시뮬레이션이 계산되는 공간. Cube를 전체 범위로 배치. 안에 있는 것만 계산됩니다.
  • Flow: 유체가 나오는 소스. 불꽃 기둥, 연기 출구, 물이 쏟아지는 부분.
  • Effector: 유체와 충돌하는 오브젝트.

연기/불꽃 만들기

  1. Cube(큰 것) 추가 > Physics > Fluid > Type: Domain. Domain Type: Gas.
  2. Resolution: 64~128(작업 중), 256+(최종 렌더).
  3. 작은 Cylinder 추가 > Physics > Fluid > Type: Flow. Flow Type: Fire + Smoke.
  4. Flow Source: Mesh 또는 Surface. Fuel(연료량)과 Temperature(온도)로 불꽃 크기 조절.
  5. Domain 선택 > Bake Data. 프레임이 계산됩니다.

액체 만들기

  1. Domain: Fluid Type Liquid.
  2. Flow: Flow Type Liquid. Flow Behavior: Geometry(채워진 상태) or Inflow(계속 흘러나옴).
  3. Domain > Mesh 탭: Is Resumable 켜기. Bake Mesh로 최종 표면 생성.
  4. Viscosity(점성): 0이면 물, 높을수록 꿀/용암.
!!

유체 시뮬레이션은 컴퓨터 사양을 많이 탑니다. Domain Resolution을 작업 중엔 64, 최종 렌더할 때만 256 이상으로 올리세요. Bake 파일은 별도 폴더에 저장되므로 디스크 공간도 확인하세요.

Volume 재질 (연기 색상)

Domain 오브젝트의 재질 > Shader Editor에서 Principled Volume 노드 사용. Color로 연기 색상, Density로 농도, Temperature로 불꽃 밝기를 조절합니다.

24Cycles 렌더링 심화 고급

Cycles의 물리 기반 렌더링 원리와 노이즈 제거, 최적화 설정.

Cycles 작동 원리

Cycles는 경로 추적(Path Tracing) 방식입니다. 카메라에서 빛 경로를 수천 번 역추적해 물리적으로 정확한 빛 계산을 합니다. 샘플 수가 많을수록 노이즈가 줄고 시간이 오래 걸립니다.

샘플링 설정

설정설명권장값
Render Samples렌더 시 샘플 수. 많을수록 노이즈 감소128~512 (복잡도에 따라)
Viewport Samples뷰포트 미리보기 샘플32~64
Noise Threshold이 이하로 노이즈가 줄면 자동 종료0.01 (빠름) ~ 0.001 (정확)
Adaptive Sampling밝고 단순한 영역은 샘플 자동 감소켬 권장
Light Paths: Max Bounces빛이 반사되는 최대 횟수Total 12, Diffuse 4, Specular 4

디노이저 (Denoiser)

렌더 후 AI가 노이즈를 자동으로 제거해 주는 기능입니다. 샘플 수를 낮게 해도 깔끔한 결과를 얻을 수 있어 렌더 시간이 크게 줄어듭니다.

  • Render Properties > Sampling > Denoise: 켜기. OpenImageDenoise(CPU) 또는 OptiX(Nvidia GPU).
  • Compositing에서 Denoise 노드: 더 세밀한 제어 가능.
  • 투명한 재질(유리, 물)이 많은 씬은 디노이저가 부정확할 수 있으니 샘플 수를 직접 올리는 게 낫습니다.

렌더 최적화

  • GPU 렌더링: Preferences > System > Cycles Render Devices에서 GPU 선택. CPU보다 5~20배 빠릅니다.
  • Tile Size: GPU 렌더 시 256x256, CPU 렌더 시 32x32 권장.
  • Light Paths 줄이기: 실내 장면이 아니라면 Diffuse Bounces를 2~3으로 줄여도 큰 차이 없습니다.
  • Caustics 끄기: Light Path > Caustics를 끄면 속도 향상. 물/유리 특수 반사가 없는 씬에서 유효.

HDRI + World 조명

  1. Shading 워크스페이스 > 에디터 타입 드롭다운 > World로 전환.
  2. Use Nodes > Environment Texture 노드 추가 > HDRI 파일(.hdr/.exr) 불러오기.
  3. Mapping 노드의 Rotation Z값으로 환경 회전 조절.
  4. World Output 앞에 Background 노드의 Strength로 환경광 강도 조절.

렌더 레이어 (Render Passes)

Cycles는 한 번의 렌더로 Diffuse(색), Shadow(그림자), Emission(발광), Normal(면 방향) 등을 각각 별도 이미지로 저장할 수 있습니다. 컴포지팅에서 레이어별로 후보정할 때 필수입니다.

Render Properties > Passes 탭에서 필요한 것을 켜세요.

25컴포지터 고급

렌더된 이미지를 노드로 보정하는 편집기. 글로우, 색보정, 렌즈 왜곡 등.

컴포지터란?

렌더 결과물(이미지/영상)에 후처리 효과를 추가하는 노드 기반 편집기입니다. 상단 Compositing 워크스페이스에서 Use Nodes를 켜면 됩니다.

기본 구조: Render Layers(입력) → 처리 노드들 → Composite(출력)

자주 쓰는 컴포지터 노드

노드효과활용
Glare밝은 부분에 빛 번짐/별 모양 효과조명, 태양, 발광 오브젝트
Bloom (Glare > Bloom)밝은 부분 퍼짐 (EEVEE Bloom과 동일 효과)발광 재질
Color BalanceLift/Gamma/Gain으로 색 보정영화 느낌 색보정

Blur블러 처리피사계심도(DOF) 수동 제어
Defocus피사계심도 블러배경 흐림 효과

렌더 패스 활용 (심화)

렌더 레이어에서 각 패스를 켜두면 컴포지터에서 Render Layers 노드의 출력 소켓에 Color, Diffuse, Specular, Shadow, AO 등이 각각 나타납니다. 이것들을 Mix RGB 노드로 조합해서 원하는 느낌으로 합성합니다.

  • AO를 곱하기: Diffuse * AO → 그림자가 더 깊어지는 느낌.
  • Emission만 Glare 처리: 발광 부분만 빛 번짐 적용. 전체가 번지지 않음.
  • Shadow를 별도 조절: Shadow 패스 밝기를 낮춰 그림자를 더 진하게.

27Geometry Nodes 심화 고급

프로시저럴 모델링의 핵심. 노드로 복잡한 형태를 자동으로 생성합니다.

Geometry Nodes란?

노드를 연결해서 지오메트리(형태)를 프로그래밍처럼 생성하는 시스템입니다. 규칙을 만들어 두면 파라미터 하나로 전혀 다른 형태를 만들 수 있습니다.

추가 방법: Properties > Modifier > Add Modifier > Geometry Nodes.

Random Value랜덤 숫자/벡터/색상 생성크기, 회전, 위치 랜덤화

Group Input으로 파라미터 제어

노드 에디터에서 Group Input을 추가하고 속성 이름을 정하면, Properties 패널 Modifier에 슬라이더가 생깁니다. 블렌더 파일을 열었을 때 코드 없이 형태를 실시간으로 조정할 수 있습니다.

실용 예시: 건물 랜덤 높이 생성

  1. Grid Mesh(10x10) 추가 > Geometry Nodes 모디파이어.
  2. Mesh Faces 선택 > Extrude Mesh 노드.
  3. Random Value(Float, min 1, max 8) → Extrude의 Offset Scale에 연결.
  4. 각 면이 랜덤 높이로 돌출됩니다. 씬을 새로 고칠 때마다 다른 건물 배치.
!!

Geometry Nodes로 만든 인스턴스는 FBX 내보내기 전에 반드시 Realize Instances 노드를 추가하거나 모디파이어를 Apply해야 합니다. 안 하면 메시가 비어있는 상태로 내보내집니다.

28프로시저럴 텍스처링 고급

이미지 파일 없이 노드로 텍스처를 만드는 방법. 무한 해상도, 파라미터 기반 변형이 가능합니다.

프로시저럴 텍스처의 장점

  • UV 왜곡이 없습니다. 어떤 각도에서 봐도 올바르게 보입니다.
  • 파라미터 하나로 완전히 다른 패턴으로 바꿀 수 있습니다.
  • 이미지 파일이 없으므로 용량이 작습니다.
  • 3D 공간 기반으로 계산되므로 오브젝트를 잘라봐도 내부까지 패턴이 이어집니다.

주요 텍스처 노드와 활용

노드패턴 특징현실 소재 표현
Noise Texture구름 같은 불규칙 패턴. Scale, Detail, Roughness, Distortion으로 제어대리석 결, 구름, 피부, 흙
Voronoi Texture세포 분열 패턴. Distance(거리) 또는 Color 출력뱀 비늘, 거북 등껍질, 조약돌, 균열
Wave Texture물결 또는 동심원(Rings) 패턴나무 나이테, 물결, 음파
Musgrave Texturefractal 노이즈. Noise와 비슷하지만 더 세밀산, 거친 지형, 낡은 금속
Gradient Texture거리 기반 그라데이션그라데이션 색상 전환, Fresnel 흉내

대리석 재질 레시피 (노드 연결 순서)

  1. Texture Coordinate(Object) → Mapping(Scale Z 5~10으로 줄무늬 방향 설정).
  2. Wave Texture(Bands, Distortion 2.5, Scale 5) → Color Ramp(검정~흰색 대비).
  3. Noise Texture(Scale 12, Detail 16) → Wave의 Distortion에 연결. 결이 불규칙해짐.
  4. Color Ramp 출력 → Principled BSDF Base Color.
  5. Metallic 0, Roughness 0.08. 매끄럽고 반사되는 대리석.

29셰이더 노드 고급 고급

Principled BSDF를 넘어서 커스텀 BSDF 조합과 특수 재질 표현.

Principled BSDF 고급 파라미터

파라미터설명활용
Subsurface Weight빛이 표면 안쪽을 뚫고 산란되는 효과피부(0.1~0.3), 밀랍(0.5), 옥(0.7)
Specular Tint반사광에 Base Color 색상 반영 정도과일, 플라스틱의 색감 있는 하이라이트
Anisotropic방향성 있는 반사 (방사형 긁힘)헤어라인 금속, LP 판, 솔질한 알루미늄
Sheen천의 가장자리 솜털 산란벨벳, 청바지, 극세사
Coat Weight / Roughness위에 투명 코팅층 추가자동차 페인트, 광택 나무
IOR (굴절률)빛이 꺾이는 정도. 진공=1.0유리 1.45 / 물 1.33 / 다이아몬드 2.42
Transmission Weight투과도. 빛이 통과하는 비율유리(1.0), 반투명 플라스틱(0.3~0.7)

커스텀 셰이더 조합

Principled BSDF 하나로 안 되는 재질은 기본 BSDF들을 Mix Shader로 조합합니다:

  • 양면 재질: Geometry 노드의 Backfacing → Mix Shader Fac. 앞면/뒷면 다른 재질.
  • 마모 효과: AO 텍스처(어두운 부분) → Mix Shader. 구석은 녹슬고 돌출부는 반짝임.
  • 얼음: Principled BSDF(Transmission 1, IOR 1.31, Roughness 0.05) + Volume Scatter(매우 낮은 Density).
  • 홀로그램: Emission + Transparent BSDF, Wave Texture를 Fac으로 사용. Blend Mode Alpha Blend.

Normal Map vs Bump Map

방식특징Principled BSDF 연결
Normal MapRGB 이미지. 베이킹으로 생성. 정확함Image Texture(Non-Color) → Normal Map 노드 → Normal
Bump Map흑백 이미지(높이 정보). 단순하지만 빠름Texture → Bump 노드(Strength 조절) → Normal
Displacement실제로 표면을 올록볼록하게. Cycles 전용Texture → Displacement 노드 → Material Output의 Displacement

30Python 자동화 고급

반복 작업을 자동화하는 Python 스크립팅. 상단 Scripting 워크스페이스에서 작성합니다.

bpy 기초 구조

import bpy

# 데이터 접근 경로
bpy.context      # 현재 상태 (선택, 모드, 뷰포트 등)
bpy.data         # 모든 데이터 (오브젝트, 재질, 메시 등)
bpy.ops          # 연산(오퍼레이터). 메뉴 버튼과 동일
bpy.types        # 타입 정의 (애드온 개발용)

# 현재 선택 오브젝트
obj = bpy.context.active_object
print(obj.name, obj.type)

# 씬의 모든 오브젝트 순회
for o in bpy.data.objects:
    print(o.name)

실용 예시 1: 원형 배열 자동 생성

import bpy, math

base = bpy.context.active_object  # 복제할 오브젝트 선택 후 실행
count  = 12
radius = 6.0

for i in range(count):
    angle = (math.pi * 2 / count) * i
    new = base.copy()
    new.data = base.data.copy()
    new.location = (radius * math.cos(angle),
                    radius * math.sin(angle), 0)
    bpy.context.collection.objects.link(new)

실용 예시 2: FBX 일괄 내보내기

import bpy, os

out = r"C:\Exports"
os.makedirs(out, exist_ok=True)

for obj in bpy.data.objects:
    if obj.type != 'MESH': continue
    bpy.ops.object.select_all(action='DESELECT')
    obj.select_set(True)
    bpy.context.view_layer.objects.active = obj
    bpy.ops.export_scene.fbx(
        filepath=os.path.join(out, obj.name + ".fbx"),
        use_selection=True,
        apply_scale_options='FBX_SCALE_NONE',
        mesh_smooth_type='FACE'
    )
    print(f"내보내기 완료: {obj.name}")

실용 예시 3: 재질 일괄 교체

import bpy

old_name = "Material.001"  # 교체할 재질 이름
new_mat  = bpy.data.materials["FinalMaterial"]

for obj in bpy.data.objects:
    if obj.type != 'MESH': continue
    for i, slot in enumerate(obj.material_slots):
        if slot.material and slot.material.name == old_name:
            obj.material_slots[i].material = new_mat
            print(f"{obj.name}: 재질 교체됨")

EX실전 — 게임 스킬 이펙트 메시 실전

킹 레가시(킹피스), 블록스 프루츠 스타일 스킬을 구성하는 이펙트 메시 5종을 블렌더로 만듭니다. 전부 같은 원리 — 단순한 메시 + 알파 그라데이션 텍스처입니다.

스킬 이펙트의 공통 원리

게임 속 화려한 스킬은 복잡한 시뮬레이션이 아니라, 블렌더에서 만든 아주 단순한 메시에 투명 그라데이션 텍스처를 입히고 Studio에서 코드로 키우고 돌리고 사라지게 하는 것입니다. 폴리곤이 적을수록 좋습니다 — 이펙트는 0.5초 만에 사라지므로 디테일보다 실루엣과 타이밍이 전부입니다.

예시 1 — 검기 (슬래시)

킹 레가시, 블록스 프루츠의 칼 휘두를 때 나가는 그 검기입니다. 모든 이펙트 메시의 출발점.

  1. Shift+A > Mesh > Torus. Major Segments 48, Minor Segments 3, Minor Radius 0.05로 얇게.
  2. 에디트 모드(Tab)에서 토러스의 2/3를 박스 선택해 삭제 — 남은 1/3이 초승달 호.
  3. 양 끝을 비례 편집(O)으로 모아 뾰족하게. 검기 끝이 날카로워집니다.
  4. 위에서 봤을 때 살짝 납작하게 S > Z.
  5. UV가 핵심: U > Unwrap 후, UV 에디터에서 호가 일자로 펴진 아일랜드를 회전시켜 호의 길이 방향이 U축(가로)이 되게 정렬합니다. 그래야 가로 그라데이션 텍스처가 검기의 머리에서 꼬리로 흐릅니다.
  6. 텍스처: 256x64 PNG, 왼쪽 흰색(불투명) → 오른쪽 알파 0(투명). 위아래 가장자리도 투명해지게 하면 더 부드럽습니다.
  7. Ctrl+A > All Transforms 적용 후 FBX 내보내기.
TIP

더 간단한 방법: Plane에 Simple Deform 모디파이어(Bend, 120도)를 걸어도 검기 호가 됩니다. 폴리곤이 더 적어 가볍습니다. 내보내기 전 Apply 필수.

예시 2 — 에너지 구체 (오라 스피어)

킹피스의 차지 구체, 발사체 구체의 기본입니다.

  1. Shift+A > Mesh > UV Sphere. Segments 16, Rings 8이면 충분합니다.
  2. 같은 구를 하나 더 복제(Shift+D)하고 1.3배 키웁니다. 안쪽이 코어, 바깥쪽이 글로우.
  3. UV는 기본 Sphere Projection 그대로 사용. 텍스처는 가운데가 흰색이고 위아래 극지방으로 갈수록 투명해지는 세로 그라데이션.
  4. 두 구를 Ctrl+J로 합치지 말고 따로 내보내세요. Studio에서 코어는 흰색, 글로우는 스킬 색(파랑/보라)으로 각각 칠하고 다른 속도로 회전시키면 살아있는 느낌이 납니다.

예시 3 — 빔 / 레이저

  1. Shift+A > Mesh > Cylinder. Vertices 12. Cap Fill Type을 Nothing으로 — 양끝 뚜껑이 없어야 안이 비쳐 보입니다.
  2. S > Z로 길게 늘립니다 (길이 10, 두께 0.3 비율).
  3. 에디트 모드 > 옆면 세로 엣지 하나에 Mark Seam > Unwrap. 원통이 직사각형으로 펴집니다.
  4. UV에서 원통의 길이 방향을 U축(가로)에 맞춥니다. 가로로 반복되는 에너지 줄무늬 텍스처를 쓰면 Studio에서 Texture의 OffsetStudsU를 Tween해서 에너지가 흐르는 연출이 됩니다.
  5. 끝부분 버텍스를 비례 편집으로 살짝 모아 뾰족하게 하면 레이저 끝이 자연스럽습니다.
TIP

굵기가 다른 원통 2~3겹(코어 가늘고 흰색 + 바깥 굵고 색깔)을 겹치면 킹피스 레이저 특유의 속이 빛나는 빔이 됩니다. 검기와 같은 겹치기 공식입니다.

예시 4 — 충격파 링

스킬이 땅에 꽂힐 때 퍼지는 둥근 파동입니다.

  1. Shift+A > Mesh > Torus. Major Segments 32, Minor Segments 4, Minor Radius 0.15.
  2. S > Z > 0.3 으로 납작하게 눌러 도넛이 아닌 파동 링 형태로.
  3. Unwrap 후, 링의 안쪽이 밝고 바깥쪽이 투명해지는 그라데이션 텍스처.
  4. Studio에서 Size를 0.2배 → 6배로 0.4초 Tween + Transparency 0 → 1. 끝.

예시 5 — 마법진

  1. Shift+A > Mesh > Plane. 이게 전부입니다. 폴리곤 1개.
  2. UV는 기본 그대로. 마법진 그림(원형 문양, 룬 문자, 기하 패턴)을 투명 배경 PNG로 준비해서 통째로 입힙니다.
  3. 마법진 텍스처는 직접 그려도 되고, 블렌더 셰이더 노드(Wave + Voronoi 조합)로 만들어 베이킹해도 됩니다 — 챕터 28 프로시저럴 텍스처링 참고.
  4. Studio에서 바닥에 깔고 Y축 회전만 계속 돌리면 차지 연출 완성. 위로 올라가는 링을 함께 띄우면 소환 느낌.

예시 6 — 회오리 (토네이도)

  1. Shift+A > Mesh > Plane > 에디트 모드에서 세로로 길고 가는 직사각형으로.
  2. 모디파이어 > Screw 추가. Axis Z, Screw 값 4~6, Iterations 2. 직사각형이 나선 리본으로 감깁니다.
  3. Angle을 720도로 하면 두 바퀴 감긴 회오리. 아래는 좁고 위는 넓게 — Simple Deform(Taper)을 겹치면 됩니다.
  4. 아래쪽이 진하고 위로 갈수록 투명한 세로 그라데이션 텍스처.
  5. 내보내기 전 Screw 모디파이어 Apply 필수.

킹피스 스킬 조합 공식

실제 게임 스킬 하나는 이 부품들의 시간차 조합입니다:

  1. 0.0초 — 차지: 발밑에 마법진 회전 + 손에 에너지 구체 맥동.
  2. 0.5초 — 발사: 구체가 커지며 빔 발사. 빔 텍스처 스크롤.
  3. 0.8초 — 히트: 명중 지점에 충격파 링 2겹(시간차 0.1초) + 파티클 폭발.
  4. 1.0초 — 여운: 모든 메시 Transparency 페이드 + Debris 정리.

!!

이펙트 메시는 전부 Cap/뚜껑 제거, 폴리곤 최소화, 모디파이어 Apply 후 내보내기. 한 스킬에 메시가 5~6개씩 쓰이므로 하나하나가 가벼워야 합니다.
