01설치와 첫 화면 입문

무료입니다. 전부 다. 평생.

  1. blender.org/download에서 최신 안정 버전 다운로드. Windows / macOS / Linux 전부 지원.
  2. 설치 후 첫 실행 시 언어를 영어(English)로 권장합니다. 전 세계 튜토리얼과 포럼이 영어 메뉴 기준이라 검색이 100배 빨라집니다.
  3. 첫 화면에 큐브, 카메라, 라이트가 놓여 있으면 성공.

Preferences 필수 설정

Edit > Preferences를 열고 다음을 확인하세요:

  • Input > Emulate Numpad: 넘패드 없는 노트북이면 체크. 상단 숫자키가 넘패드처럼 동작.
  • Input > Emulate 3 Button Mouse: 2버튼 마우스/트랙패드면 체크. Alt+클릭이 MMB 대체.
  • System > Undo Steps: 기본 32를 64 이상으로. 실수 복구 여유.
  • Save & Load > Auto Save: 2~5분 간격 권장. 크래시 대비.
  • Add-ons: Node Wrangler를 검색해서 활성화. 셰이더 작업 필수 애드온.

당장 외울 단축키 3개만:

  • Ctrl+Z — 실행 취소.
  • F3 — 기능 이름 검색. 단축키가 생각 안 나면 여기에 영어로 타이핑.
  • Ctrl+S — 저장. 5분에 한 번.

02뷰포트 조작 — 카메라를 손에 붙이기 입문

모델링 실력의 절반은 시점 이동 속도입니다.

마우스 3종 조합

넘패드 시점 전환

Gizmo와 뷰포트 헤더

뷰포트 우상단에 반투명 축 기즈모가 있습니다. 축 글자(X/Y/Z)를 클릭하면 해당 방향 정면 시점으로 바로 전환됩니다. 드래그하면 자유 회전.

뷰포트 헤더 좌측에는 현재 모드(Object Mode/Edit Mode/Sculpt Mode 등)가 표시되고, 우측에는 셰이딩 모드 아이콘 4개 — Wireframe, Solid, Material Preview, Rendered — 가 나란히 있습니다. 또는 Z로 파이 메뉴.

Walk/Fly 모드

Shift+` (백틱)을 누르면 FPS처럼 WASD로 씬 안을 걸어다닐 수 있습니다. 넓은 맵의 스케일감을 확인할 때 유용. 마우스로 방향, 휠로 속도 조절, Esc로 빠져나옵니다.

!!

길을 잃었을 때: Shift+C = 커서 원점 복귀 + 씬 전체 보기. 넘패드 . = 선택한 오브젝트로 줌. 이 두 가지면 어떤 미아 상태에서도 복귀 가능.

03오브젝트 모드 — 물체 단위로 다루기 입문

블렌더에는 두 세계가 있습니다. 오브젝트 모드는 "물체 통째로", 에디트 모드는 "물체의 표면". Tab으로 오갑니다.

오브젝트 추가와 기본 조작

  • Shift+A — 오브젝트 추가 메뉴 (Mesh > Cube/Cylinder/Sphere 등).
  • X / Delete — 삭제.
  • Shift+D — 복제. 우클릭하면 원래 자리에 복제.
  • Alt+D — 링크 복제. 메시 데이터를 공유하므로 하나를 수정하면 전부 바뀝니다. 같은 타워를 여러 개 배치할 때 메모리 절약.
  • F2 — 이름 변경. TowerBase처럼 영문으로 정리하면 Studio에서 편합니다.
  • F9 — 방금 한 작업의 옵션 패널. Cylinder 추가 직후 F9로 Vertices를 8~12로 줄이면 로우폴리 원기둥.

GRS — 이동 / 회전 / 크기

원점(Origin), 3D 커서, 피봇

오렌지색 점이 원점 — 회전과 크기 변환의 기준. 터렛이 제자리에서 회전하려면 원점이 회전축 중심에 있어야 합니다.

  1. 회전축 위치의 버텍스를 선택하고 Shift+S > Cursor to Selected.
  2. 오브젝트 모드에서 Object > Set Origin > Origin to 3D Cursor.

피봇 포인트(.): 변환의 중심을 개별 원점, 중위점, 3D 커서, 활성 요소 등으로 바꿉니다. 여러 오브젝트를 각자 제자리에서 회전시키려면 Individual Origins.

그 외 오브젝트 모드 필수

  • Ctrl+J — 합치기 (Join).
  • H / Alt+H — 숨기기 / 전부 보이기.
  • 이동 중 Ctrl — 그리드 스냅.
  • Ctrl+A > All Transforms — 내보내기 전 필수. 크기/회전 값을 1과 0으로 굽습니다.
  • Ctrl+P — 부모 설정(Parent). 여러 파츠를 하나의 Empty 아래에 묶을 때.
  • Alt+P — 부모 해제.

05핵심 편집 도구 입문

에디트 모드 안에서 형태를 만드는 핵심 10가지.

E — Extrude (돌출)

I — Inset (내부 면)

면 안에 작은 면을 만듭니다. I 후 바로 E = 안쪽만 돌출하는 가장 흔한 패턴. I를 두 번 누르면 각 면을 개별 인셋.

Ctrl+R — Loop Cut

고리 모양으로 면을 분할. 마우스를 갖다 대면 미리보기가 노란색으로 표시됩니다. 휠로 컷 수 증가, 클릭 후 Esc로 정중앙 고정, 드래그로 위치 조절.

Ctrl+B — Bevel

모서리를 깎아 부드럽게. 마우스로 폭, 휠로 둥글기 단계. Ctrl+Shift+B는 버텍스 베벨. 로우폴리의 마감. 주요 모서리에 1단계만 베벨해도 완성도가 올라갑니다.

나머지 핵심 도구들

K — Knife

마우스로 자유롭게 칼질. Enter로 확정. C로 45도 스냅, Z로 관통 컷.

F — Fill

선택한 버텍스/엣지 사이를 면으로 채우기. 빈 구멍을 덮을 때.

M > By Distance

겹친 버텍스 합치기. 내보내기 전 A 전체 선택 후 반드시 실행.

P — Separate

선택한 면을 별도 오브젝트로 분리. 터렛+베이스 나누기.

GG — Edge Slide

엣지를 기존 면 위로 미끄러뜨려 위치 조정. 루프컷 후 미세 조정에.

Ctrl+F — Face 메뉴

Triangulate, Solidify, Grid Fill 등 면 관련 명령 모음.

Ctrl+E — Edge 메뉴

Mark Seam, Mark Sharp, Crease 등 엣지 속성 설정.

Ctrl+V — Vertex 메뉴

Connect, Bevel, Rip(찢기) 등 버텍스 관련 명령.

노멀(Normal) — 면의 안과 밖

Overlays에서 Face Orientation을 켜면 파란색=앞, 빨간색=뒤. 빨간 면이 보이면 선택 후 Shift+N(Recalculate Outside). Alt+N으로 노멀 메뉴를 열어 Flip(뒤집기)도 가능.

!!

Delete vs Dissolve: X 메뉴에서 Delete는 면째로 뚫고, Dissolve는 지우되 구멍은 안 냅니다. 대부분 Dissolve가 원하는 동작.

07토폴로지와 리토폴로지 중급

폴리곤이 얼마나 깔끔하게 연결되어 있는가 = 토폴로지.

쿼드 vs 트라이 vs N-gon

좋은 토폴로지 원칙

  • 쿼드 위주: Subdivision, Loop Cut, Bevel 모두 쿼드에서 가장 예측 가능.
  • N-gon은 평탄한 면에서만: 곡면에 N-gon이 있으면 셰이딩 깨짐.
  • 폴(Pole): 한 점에 5개 이상 엣지가 모이는 곳. 변형이 적은 부위로 밀어내기.
  • 등간격 엣지 루프: 관절, 접히는 곳은 촘촘하고 균일하게.
  • Edge Flow: 엣지가 형태의 흐름(근육, 곡면, 관절)을 따라가야 변형 시 자연스러움.

리토폴로지

엉망이 된 메시를 깔끔하게 다시 덮는 과정.

  1. 원본을 복제하여 _highpoly로 이름 변경, 숨기기.
  2. 새 Plane을 추가, Shrinkwrap 모디파이어로 원본 표면에 달라붙게 설정.
  3. 스냅을 Face로 설정하고 면을 하나하나 깔아가기.
ADV

Instant Meshes(무료 외부 도구)를 쓰면 하이폴리를 넣고 원하는 폴리곤 수의 쿼드 메시를 자동 생성합니다.

Dyntopo vs Remesh

Dyntopo (Dynamic Topology): 브러시가 닿는 곳만 자동으로 폴리곤을 늘려줍니다. 스컬프트 중 Detail Size로 해상도 조절. 초벌 작업에 적합.

Remesh (Voxel): 전체 메시를 균일한 복셀 그리드로 재구성. 해상도를 직접 지정. Dyntopo로 대충 잡은 뒤 Remesh로 정리하는 루틴이 일반적.

09커브와 서피스 입문

부드러운 곡선이 필요할 때. 로프, 파이프, 레일, 칼날 곡선, 장식 스크롤 등.

커브 추가와 편집

  • Shift+A > Curve > Bezier / Nurbs / Path.
  • 에디트 모드에서 컨트롤 포인트를 G로 이동. 핸들을 드래그해서 곡률 조절.
  • E로 끝점을 연장해서 커브를 늘려갑니다.
  • 핸들 타입: V로 전환 — Auto(부드러운), Vector(꺾이는), Free(자유), Aligned(대칭).

커브를 메시로 활용하기

  • Bevel Depth: 커브 Properties > Data에서 Bevel > Round로 두께를 주면 파이프/튜브가 됩니다.
  • Bevel Object: 별도 프로파일 커브를 Bevel Object에 연결하면 그 단면 모양으로 파이프가 만들어집니다. 검날, 몰딩, 장식 테두리.
  • Array + Curve 모디파이어: 오브젝트를 커브를 따라 반복 배치. 울타리, 체인, 레일.
  • Convert to Mesh: 최종적으로 Object > Convert > Mesh로 변환해야 FBX 내보내기 가능.

텍스트(Text) 오브젝트

Shift+A > Text. Tab으로 에디트 모드에 들어가 글자를 수정합니다. Properties에서 폰트 변경, Extrude로 두께, Bevel로 모서리 마감. 타워 이름 표지판, UI 아이콘 등에. 최종적으로 Convert > Mesh.

TIP

커브를 경로로: 오브젝트에 Follow Path 컨스트레인트를 걸면 커브를 따라 이동하는 애니메이션이 자동 생성됩니다. TD 맵에서 적 유닛의 이동 경로 시각화에 활용.

Shade Flat vs Shade Smooth vs Auto Smooth

우클릭 > Shade Smooth = 각진 면이 부드러운 곡면으로. Shade Flat = 다시 각지게.

Auto Smooth: Blender 4.0+에서는 우클릭 > Shade Auto Smooth. 30도 미만 모서리는 날카롭게, 나머지는 부드럽게 — 로우폴리에 최적. 3.x에서는 Properties > Object Data > Normals > Auto Smooth 체크 + 각도 30~40도.

  4. 불필요한 슬롯은 -로 삭제. 사용되지 않는 재질이 남아 있으면 내보내기 시 혼란.

카메라 설정

  • 넘패드 0 — 카메라 시점으로 전환.
  • N > View > Lock Camera to View 체크 — 카메라를 뷰포트처럼 자유롭게 조작.
  • Properties > Camera > Focal Length: 50mm가 기본. 35mm=넓은 화각(역동적), 85mm=망원(왜곡 적음, 제품 사진).
  • Properties > Camera > Depth of Field: 배경 흐림. f-stop을 낮추면(1.4~2.8) 피사체만 선명한 시네마틱 느낌.

렌더 출력

F12로 렌더. Output Properties에서 해상도(1920x1080 기본), 파일 형식(PNG 권장), 저장 경로 설정. Ctrl+F12는 애니메이션 렌더(전 프레임 순차 렌더).

Eevee 이펙트 세팅

  • Bloom: Render Properties에서 켜기. Emission 재질의 빛 번짐. 이펙트 컨셉에 필수.
  • Screen Space Reflections: 반사. 금속/유리 표면에.
  • Ambient Occlusion: 좁은 틈새의 그림자. 디테일 추가.
  • Volumetric: 대기 안개/빛줄기. World > Volume > Principled Volume으로 안개 효과.

  2. 에디트 모드(Tab)에서 뼈 끝(Tail)을 선택, E로 새 뼈 돌출. 관절마다 하나씩.
  3. 뼈 이름 정리: Spine, Turret, ArmL 등. .L / .R 접미사를 쓰면 블렌더가 대칭 인식.
  4. 대칭 복제: 한쪽 뼈를 다 만든 뒤 Armature > Symmetrize하면 .L → .R 자동 생성.
  5. 오브젝트 모드에서 메시 먼저 선택 → Shift+클릭으로 Armature 추가 선택 → Ctrl+P > Armature Deform > With Automatic Weights.

가중치 페인팅 (Weight Paint)

Automatic Weights가 완벽하지 않을 때 직접 칠합니다. 메시 선택 → Ctrl+Tab > Weight Paint.

  • 빨간색 = 뼈에 100% 영향, 파란색 = 0%.
  • Properties > Object Data > Vertex Groups에 뼈 이름별 그룹이 있습니다.
  • 해당 뼈의 그룹을 선택하고, 영향 범위를 칠하세요.
  • 타워디펜스 팁: 회전하는 터렛은 Turret 뼈에 빨강, 고정 베이스는 파랑으로 깔끔 분리.
  • Normalize All: Weights 메뉴에서. 모든 뼈의 가중치 합이 1이 되도록 정리.

IK (Inverse Kinematics)

FK(Forward Kinematics)는 어깨→팔꿈치→손 순서로 직접 회전. IK는 손 위치만 정하면 나머지가 자동 계산.

  1. 포즈 모드에서 손 뼈 선택 → Bone Constraints > Inverse Kinematics 추가.
  2. Chain Length: 영향받을 뼈의 수 (손~어깨면 3).
  3. Target: IK 타겟 오브젝트 (보통 빈 Empty)를 지정하면 그 Empty를 움직여 자세 제어.
  4. Pole Target: 팔꿈치/무릎이 향하는 방향을 고정하는 보조 타겟.

키프레임 기초

  • Space 또는 Alt+A — 재생/정지.

Dope Sheet

모든 키프레임을 한눈에 보는 타임라인의 확장판. 키프레임을 선택해서 G로 시간 이동, S로 간격 조절(속도 변경), Shift+D로 복제.

NLA Editor (Non-Linear Animation)

액션(걷기, 공격, 대기)을 각각 별도 스트립으로 만들어 조합하는 시퀀서. 같은 캐릭터에 여러 애니메이션을 담아 FBX로 내보낼 때 핵심.

  2. 현재 액션의 이름을 지정 (예: Idle).
  3. Push Down 버튼으로 NLA 스트립으로 변환.
  4. New로 새 액션을 만들고 공격 애니메이션 작업 → 다시 Push Down.
  5. NLA Editor에서 스트립들을 블렌드/반복/트림 가능.

Shape Keys (모프 타겟)

메시 변형을 키프레임 없이 슬라이더로 제어. 얼굴 표정(눈 감기, 입 벌리기), 무기 변형 등에 활용.

  • Properties > Object Data > Shape Keys에서 Basis(기본 형태) 추가.
  • +로 새 키 추가 → 에디트 모드에서 원하는 형태로 변형.
  • Value 슬라이더(0~1)로 블렌딩. 이것도 I로 키프레임 가능.

Cloth (천)

깃발, 망토, 커튼. 메시에 Cloth 물리를 추가하고 재생하면 중력에 의해 자연스럽게 늘어집니다.

Rigid Body (강체)

물체끼리 충돌하고 튕기는 시뮬레이션. 파편 날리기, 도미노, 건물 붕괴 컨셉.

  • 움직일 물체: Active 타입. 영향받는 것.
  • 고정된 바닥/벽: Passive 타입. 충돌면 역할만.

  • Shape: Convex Hull이 기본. 복잡한 형태는 Mesh로 해야 정확하지만 느림.
  • 원하는 프레임에서 Object > Rigid Body > Apply Transformation으로 고정.

Soft Body

말랑한 젤리, 슬라임 느낌. 중력에 의해 처지고 충돌 시 찌그러짐. Goal로 원래 형태 복원력 조절.

Fluid (유체)

물, 연기, 불 시뮬레이션. Domain(시뮬 공간) 안에 Flow(방출원)를 배치.

19Geometry Nodes 고급

노드 기반 절차적 모델링. 나뭇잎 흩뿌리기, 돌담 자동 생성, 패턴 반복 — 수작업으로는 미친 짓인 걸 자동화.

Geometry Nodes 진입

  1. 오브젝트 선택 → Properties > Modifiers > Add Modifier > Geometry Nodes.
  2. New 클릭하면 Geometry Nodes 에디터가 열립니다.
  3. 기본: Group Input(원본 메시) → Group Output(결과 메시). 중간에 노드를 끼워넣습니다.

핵심 노드 패턴

Distribute Points on Faces

표면에 포인트를 랜덤 분포. 풀, 나무, 돌 흩뿌리기의 시작점. Density로 밀도 조절.

Instance on Points

각 포인트에 오브젝트를 복제 배치. 나뭇잎, 벽돌, 타일을 자동 인스턴싱.

Mesh to Curve / Curve to Mesh

메시를 커브로, 커브를 메시로 변환. 파이프, 레일 자동 생성.

Random Value

크기, 회전, 색상에 랜덤 변화를 줘서 자연스러운 변형.

Set Material / Set Position

각 인스턴스에 다른 재질/위치를 프로그래밍적으로 지정.

Noise Texture

절차적 노이즈로 지형 높낮이, 불규칙 패턴 생성.

Boolean Mesh

노드 안에서 Boolean 연산. 비파괴적으로 구멍/합치기를 체인.

Realize Instances

인스턴스를 실제 메시로 변환. 내보내기 전 필수.

실전 예시: 숲 자동 생성

  1. 지형 Plane에 Geometry Nodes 추가.
  2. Distribute Points on Faces (Density=0.5, Seed=1).
  3. Instance on Points → Object에 나무 모델 연결.
  4. Random Value로 Scale(0.8~1.3)과 Rotation Z(0~360) 변화.
  5. Density를 Weight Paint로 제어하면 "숲이 있는 영역"과 "빈 영역"을 자유롭게.

AccelerationV3. 방향 가속(0,2,0): 위로 상승

Beam — 두 점 사이 광선

  • 두 Attachment 사이에 텍스처를 이은 광선. 수직 빔/레이저에 적합.
  • Width0/Width1: 양 끝 두께.
  • LightEmission=1: 글로우.
  • CurveSize0/1: S자 커브. TextureSpeed로 흐르는 느낌.
  • Segments: 높일수록 곡선이 부드러움 (8~16 권장).

Trail — 이동 잔상

  • 두 Attachment 사이 폭을 가진 궤적. 무기 궤적, 이동 잔상.
  • Lifetime: 잔상 지속 시간. 0.3~1초.
  • MinLength: 이 거리 이상 움직여야 그려짐. 떨림 방지.
  • Color/Transparency를 시간에 따라 변화시키면 자연스러운 페이드.

솔스RNG 스타일 오라 구현 6단계

  1. 몸통 글로우: HumanoidRootPart에 ParticleEmitter. Rate=50, Speed=1~2, Size=3→5, Transparency=0.4→1, LightEmission=1, SpreadAngle=(180,180).
  2. 상승 파티클: 별도 ParticleEmitter. Speed=3~6, EmissionDirection=Top, Size=0.3→0, Rate=30.
  3. 수직 빔: HRP에서 20 studs 위 Part까지 Beam. Width=2~4, LightEmission=1, TextureSpeed=1.
  4. 회전 링: 얇은 원형 MeshPart를 허리 높이에 배치. TweenService로 Y축 연속 회전.
  5. 트레일: 양 손목 Attachment 쌍에 Trail. Lifetime=0.5.
  6. 펄스 애니메이션: 모든 이펙트의 Size/Rate를 TweenService로 주기적으로 변화 → "숨 쉬는" 느낌.

Luau 코드 — 오라 펄스

-- 오라 파티클 Rate를 주기적으로 펄싱
local TweenService = game:GetService("TweenService")
local emitter = script.Parent:WaitForChild("AuraEmitter")

local info = TweenInfo.new(
    1.2,                       -- 주기(초)
    Enum.EasingStyle.Sine,     -- 부드러운 사인파
    Enum.EasingDirection.InOut,
    -1,                        -- 무한 반복
    true                       -- 왕복
)
TweenService:Create(emitter, info, {Rate = 60}):Play()

-- 회전 링 연속 회전
local ring = script.Parent:WaitForChild("AuraRing")
local RunService = game:GetService("RunService")

local speed = math.rad(120)  -- 초당 120도
RunService.Heartbeat:Connect(function(dt)
    ring.CFrame = ring.CFrame * CFrame.Angles(0, speed * dt, 0)
end)

-- 티어별 오라 모듈 (재사용 구조)
local AuraModule = {}

function AuraModule.Apply(character, tierData)
    local hrp = character:WaitForChild("HumanoidRootPart")

    -- 기본 파티클
    local emitter = Instance.new("ParticleEmitter")
    emitter.Rate = tierData.rate or 40
    emitter.Lifetime = NumberRange.new(0.8, 1.5)
    emitter.Speed = NumberRange.new(1, 3)
    emitter.SpreadAngle = Vector2.new(180, 180)
    emitter.LightEmission = 1
    emitter.LightInfluence = 0
    emitter.Color = ColorSequence.new(tierData.color)
    emitter.Size = NumberSequence.new({
        NumberSequenceKeypoint.new(0, 0),
        NumberSequenceKeypoint.new(0.5, tierData.size or 2),
        NumberSequenceKeypoint.new(1, 0),
    })
    emitter.Transparency = NumberSequence.new({
        NumberSequenceKeypoint.new(0, 0.3),
        NumberSequenceKeypoint.new(1, 1),
    })
    emitter.Parent = hrp
end

return AuraModule

25최적화와 성능 고급

타워디펜스는 동시에 많은 오브젝트가 존재. 최적화가 특히 중요.

블렌더 단계 최적화

  • Decimate 모디파이어: Ratio를 줄여 Tris 수 감소. Collapse, Un-Subdivide, Planar 3가지 모드.
  • 불필요한 엣지 루프 제거: Ctrl+X(Dissolve).
  • 내부 면 삭제: Boolean 후 안쪽 보이지 않는 면. Select Interior Faces.
  • 텍스처 크기: 256x256이면 대부분 충분. 1024 이상은 낭비.
  • 중복 재질 합치기: 같은 색인데 슬롯이 여러 개면 통합.
  • LOD 수동 제작: 같은 모델의 High/Mid/Low 버전을 만들어 두면 거리별 교체 가능.

Studio 단계 최적화

  • CollisionFidelity: 충돌 정밀도. 타워처럼 정밀 불필요한 것은 Box. 성능 절약.
  • RenderFidelity: 거리별 디테일 자동 조절. Automatic 권장.
  • 인스턴싱: 같은 MeshId를 공유하면 엔진이 자동 최적화. Clone으로 복제.
  • 텍스처 아틀라스: 여러 오브젝트가 하나의 텍스처를 공유하면 드로우콜 감소.
  • 이펙트 개수 제한: ParticleEmitter의 Rate/MaxParticles 제어. 화면에 파티클 1000개 넘으면 프레임 드랍.
  • 원거리 이펙트 끄기: 카메라에서 먼 타워의 이펙트는 Rate를 0으로 낮추기.
  • StreamingEnabled: 큰 맵에서 가까운 것만 로딩. TD 맵 규모에 따라 판단.

프로파일링

  • MicroProfiler (Ctrl+F6): 프레임 드랍이 렌더/스크립트/물리 중 어디서 발생하는지 파악.
  • Stats Bar: Settings > Rendering > Performance Stats. FPS, 드로우콜, 인스턴스 수 실시간 표시.
  • Developer Console (F9): 스크립트 에러, 메모리 사용량 확인.
TIP

최적화 우선순위: (1) 폴리곤 수 줄이기 (2) 텍스처 크기 줄이기 (3) 이펙트 파티클 수 줄이기 (4) 스크립트 최적화. 이 순서로 효과가 큽니다.

26컴포지터와 후처리 고급

렌더 결과 이미지를 노드로 후처리. 포트폴리오, 썸네일, 이펙트 컨셉 이미지의 최종 퀄리티업.

컴포지터 열기

에디터 타입을 Compositor로 변경하거나 상단 Compositing 워크스페이스. Use Nodes 체크. 기본: Render Layers → Composite.

핵심 노드

Glare

밝은 부분에서 빛 줄기(Streaks) 또는 안개(Fog Glow) 효과. 이펙트 이미지의 마법 느낌.

Color Balance / Hue Saturation

전체 색감 보정. Lift(어두운 톤)/Gamma(중간)/Gain(밝은 톤) 조절.

Lens Distortion

카메라 렌즈 왜곡. Dispersion으로 색수차(RGB 번짐) = 시네마틱 느낌.

Vignette (Ellipse Mask + Mix)

화면 가장자리를 어둡게. 시선 집중 효과.

Blur

Depth of Field 외의 추가 블러. Bokeh 효과.

Denoise

Cycles 렌더의 노이즈 제거. 적은 샘플로도 깨끗한 결과.

기본 컴포지팅 체인

  1. Render Layers → Denoise (Cycles일 때).
  2. → Glare (Streaks, Threshold=0.8, Streaks=4).
  3. → Color Balance (블루 쉬프트 또는 따뜻한 톤).
  4. → Lens Distortion (Dispersion=0.02).
  5. → Mix (비네트용 Ellipse Mask와 합성).
  6. → Composite + Viewer (미리보기).
TIP

Backdrop: 컴포지터 헤더에서 Backdrop을 켜면 노드 에디터 배경에 결과 이미지가 실시간으로 표시됩니다. Viewer 노드를 연결하면 중간 결과도 확인.

28프로시저럴 텍스처링 고급

이미지 파일 없이 노드만으로 무한 반복 없는 텍스처. TD 에셋의 금속, 에너지 글로우, 마법 인쇄를 100% 프로시저럴로.

왜 프로시저럴인가

  • 해상도 독립: 얼마나 확대해도 픽셀 깨짐 없음.
  • 반복 없음: Noise 기반 패턴은 타일링 없이 자연스럽.
  • 파라미터 제어: 색상, 거칠기, 패턴 형태를 수치 하나로 바꿈.
  • UV 불필요(일부): Object 또는 Generated 좌표계로 UV 없이도 작동.

텍스처 좌표계

Texture Coordinate 노드 출력:

  • Generated: 오브젝트 바운딩박스 기준 0~1. UV 없이 텍스처 적용의 기본.
  • UV: UV 맵 좌표. Image Texture와 연동.
  • Object: 특정 오브젝트 기준 좌표. Empty 오브젝트를 이동하면 텍스처가 따라옴.
  • Normal: 표면 법선 방향. 반사 효과용.
  • Camera: 카메라 시점 기준. 카메라 향해 항상 같은 패턴.
  • Window: 화면 픽셀 좌표. 후처리/스크린스페이스 이펙트.

Mapping 노드: 좌표 이동(Location), 회전(Rotation), 크기(Scale). Texture Coordinate → Mapping → Texture Node.

레시피: 에너지 오라 재질 (프로시저럴)

  1. Layer Weight 노드 → Fresnel 출력. 외곽 엣지에 강한 값.
  2. Noise Texture (Scale 8, Detail 12, Distortion 2) → Fac 출력.
  3. ColorRamp: 검정~청록~흰색. 샤프하게 조절.
  4. Math (Multiply): Fresnel * ColorRamp 출력.
  5. Emission: Color에 원하는 색, Strength에 4~8.
  6. 전체를 Mix Shader: Transparent BSDF + Emission. Fac = 위 결과.

레시피: 금속 스크래치 (프로시저럴)

  1. Noise Texture (Scale 200, Detail 8) → Bump 노드로 미세 범프.
  2. Voronoi (Distance to Edge, Scale 50) → ColorRamp 조정 → 스크래치 선.
  3. Voronoi Fac를 Roughness에 연결: 스크래치 부분만 거칠어짐.
  4. Principled BSDF: Metallic=1, Base Color=0.7 gray, Roughness=위 결과.
RBX

프로시저럴로 만든 패턴을 게임에 쓰려면: Shader Editor에서 이미지 텍스처로 Bake(13챕터 베이킹 참조). Roughness 맵, Normal 맵, Color 맵을 각각 bake해서 FBX에 포함.

import bpy
import math

obj = bpy.context.active_object
count = 12
radius = 5.0

for i in range(count):
    angle = (2 * math.pi / count) * i
    x = radius * math.cos(angle)
    y = radius * math.sin(angle)
    new_obj = obj.copy()
    new_obj.data = obj.data.copy()
    new_obj.location = (x, y, 0)

EEVEE 품질 향상 세팅

Properties > Render > EEVEE:

  • Shadows: Cube Size 4096, Cascade Size 4096 — 그림자 품질 최대화.
  • Ambient Occlusion: ON, Distance 0.2, Factor 0.5 — 메시 접힌 부분 음영.
  • Bloom: ON, Threshold 0.8, Intensity 0.05 — Emission 빛 번짐. 이펙트 필수.
  • Screen Space Reflections: ON, Max Roughness 0.5 — 반사 효과.
  • Motion Blur: ON (애니메이션 렌더 시 속도감).
  • Depth of Field: 카메라에서 F-Stop 지정 후 활성.
  • Volumetrics: ON, Tile Size 2px — 안개, 빛 기둥 효과.

Irradiance Volume (간접광)

Shift+A > Light Probe > Irradiance Volume. 씬을 감싸도록 크기 조절. Properties > Render > Indirect Lighting > Bake Indirect Lighting. 이후 오브젝트가 주변 색상 반사(Bounce Light)를 받음.

Reflection Cubemap

Shift+A > Light Probe > Reflection Cubemap. 유리, 금속 표면의 반사 정확도 향상. 배치 후 Bake Cubemap Only.

Volumetric Lighting (빛 기둥)

  1. Render > Volumetrics 활성화.
  2. World 탭 > Volumetrics > Volume Scatter 노드 추가. Density 0.02~0.05.
  3. Spot 또는 Sun 조명에서 Shadow 활성 필수.
  4. 조명을 통과하는 빛 기둥(God Ray) 효과 완성.

EEVEE Next (블렌더 4.2+)

EEVEE Next는 전면 재설계 버전으로, 레이트레이싱 기반 반사/굴절, 더 정확한 GI, 향상된 그림자를 실시간에 가깝게 제공. 구버전 EEVEE보다 Cycles와의 격차가 크게 줄었음. Light Linking, Raytracing Reflections, Subsurface 개선이 핵심 변경사항.

ADV

이펙트 포트폴리오를 만들 때: EEVEE + Bloom + Volumetrics 조합으로 솔스RNG 스타일의 빛나는 오라를 30fps 이상 실시간 프리뷰하고, 최종 결과물은 Cycles로 바꿔 고품질 스틸 이미지를 뽑는 워크플로우가 효율적.

32고급 이펙트 레시피 고급

레시피 1: 회오리 오라

  1. 캐릭터 메시 복제 → Duplicate → Scale 1.05~1.1.
  2. Simple Deform 모디파이어 (Twist, Angle 45~90도). 비틀린 오라 형태.
  3. Solidify 모디파이어 두께 0.01~0.03.
  4. Wireframe 모디파이어 (선택): 와이어프레임 오라.
  5. 재질: Emission + Fresnel로 외곽만 발광.
  6. 애니메이션: Simple Deform의 Angle에 키프레임 -180도 → 180도 Loop.

레시피 2: 에너지 구체 (Orb)

  1. UV Sphere 추가 (Segments 64, Rings 32).
  2. 재질: Noise Texture (Scale 3, Distortion 2) → ColorRamp (어두운 파랑~밝은 시안~흰색) → Emission Strength 8~15.
  3. Displacement 모디파이어: Texture(동일 Noise) → Strength 0.05~0.1. 표면이 살아서 흔들리는 느낌.
  4. Geometry Nodes: Set Position → Noise 기반 오프셋으로 수동 Displacement 대체 가능.
  5. Animate: Noise Texture의 W 파라미터에 키프레임. 텍스처가 시간에 따라 변화.

레시피 3: 번개/전기 이펙트

  1. Bezier Curve 추가. 시작점과 끝점만 있는 간단한 선.
  2. 에디트 모드에서 W > Subdivide 여러 번 → 컨트롤 포인트 많이 생성.
  3. 모든 포인트 선택 후 Random Transform (Alt+S 또는 스크립트)으로 랜덤하게 흩뜨림.
  4. Curve Properties > Geometry > Bevel Depth 0.01~0.03. 두께 생김.
  5. Emission 재질: 밝은 파랑/흰색, Strength 15~30.
  6. Bloom 활성 (Eevee): 전기 번쩍이는 느낌.
  7. Multiple 커브: 위 과정 3~5개 반복, 각각 약간 다른 경로로 복잡한 번개.

레시피 4: 마법진 (Magic Circle)

  1. Circle 커브 추가. Bevel Depth로 링 형태.
  2. Python 스크립트 또는 Array 모디파이어로 동심원 여러 개 생성.
  3. Geometry Nodes: Curve Circle → Rotate Instances (키프레임 속도 달리 줘서 반대 방향 회전).
  4. Image Texture: 룬 문자 PNG(흰 배경/투명) → 링 면에 UV 배치.
  5. Emission: Texture * 오라 색상 → Strength 5~10.
  6. 지면 투영: Z 방향으로 납작하게 Scale. 바닥에 마법진 투영 느낌이면 plane에 텍스처.

레시피 5: 분열/파편 이펙트 (Shatter)

  1. Cell Fracture 애드온 활성화 (Built-in, 설정 필요).
  2. 부수고 싶은 메시 선택 → F3 검색 > Cell Fracture 실행.
  3. Source Limit 50~200 (파편 개수). 결과: 여러 분리된 메시.
  4. 모든 파편 선택 → Properties > Physics > Rigid Body (Active).
  5. 원본 위치에서 일정 프레임 후 Explode 시작: 파편에 초기 속도(Force Field 또는 키프레임).
  6. 렌더 후 VSE에서 파편 효과 + Glare 컴포지터 조합.

레시피 6: 연기/안개 볼륨

  1. Cube 추가. 연기가 채워질 영역 크기로 조절.
  2. Properties > Object Properties > Visibility > Shadow: 그림자 없음으로.
  3. 재질 추가 → Volume Scatter + Volume Absorption 노드.
  4. Volume Scatter: Color 연기 색상, Density 0.1~0.5, Anisotropy 0.3.
  5. Noise Texture를 Density에 연결: 불균일한 밀도 → 자연스러운 연기.
  6. 애니메이션: Noise의 W 값 키프레임으로 연기가 흐르는 느낌.

레시피 7: 홀로그램 이펙트

  1. 메시에 재질 추가.
  2. Emission: 청록(#00FFEE) 색상, Strength 3.
  3. Wave Texture (Rings, Scale 50) → ColorRamp(밝음/어둠 반복) → Alpha에 연결.
  4. 재질 설정 > Blend Mode: Alpha Blend, Shadow Mode: None.
  5. Wireframe Modifier: Relative Thickness, 두께 0.002 → 선 격자 오버레이.
  6. 두 재질(기본 홀로그램 + Wireframe 오버레이)을 Mix Shader로 합성.
