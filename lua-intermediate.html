Luau 중급 — 이벤트와 게임 기능

테이블, 이벤트, 클라이언트/서버, GUI, Tween. 문법을 넘어 진짜 게임 기능을 만드는 단계.

10챕터 이벤트 서버/클라이언트 GUI

11테이블 — 만능 보관함 중급

Luau의 유일하고 가장 중요한 자료구조. 목록도, 사전도, 객체도 전부 테이블입니다.

배열 — 순서 있는 목록

local fruits = {"사과", "바나나", "포도"}
print(fruits[1])        -- 사과 (1번부터 시작!)
print(#fruits)          -- 3 (#은 개수)
table.insert(fruits, "수박")   -- 끝에 추가
table.remove(fruits, 2)        -- 2번째 제거

딕셔너리 — 이름표 달린 보관함

local player = {
    name = "문앙",
    level = 5,
    isVip = true,
}
print(player.name)      -- 문앙
player.level = player.level + 1

순회 — 전부 꺼내보기

for i, fruit in ipairs(fruits) do   -- 배열은 ipairs
    print(i, fruit)
end
for key, value in pairs(player) do   -- 딕셔너리는 pairs
    print(key, value)
end
TIP

Luau의 인덱스는 1부터입니다. 다른 언어의 0부터에 익숙하다면 가장 자주 틀리는 부분.

12인스턴스 다루기 중급

게임 속 모든 것을 코드로 찾고, 만들고, 지우는 법.

찾기

local part = workspace.MyPart              -- 직접 접근 (없으면 에러)
local p2 = workspace:FindFirstChild("MyPart")  -- 없으면 nil (안전)
local p3 = workspace:WaitForChild("MyPart")    -- 생길 때까지 기다림
local kids = workspace.Map:GetChildren()       -- 자식 전부 (테이블)
local all = workspace.Map:GetDescendants()     -- 자손 전부

만들기와 지우기

local part = Instance.new("Part")   -- 새 파트 생성
part.Size = Vector3.new(4, 1, 4)
part.Anchored = true
part.Parent = workspace              -- Parent를 정해야 세상에 나타남 (마지막에!)

part:Destroy()                       -- 삭제

local copy = workspace.Tree:Clone()  -- 복제
copy.Parent = workspace
  • Parent는 마지막에: 속성을 다 정한 뒤 Parent를 지정하는 것이 성능상 올바른 순서입니다.
  • :IsA("BasePart") — 종류 검사. GetDescendants 순회와 조합해 "모든 파트만 골라서" 작업.
-- 맵의 모든 파트를 반투명하게
for _, obj in ipairs(workspace.Map:GetDescendants()) do
    if obj:IsA("BasePart") then
        obj.Transparency = 0.5
    end
end

13이벤트 — 일이 생기면 알려줘 중급

게임은 이벤트로 굴러갑니다. "닿으면", "들어오면", "클릭하면" — 전부 이벤트.

-- 패턴: 무언가.이벤트이름:Connect(실행할 함수)
workspace.Lava.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChildOfClass("Humanoid")
    if humanoid then
        humanoid.Health = 0      -- 용암 밟으면 사망
    end
end)

자주 쓰는 이벤트

이벤트언제 발생대표 용도
part.Touched파트에 무언가 닿을 때용암, 체크포인트, 아이템 획득
Players.PlayerAdded플레이어 입장데이터 로드, 환영 메시지
Players.PlayerRemoving플레이어 퇴장데이터 저장
humanoid.Died캐릭터 사망리스폰 처리, 킬 카운트
ClickDetector.MouseClick파트 클릭버튼, 상호작용
ProximityPrompt.TriggeredE키 상호작용문, 상점, 줍기

연결 해제와 디바운스

local conn = part.Touched:Connect(handler)
conn:Disconnect()                -- 더 이상 안 들음

-- 디바운스: Touched는 1초에 수십 번 발생 — 한 번만 처리하게 잠금
local busy = false
part.Touched:Connect(function(hit)
    if busy then return end
    busy = true
    -- 처리...
    task.wait(1)
    busy = false
end)

14플레이어와 캐릭터 중급

Player(계정 정보)와 Character(월드 속 몸)는 다른 것입니다. 이 구분이 모든 캐릭터 코드의 출발점.

local Players = game:GetService("Players")

Players.PlayerAdded:Connect(function(player)
    print(player.Name .. " 입장!")

    player.CharacterAdded:Connect(function(character)
        local humanoid = character:WaitForChild("Humanoid")
        humanoid.WalkSpeed = 24      -- 이동속도 (기본 16)
        humanoid.JumpPower = 70      -- 점프력 (기본 50)
        humanoid.MaxHealth = 150
        humanoid.Health = 150
    end)
end)

구조 이해

  • Player — game.Players 안. 이름, 팀, 데이터. 게임 내내 유지.
  • Character — workspace 안의 모델. 죽으면 사라지고 리스폰 때 새로 생김 — 그래서 CharacterAdded로 매번 다시 설정.
  • 인스턴스. 체력·이동속도·점프·사망 담당">Humanoid — 캐릭터 안의 "생명" 인스턴스. 체력, 속도, 점프, 사망 이벤트 담당.
  • HumanoidRootPart — 캐릭터의 물리 중심. 순간이동은 이것의 CFrame을 바꿉니다.
-- 순간이동
character:PivotTo(CFrame.new(0, 50, 0))

15클라이언트와 서버 중급

Roblox 게임은 한 대가 아니라 여러 대의 컴퓨터에서 동시에 돌아갑니다. 이걸 이해해야 중급입니다.

두 개의 세계

서버클라이언트 (각 플레이어 기기)
스크립트 종류ScriptLocalScript
두는 곳ServerScriptServiceStarterPlayerScripts, StarterGui
담당게임 규칙, 점수, 데이터 — 진실화면, 입력, 카메라, 효과 — 보이는 것
신뢰신뢰 가능해커가 조작 가능 — 신뢰 금지
  • 서버에서 바꾼 것은 모두에게 복제(replicate)되어 보입니다. 클라이언트에서 바꾼 것은 자기 화면에만 보입니다.
  • 점수, 돈, 체력 같은 "진짜 값"은 반드시 서버에서만 변경 — 클라이언트 값은 해킹될 수 있습니다.
  • 둘 사이 대화는 RemoteEvent로 — 다음 챕터.
!!

LocalScript를 ServerScriptService에 넣으면 실행되지 않고, Script를 StarterGui에 넣으면 의도와 다르게 동작합니다. "어디에 두는가"가 "어디서 실행되는가"입니다.

16RemoteEvent — 서버와 대화 중급

클라이언트가 서버에게 "이거 해줘"라고 부탁하는 공식 통로.

  1. ReplicatedStorage에 RemoteEvent 추가, 이름 BuyItem.
  2. 클라이언트(LocalScript)에서 발사:
-- LocalScript (버튼 클릭 시)
local RS = game:GetService("ReplicatedStorage")
RS.BuyItem:FireServer("Sword")   -- 서버에게 "Sword 살게요"
-- Script (서버에서 수신)
local RS = game:GetService("ReplicatedStorage")
RS.BuyItem.OnServerEvent:Connect(function(player, itemName)
    -- 서버가 검증: 돈이 충분한가? 그런 아이템이 있는가?
    if itemName == "Sword" and playerCoins[player] >= 100 then
        playerCoins[player] -= 100
        giveItem(player, itemName)
    end
end)
  • 첫 매개변수 player는 자동으로 들어옵니다 — 누가 보냈는지.
  • 서버는 클라이언트가 보낸 값을 절대 믿지 않습니다. "돈 있어요"라고 보내와도 서버가 직접 확인 — 이것이 보안의 기본.
  • 서버→클라이언트는 FireClient(player, ...) / FireAllClients(...), 수신은 OnClientEvent.
  • 답장이 필요하면 RemoteFunction(InvokeServer) — 단, 서버가 클라이언트를 Invoke하는 것은 위험하니 금지.

17GUI 만들기 중급

화면에 붙는 점수판, 버튼, 상점창. GUI는 클라이언트(LocalScript)의 영역입니다.

구조

  1. StarterGui에 ScreenGui 추가 — GUI의 캔버스.
  2. 그 안에 Frame(패널), TextLabel(글자), TextButton(버튼), ImageLabel(그림)을 배치.
  3. 위치/크기는 UDim2: UDim2.new(0.5, 0, 0.1, 0) = (가로 50% + 0px, 세로 10% + 0px). 비율을 쓰면 모든 화면 크기에 대응.
  4. AnchorPoint (0.5, 0.5)로 중심 기준 배치 — 중앙 정렬의 핵심.

버튼 동작

-- LocalScript (ScreenGui 안)
local button = script.Parent.ShopButton
local frame = script.Parent.ShopFrame

button.MouseButton1Click:Connect(function()
    frame.Visible = not frame.Visible   -- 켜고 끄기 토글
end)

점수 표시 실시간 갱신

-- leaderstats 값을 화면 라벨에 연동
local player = game.Players.LocalPlayer
local coins = player:WaitForChild("leaderstats"):WaitForChild("Coins")
local label = script.Parent.CoinLabel

label.Text = "코인: " .. coins.Value
coins.Changed:Connect(function(v)
    label.Text = "코인: " .. v
end)
TIP

UICorner(둥근 모서리), UIStroke(테두리), UIListLayout(자동 정렬)을 Frame에 추가하면 코드 없이 디자인 품질이 올라갑니다.

18TweenService — 부드러운 움직임 중급

뚝뚝 끊기지 않고 스르륵 — 문, GUI, 이펙트의 모든 부드러운 변화.

local TweenService = game:GetService("TweenService")
local door = workspace.Door

local info = TweenInfo.new(
    1.5,                              -- 시간(초)
    Enum.EasingStyle.Quad,            -- 가감속 곡선
    Enum.EasingDirection.Out          -- 끝에서 감속
)
local goal = {
    Position = door.Position + Vector3.new(0, 8, 0),  -- 목표값
    Transparency = 0.5,
}
local tween = TweenService:Create(door, info, goal)
tween:Play()
tween.Completed:Wait()    -- 끝날 때까지 대기 (선택)

EasingStyle 고르기

스타일느낌용도
Linear일정 속도회전, 컨베이어
Quad / Quart자연스러운 감속문, 카메라, 대부분
Back살짝 넘었다 돌아옴GUI 팝업
Bounce통통 튐떨어지는 보상
Elastic용수철강조 연출

GUI에도 똑같이 적용됩니다 — Frame의 Position/Size를 Tween하면 슬라이드 인 메뉴가 됩니다. 블렌더에서 만든 검기 메시를 움직이는 것도 바로 이 도구입니다.

19자주 쓰는 서비스 모음 중급

Roblox가 미리 만들어 둔 기능 창고들. game:GetService()로 꺼내 씁니다.

서비스담당대표 사용

ReplicatedStorage서버·클라 공유 보관함RemoteEvent, 공용 모듈

-- 낮밤 순환 예시 (RunService + Lighting)
local RunService = game:GetService("RunService")
local Lighting = game:GetService("Lighting")
RunService.Heartbeat:Connect(function(dt)
    Lighting.ClockTime = (Lighting.ClockTime + dt * 0.05) % 24
end)

키 입력 (LocalScript)

local UIS = game:GetService("UserInputService")
UIS.InputBegan:Connect(function(input, processed)
    if processed then return end   -- 채팅 입력 중이면 무시
    if input.KeyCode == Enum.KeyCode.E then
        print("E 눌림!")
    end
end)

20종합 프로젝트 — 코인 줍기 게임 중급

중급 전부를 조합: 서버가 코인을 관리하고, 닿으면 획득, GUI에 표시.

-- Script (ServerScriptService): leaderstats + 코인 시스템
local Players = game:GetService("Players")

Players.PlayerAdded:Connect(function(player)
    local stats = Instance.new("Folder")
    stats.Name = "leaderstats"      -- 이 이름이면 자동으로 순위표에 표시
    local coins = Instance.new("IntValue")
    coins.Name = "Coins"
    coins.Parent = stats
    stats.Parent = player
end)

-- 코인 파트 동작
for _, coin in ipairs(workspace.Coins:GetChildren()) do
    local taken = false
    coin.Touched:Connect(function(hit)
        if taken then return end
        local player = Players:GetPlayerFromCharacter(hit.Parent)
        if player then
            taken = true
            player.leaderstats.Coins.Value += 1
            coin:Destroy()
        end
    end)
end
  1. Workspace에 Coins 폴더 > 노란 Neon 파트 여러 개 (Anchored, CanCollide false).
  2. 위 스크립트를 ServerScriptService에.
  3. 화면 GUI는 챕터 17의 코인 라벨 코드를 그대로 — leaderstats.Coins에 연동되어 자동 갱신.
  4. 도전 과제: 코인이 빙글빙글 돌게 (Heartbeat + CFrame.Angles), 먹을 때 소리 추가.
TIP

여기까지 오면 작은 게임을 만들 수 있습니다. 고급에서는 이 코드를 "제대로 된 구조"로 다시 짜는 법 — 모듈, 데이터 저장, CFrame 수학을 배웁니다.
