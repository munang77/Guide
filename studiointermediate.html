11Luau 문법 정리 중급

Roblox의 스크립트 언어 Luau. 핵심 문법을 한 번에 정리합니다.

변수와 타입

local text = "문자열"       -- string
local count = 42            -- number
local isOpen = true         -- boolean
local nothing = nil         -- 값 없음

-- 문자열 연결은 .. 사용
print("개수: " .. count)

조건문

if count > 50 then
    print("많음")
elseif count > 10 then
    print("보통")
else
    print("적음")
end

-- 비교 연산자: ==, ~=(다름), <, >, <=, >=
-- 논리 연산자: and, or, not

반복문

-- 숫자 반복: 1부터 10까지
for i = 1, 10 do
    print(i)
end

-- 조건 반복
local hp = 100
while hp > 0 do
    hp = hp - 10
    task.wait(1)  -- 1초 대기. 없으면 게임이 멈춤!
end

함수

local function add(a, b)
    return a + b
end

local result = add(3, 7)  -- 10
!!

while true do 같은 무한 루프 안에는 반드시 task.wait()를 넣으세요. 없으면 Studio가 멈춥니다.

12테이블 완전 이해 중급

Luau의 유일한 자료구조. 배열도, 딕셔너리도 전부 테이블입니다.

배열 형태

local fruits = {"사과", "바나나", "포도"}

print(fruits[1])        -- 사과 (1부터 시작!)
print(#fruits)          -- 3 (개수)

table.insert(fruits, "수박")   -- 끝에 추가
table.remove(fruits, 1)        -- 1번 삭제

-- 전체 순회
for i, fruit in ipairs(fruits) do
    print(i, fruit)
end

딕셔너리 형태

local player = {
    name = "Kim",
    level = 12,
    isVip = true,
}

print(player.name)       -- Kim
player.level = 13        -- 수정

for key, value in pairs(player) do
    print(key, value)
end

중첩 테이블 (실전 구조)

local shop = {
    {name = "검",   price = 100},
    {name = "방패", price = 150},
    {name = "물약", price = 30},
}

for _, item in ipairs(shop) do
    print(item.name .. ": " .. item.price .. "원")
end
TIP

배열 순회는 ipairs, 딕셔너리 순회는 pairs. ipairs는 순서 보장, pairs는 순서 비보장입니다.

13인스턴스 다루기 중급

월드의 모든 오브젝트(파트, 모델, 스크립트...)는 인스턴스입니다. 코드로 만들고, 찾고, 지우는 법.

찾기

-- 직접 경로 (없으면 에러)
local part = workspace.MyPart

-- 안전하게 찾기 (없으면 nil)
local part = workspace:FindFirstChild("MyPart")
if part then
    print("찾음")
end

-- 생길 때까지 기다리기 (로딩 순서 문제 해결)
local part = workspace:WaitForChild("MyPart")

-- 자식 전부 가져오기
for _, child in ipairs(workspace:GetChildren()) do
    print(child.Name)
end

만들기와 복제

-- 새 파트 생성
local part = Instance.new("Part")
part.Size = Vector3.new(4, 1, 4)
part.Position = Vector3.new(0, 10, 0)
part.Anchored = true
part.Parent = workspace  -- Parent는 마지막에!

-- 기존 모델 복제
local template = game.ServerStorage.TreeModel
local copy = template:Clone()
copy.Parent = workspace

-- 삭제
part:Destroy()
TIP

속성을 전부 설정한 뒤 마지막에 Parent를 지정하는 것이 성능상 좋습니다. Parent를 먼저 지정하면 속성을 바꿀 때마다 월드가 갱신됩니다.

주요 클래스 확인

if obj:IsA("BasePart") then   -- 파트 종류 전부
if obj:IsA("Model") then
if obj:IsA("Humanoid") then

14이벤트 중급

"무언가 일어났을 때 실행"이 이벤트입니다. 게임 상호작용의 전부가 이벤트로 만들어집니다.

Touched — 닿았을 때

local part = script.Parent
local debounce = false  -- 중복 실행 방지

part.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChildOfClass("Humanoid")
    if humanoid and not debounce then
        debounce = true
        humanoid.Health = humanoid.Health - 10  -- 데미지
        task.wait(1)
        debounce = false
    end
end)
!!

Touched는 1초에 수십 번 발생할 수 있습니다. 위처럼 debounce(디바운스) 변수로 중복 실행을 막는 패턴은 필수입니다.

ClickDetector — 클릭했을 때

-- 파트 안에 ClickDetector를 추가하고 이 스크립트도 추가
local click = script.Parent.ClickDetector

click.MouseClick:Connect(function(player)
    print(player.Name .. "이(가) 클릭")
end)

플레이어 입장/퇴장

local Players = game:GetService("Players")

Players.PlayerAdded:Connect(function(player)
    print(player.Name .. " 입장")
end)

Players.PlayerRemoving:Connect(function(player)
    print(player.Name .. " 퇴장")
end)

이벤트 연결 해제

local connection = part.Touched:Connect(handler)
connection:Disconnect()  -- 더 이상 필요 없을 때

15캐릭터와 Humanoid 중급

플레이어 캐릭터를 코드로 제어합니다. 체력, 속도, 점프, 사망 처리.

캐릭터 구조

  • Player: Players 서비스 안의 플레이어 정보 (이름, ID 등).
  • Character: Workspace에 있는 실제 몸 (Model).
  • Humanoid: Character 안에 있는 생명체 제어 인스턴스. 체력, 이동속도 담당.
  • HumanoidRootPart: 캐릭터의 중심 파트. 위치 이동(텔레포트)에 사용.

캐릭터 가져오기와 속성 변경

local Players = game:GetService("Players")

Players.PlayerAdded:Connect(function(player)
    player.CharacterAdded:Connect(function(character)
        local humanoid = character:WaitForChild("Humanoid")

        humanoid.WalkSpeed = 24     -- 기본 16
        humanoid.JumpPower = 75     -- 기본 50
        humanoid.MaxHealth = 200
        humanoid.Health = 200
    end)
end)

사망 감지

humanoid.Died:Connect(function()
    print("사망")
    -- 점수 차감, 리스폰 메시지 등
end)

텔레포트

local root = character:WaitForChild("HumanoidRootPart")
root.CFrame = CFrame.new(0, 50, 0)  -- 좌표로 순간이동

16GUI 만들기 중급

화면에 표시되는 버튼, 점수판, 메뉴. StarterGui에 만듭니다.

GUI 기본 구조

  1. StarterGui > + > ScreenGui 추가. 화면 전체를 덮는 보이지 않는 캔버스.
  2. ScreenGui 안에 Frame(사각형 패널), TextLabel(글자), TextButton(버튼), ImageLabel(이미지)을 추가.

크기와 위치 — UDim2

GUI의 Size와 Position은 UDim2로 표현합니다: {X스케일, X픽셀}, {Y스케일, Y픽셀}. 스케일은 화면 비율(0~1), 픽셀은 고정값입니다.

-- 화면 가로 30%, 세로 10% 크기
frame.Size = UDim2.new(0.3, 0, 0.1, 0)

-- 화면 정중앙에 배치
frame.AnchorPoint = Vector2.new(0.5, 0.5)
frame.Position = UDim2.new(0.5, 0, 0.5, 0)
TIP

모바일/PC 모두 대응하려면 픽셀보다 스케일을 위주로 쓰세요. 픽셀 고정값은 작은 화면에서 깨집니다. 모서리를 둥글게 하려면 UICorner를 자식으로 추가하면 됩니다.

버튼 클릭 처리

-- LocalScript를 버튼 안에 넣으세요
local button = script.Parent
local count = 0

button.MouseButton1Click:Connect(function()
    count = count + 1
    button.Text = "클릭: " .. count
end)
!!

GUI를 다루는 스크립트는 LocalScript여야 합니다. 일반 Script는 서버에서 돌기 때문에 개인 화면의 GUI를 건드릴 수 없습니다. 다음 챕터에서 이 구분을 자세히 다룹니다.

자주 쓰는 GUI 요소

요소용도
Frame배경 패널, 컨테이너
TextLabel점수, 제목 등 글자 표시
TextButton클릭 가능한 버튼
ImageLabel / ImageButton이미지 표시 / 이미지 버튼
ScrollingFrame스크롤 가능한 목록
UIListLayout자식들을 자동 정렬

17클라이언트와 서버 중급

Roblox 게임의 가장 중요한 개념. 이걸 이해해야 멀티플레이 게임을 만들 수 있습니다.

두 개의 세계

구분서버클라이언트
무엇게임을 운영하는 Roblox 컴퓨터각 플레이어의 기기
스크립트ScriptLocalScript
두는 곳ServerScriptServiceStarterPlayerScripts, StarterGui, StarterCharacterScripts
담당게임 규칙, 점수, 데이터 저장GUI, 카메라, 입력, 개인 연출
신뢰도신뢰 가능해킹 가능 — 절대 신뢰 금지

판단 기준

  • 모든 플레이어에게 영향: 서버 (Script). 예: 문 열기, 점수 변경, 적 스폰.
  • 한 플레이어 화면에만: 클라이언트 (LocalScript). 예: 버튼 UI, 카메라 흔들림, 효과음.

전용 저장소

  • ServerStorage: 서버만 접근. 적 템플릿, 보상 아이템 원본 보관.
  • ReplicatedStorage: 양쪽 다 접근. RemoteEvent, 공용 모듈 보관.
!!

클라이언트에서 바꾼 것(파트 색, 위치 등)은 그 플레이어 화면에만 보입니다. 다른 사람에게도 보이게 하려면 서버에서 바꿔야 합니다. "왜 나한테만 보이지?"의 원인 대부분이 이것입니다.

18RemoteEvent 통신 중급

클라이언트와 서버가 대화하는 방법. 버튼을 눌러 서버에 요청하는 구조를 만듭니다.

구조

  1. ReplicatedStorage에 RemoteEvent 추가. 이름: BuyItem.
  2. 클라이언트(LocalScript)에서 FireServer()로 요청.
  3. 서버(Script)에서 OnServerEvent로 받기.

클라이언트 쪽 (LocalScript, 버튼 안)

local RS = game:GetService("ReplicatedStorage")
local buyEvent = RS:WaitForChild("BuyItem")
local button = script.Parent

button.MouseButton1Click:Connect(function()
    buyEvent:FireServer("Sword")  -- 서버에 요청
end)

서버 쪽 (Script, ServerScriptService)

local RS = game:GetService("ReplicatedStorage")
local buyEvent = RS:WaitForChild("BuyItem")

buyEvent.OnServerEvent:Connect(function(player, itemName)
    -- 첫 인자는 항상 요청한 player가 자동으로 들어옴

    -- 서버에서 검증! 클라이언트가 보낸 값은 신뢰 금지
    if itemName ~= "Sword" and itemName ~= "Shield" then
        return  -- 이상한 값이면 무시
    end

    print(player.Name .. "이(가) " .. itemName .. " 구매")
end)

서버 → 클라이언트 방향

-- 서버에서
event:FireClient(player, "알림 메시지")     -- 한 명에게
event:FireAllClients("전체 공지")            -- 전원에게

-- 클라이언트에서 받기
event.OnClientEvent:Connect(function(msg)
    print(msg)
end)
!!

클라이언트가 보내는 데이터는 익스플로잇(핵)으로 조작될 수 있습니다. "돈이 충분한지", "그 아이템이 존재하는지" 같은 검증은 반드시 서버에서 하세요.

19TweenService 중급

속성을 부드럽게 변화시키는 애니메이션 도구. 문 열림, UI 슬라이드, 색 변화 전부 Tween으로.

기본 사용법

local TweenService = game:GetService("TweenService")
local part = workspace.Door

-- 1) 어떻게 움직일지 (시간, 가감속 스타일)
local info = TweenInfo.new(
    2,                          -- 2초 동안
    Enum.EasingStyle.Quad,      -- 가감속 곡선
    Enum.EasingDirection.Out    -- 끝에서 감속
)

-- 2) 어떤 속성을 어떤 값으로
local goal = {
    Position = part.Position + Vector3.new(0, 8, 0),
    Transparency = 0.5,
}

-- 3) 생성하고 실행
local tween = TweenService:Create(part, info, goal)
tween:Play()

tween.Completed:Wait()  -- 끝날 때까지 대기 (선택)

EasingStyle 종류

스타일느낌용도
Linear일정한 속도기계적 움직임
Quad / Sine자연스러운 가감속대부분의 상황. 기본 추천
Back살짝 넘어갔다 돌아옴탱탱한 UI 팝업
Bounce바닥에 튕기는 느낌떨어지는 연출
Elastic고무줄처럼 진동과장된 연출

GUI에도 동일하게

-- 패널이 화면 밖에서 미끄러져 들어오는 연출
local tween = TweenService:Create(frame,
    TweenInfo.new(0.4, Enum.EasingStyle.Back, Enum.EasingDirection.Out),
    {Position = UDim2.new(0.5, 0, 0.5, 0)}
)
tween:Play()

20사운드와 조명 중급

게임의 분위기를 만드는 두 축. 소리와 빛.

사운드 재생

  1. 파트(또는 Workspace)에 Sound 인스턴스 추가.
  2. SoundId에 rbxassetid://사운드ID 입력. Toolbox > Audio에서 검색 가능.
  3. 파트 안에 넣으면 3D 사운드(가까울수록 크게), Workspace 직속이면 전역 사운드.
local sound = script.Parent.Sound
sound.Volume = 0.5
sound.Looped = true   -- 배경음악용
sound:Play()
-- sound:Stop(), sound:Pause()

Lighting 핵심 속성

속성역할
ClockTime0~24 시간. 태양 위치와 하늘색 결정
Brightness전체 밝기
Ambient / OutdoorAmbient그림자 영역 / 야외 최소 밝기
FogEnd / FogColor안개 거리와 색
Technology렌더링 방식. Future가 가장 고품질

Atmosphere와 후처리 효과

Lighting 안에 추가하는 인스턴스들:

  • Atmosphere: 대기 산란, 안개, 노을. Density 0.3~0.5가 자연스러움.
  • BloomEffect: 밝은 부분이 빛나는 효과. Neon과 조합하면 화려함.
  • ColorCorrectionEffect: 전체 색감 보정. 채도, 대비, 색조.
  • BlurEffect: 화면 흐림. 메뉴 열렸을 때 배경 흐림 등.

개별 조명

-- 파트에 PointLight 추가 (코드로)
local light = Instance.new("PointLight")
light.Color = Color3.fromRGB(255, 180, 80)  -- 따뜻한 빛
light.Range = 16
light.Brightness = 2
light.Parent = workspace.Lamp
