💡

01설치와 첫 화면

무료입니다. 전부 다. 평생.

설치

blender.org/download에서 받으면 끝. Steam이나 Microsoft Store에도 같은 게 있어서 자동 업데이트가 편하면 그쪽도 좋아요. 사양 걱정은 거의 안 해도 됩니다 — 로우폴리 모델링은 10년 된 노트북에서도 잘 돌아가요.

🌐

한국어 vs 영어 UI: Edit ▸ Preferences ▸ Interface ▸ Translation에서 한국어로 바꿀 수 있지만, 영어 UI를 추천합니다. 유튜브 강좌·검색 결과의 99%가 영어 메뉴 기준이라, 영어로 두는 게 오히려 배우기 쉬워요. 이 가이드도 메뉴명은 영어로 표기합니다.

화면은 5개 구역만 알면 됩니다

처음 켜면 정보가 쏟아지는 것 같지만, 실제로 쓰는 건 아래 5개 구역이 전부입니다. 가운데 큐브가 있는 회색 공간이 우리가 99% 머무를 3D 뷰포트예요.

  • 3D 뷰포트 — 모델을 보고 만지는 공간. T로 왼쪽 툴바, N으로 오른쪽 사이드바(좌표·치수)를 열고 닫아요.
  • 아웃라이너 — 씬에 있는 모든 오브젝트 목록. 더블클릭으로 이름을 바꾸세요(F2도 동일). Tower_Base, Tower_Head처럼 이름을 잘 지어두면 로블록스에서 그대로 보입니다.
  • 프로퍼티 — 아이콘 탭 중 🔧 렌치(모디파이어)와 🔴 구슬(머티리얼) 두 개만 기억하면 됩니다.
  • 워크스페이스 탭 — 상단의 Modeling, UV Editing 탭은 작업 종류별 화면 프리셋. 클릭만 하면 화면이 알아서 바뀝니다.
🧯

망했을 때 3종 세트: 실행 취소 Ctrl+Z (Preferences ▸ System에서 횟수를 64+로 늘려두세요), 모든 기능 검색 F3 (이름만 알면 뭐든 찾아줌), 그리고 저장은 Ctrl+S 습관화. 블렌더는 종료 시 안 물어보고 꺼질 수 있어요!

02뷰포트 조작 — 카메라를 손에 붙이기

모델링 실력의 절반은 시점 이동 속도입니다. 여기서 10분만 연습하세요.

마우스 3종 조작

💻

노트북·휠 없는 마우스라면: Edit ▸ Preferences ▸ Input에서 Emulate 3 Button Mouse(Alt+좌클릭 드래그 = 회전)와 Emulate Numpad(키보드 위 숫자열로 시점 전환)를 체크하세요. 이 두 옵션이 노트북 유저의 생명줄입니다.

넘패드 = 시점 순간이동

🧭

"내 큐브 어디 갔어요?!" — 뷰포트에서 가장 흔한 사고. 해결: 물체를 클릭하고 넘패드 . 또는 Home. 3D 커서가 이상한 곳에 가 있으면 Shift+C로 원점 복귀.

03오브젝트 모드 — 물체 단위로 다루기

블렌더에는 두 개의 세계가 있습니다. 오브젝트 모드는 "물체 통째로", 에디트 모드는 "물체의 표면". 둘 사이는 Tab으로 오갑니다.

물체 추가하고 지우기

  • Shift+A — 추가 메뉴. Mesh 안의 Cube·Cylinder·UV Sphere·Plane이 모델링의 4대 재료입니다.
  • X 또는 Delete — 선택한 물체 삭제.
  • Shift+D — 복제. 마우스를 움직여 위치 잡고 클릭으로 확정.
  • F2 — 이름 변경. 나중에 로블록스에서 그대로 보이니 영어로 깔끔하게!
📐

화면 왼쪽 아래의 작은 패널을 절대 놓치지 마세요. 물체를 추가한 직후에만 나타나는 "오퍼레이터 패널"인데, 여기서 Cylinder의 Vertices를 32 → 8~12로 낮추면 그게 바로 로우폴리 타워의 시작입니다. 닫혔다면 F9로 다시 열 수 있어요.

이동·회전·크기 — G, R, S

블렌더에서 가장 많이 누르게 될 세 키. 누른 뒤 마우스를 움직이고, 좌클릭으로 확정, 우클릭(또는 Esc)으로 취소합니다. 마법은 그 다음입니다 — 키를 누른 상태에서 축 키와 숫자를 이어서 입력할 수 있어요.

원점(Origin)과 3D 커서 — 주황 점과 과녁

물체마다 있는 주황색 점이 원점입니다. 회전·크기 조절의 중심이고, 로블록스에서 파츠의 피벗이 돼요. 타워 포탑처럼 회전할 파츠는 원점을 회전축 위치에 두는 게 핵심입니다 (방법: 회전 중심에 Shift+우클릭으로 3D 커서를 놓고 → 우클릭 메뉴 Set Origin ▸ Origin to 3D Cursor). 빨강·하양 과녁 모양이 3D 커서로, 새 물체가 생성되는 위치이기도 합니다. Shift+C로 월드 중앙 복귀, Shift+S는 커서/선택을 서로 붙이는 스냅 파이 메뉴.

자주 쓰는 나머지

합치기 Ctrl+J

여러 물체를 하나로. 마지막에 선택한 물체 기준으로 합쳐집니다. 정적인 장식들은 합쳐서 내보내면 관리가 편해요.

숨기기 H / Alt+H

방해되는 물체 잠깐 치우기 / 전부 다시 보이기. /는 선택한 것만 보는 격리 모드(다시 누르면 해제).

스냅 이동 Ctrl 누른 채 드래그

격자 단위로 딱딱 끊어서 이동. 맵 타일을 배치할 때 필수입니다. 상단 자석 아이콘을 켜면 항상 스냅.

마지막 작업 수정 F9

방금 한 작업의 수치(베벨 폭, 회전 각도 등)를 다시 조절하는 패널을 띄웁니다.

⚠️

Apply — 내보내기 전 의식(儀式): 오브젝트 모드에서 S로 크기를 바꾸면 블렌더는 "1.7배로 보여주는 중"이라고 기억만 합니다. 이 상태로 베벨을 하면 두께가 뒤틀리고, FBX로 내보내면 크기가 이상해져요. 해결: Ctrl+A ▸ All Transforms. 내보내기 전엔 무조건 한 번! (10장 체크리스트에도 있습니다)

04에디트 모드 — 표면을 조각하기

물체를 선택하고 Tab. 이제 점·선·면의 세계입니다.

메시의 3요소: 버텍스 · 엣지 · 페이스

모든 3D 모델은 버텍스(점)를 엣지(선)로 잇고, 엣지가 둘러싼 페이스(면)로 이루어져요. 에디트 모드에서 키보드 위쪽 숫자 1 2 3으로 무엇을 선택할지 전환합니다.

선택 기술 — 모델링 속도의 8할

방법키이런 때
클릭 / 추가클릭 · Shift+클릭하나씩 콕콕
박스 선택B 또는 그냥 드래그영역으로 한꺼번에
서클(붓) 선택C (휠로 크기, 우클릭 종료)칠하듯 선택
루프 선택Alt+클릭물체를 한 바퀴 도는 줄 전체 — 제일 많이 씀
최단 경로Ctrl+클릭A에서 B까지 이어 선택
연결된 덩어리L (마우스 올리고) · Ctrl+L한 몸인 부분 전체

🩻

"뒤쪽 점이 선택이 안 돼요" — 뉴비가 반드시 겪는 함정. 기본값은 보이는 면만 선택됩니다. Alt+Z로 X-Ray(투시) 모드를 켜면 박스 선택이 뒷면 버텍스까지 관통해요. 정면 뷰(넘패드 1) + X-Ray + 박스 선택은 황금 조합.

보너스: O는 프로포셔널 편집 — 하나를 움직이면 주변이 부드럽게 따라옵니다(휠로 영향 반경 조절). 언덕 지형 만들 때 재밌어요.

05핵심 편집 도구 8가지

이 8개로 타워디펜스 에셋의 95%를 만들 수 있습니다. 큐브 하나 놓고 전부 한 번씩 눌러보세요.

E Extrude — 면을 뽑아내기 (모델링의 왕)

I Inset — 면 안에 작은 면

Ctrl+R Loop Cut — 허리선 추가

Ctrl+B Bevel — 모서리 다듬기

나머지 필살기 4종

나이프 K

표면에 자유롭게 칼집 내기. 클릭으로 경로를 긋고 Enter로 확정. C=45° 고정, Z=뒷면까지 관통.

채우기 F

버텍스/엣지를 선택하고 F를 누르면 사이에 면이 생깁니다. 뚫린 구멍 메울 때. (반대로 V는 찢기)

합치기 M ▸ By Distance

겹쳐진 중복 버텍스를 하나로. 내보내기 전 청소 루틴의 핵심입니다. "Removed N vertices" 메시지가 뜨면 성공.

분리 P / 슬라이드 GG

P ▸ Selection = 선택 부분을 별도 오브젝트로 (포탑 분리!). G 두 번 = 엣지를 표면 위에서만 미끄러뜨리기.

🔦

면이 시커멓게/투명하게 보인다면 노멀(면의 앞뒤)이 뒤집힌 것. 에디트 모드에서 A 전체 선택 → Shift+N으로 바깥 방향 재계산. 뷰포트 우상단 Overlays(겹친 원 아이콘) ▸ Face Orientation을 켜면 파랑=정상, 빨강=뒤집힘으로 보여줍니다. 로블록스에서 면이 사라져 보이는 사고의 1순위 원인!

삭제 X 메뉴에서 Delete는 구멍을 내고, Dissolve는 형태를 유지한 채 선만 녹입니다 — 폴리곤 정리엔 Dissolve가 보통 정답.

06모디파이어 — 자동화 마법

프로퍼티 패널의 🔧 렌치 탭. 원본을 건드리지 않고 효과를 "겹쳐 끼우는" 비파괴 시스템입니다.

모디파이어는 필터처럼 스택(위→아래 순서)으로 적용되고, 언제든 끄거나 수치를 바꿀 수 있어요. 확정하고 싶으면 모디파이어 드롭다운(∨)에서 Apply. 참고로 FBX 내보내기에는 Apply Modifiers 옵션이 있어서, 굳이 미리 적용하지 않아도 결과물에 반영됩니다.

Mirror — 반만 만들면 반은 공짜

Array — 복붙 공장

Boolean — 도형으로 도형 깎기

그 외 자주 쓰는 모디파이어

Subdivision Surface Ctrl+1~5

면을 잘게 쪼개 부드럽게. 주의: 레벨 1마다 폴리곤이 약 4배. 로우폴리 로블록스 에셋에는 거의 안 쓰거나 레벨 1까지만.

Bevel (모디파이어판)

모든 모서리를 일괄로 살짝 깎기. 수치 하나로 전체 분위기를 다듬을 수 있어 마감용으로 최고.

Solidify

종이처럼 얇은 면에 두께 부여. 벽, 방패, 깃발, 그릇 만들 때.

Decimate

폴리곤 수를 비율로 줄여주는 구조대. 로블록스 1만 트라이앵글 제한을 넘었을 때 Ratio를 낮춰 구출.

07셰이딩과 재질 — 색 입히기 1단계

로우폴리 스타일은 "단색 + 또렷한 면"이 전부라서, 알아야 할 게 의외로 적습니다.

Flat vs Smooth

오브젝트 모드에서 우클릭 ▸ Shade Smooth / Shade Flat. 로우폴리 감성은 보통 Flat(면이 또렷하게)이 정답이고, 원기둥처럼 둥근 부분만 부드럽게 하고 싶으면 Shade Auto Smooth(각도 기준 자동 — 최신 버전은 우클릭 메뉴에 있어요)를 쓰면 됩니다.

재질 만들고 면에 칠하기 — 로우폴리 색칠 표준 루틴

  1. 프로퍼티 패널의 구슬 아이콘(Material Properties) ▸ New. 이름을 Body처럼 지정.
  2. Base Color를 클릭해 색 선택. (Roughness ↑ = 무광, Metallic ↑ = 금속 — 일단 이 셋만 알면 충분)
  3. 슬롯 목록의 +로 재질을 2~3개 더 추가 (예: Roof, Metal).
  4. Tab 에디트 모드 → 3 페이스 모드로 색칠할 면 선택.
  5. 원하는 재질 슬롯을 고르고 Assign 버튼 클릭. 끝!

뷰포트에서 색이 안 보이면 Z ▸ Material Preview로 표시 모드를 바꾸세요.

재질 "색"만으로는 Studio에서 회색으로 보일 수 있습니다. 임포터가 머티리얼 컬러를 못 가져오는 경우가 있거든요. 가장 확실하고 업계 표준인 방법은 다음 장의 팔레트 텍스처 — 색을 작은 이미지 한 장에 담아 보내는 방식입니다. 재질 슬롯으로 면을 나눠두는 위 루틴은 그 준비 단계로도 그대로 유효해요.

08UV와 팔레트 텍스처 — 색 입히기 2단계

UV = 3D 표면을 2D 이미지에 펼쳐 대응시키는 지도. 선물 포장의 반대라고 생각하면 됩니다.

언랩(펼치기)의 원리

  • 정석: 잘 안 보이는 엣지를 선택 → Ctrl+E ▸ Mark Seam → A 전체 선택 → U ▸ Unwrap.
  • 급할 때: U ▸ Smart UV Project — 알아서 잘라줍니다. 팔레트 방식에는 이거면 충분!
  • UV 에디터 안에서도 똑같이 GRS로 조각(아일랜드)을 움직입니다.

⭐ 팔레트 텍스처 — 로블록스 로우폴리의 표준 색칠법

색깔 칸이 모인 아주 작은 PNG 한 장(예: 8×8칸, 256px)을 만들고, 각 부위의 UV 조각을 점만 하게 줄여 원하는 색 칸 위에 올려두는 기법입니다.

  1. 그림판/포토샵/Piskel 등으로 색상 칸 PNG 제작 (게임 전체가 공유할 팔레트).
  2. 블렌더 재질의 Base Color 옆 ⚪ 점 클릭 ▸ Image Texture ▸ Open으로 PNG 연결.
  3. U ▸ Smart UV Project로 펼치기.
  4. UV 에디터에서 부위별로 선택 → S로 아주 작게 → G로 원하는 색 칸 위에 이동.
  5. 뷰포트(Z Material Preview)에서 색 확인. 끝!
🎨

왜 이 방식이 최고냐면: 게임 전체가 텍스처 1장을 공유합니다 → 로블록스 메모리·로딩에 유리, 색 통일감 자동 확보, 나중에 PNG의 색만 바꾸면 전 에셋 리컬러까지. 텍스처를 그릴 줄 몰라도 됩니다.

참고로 표면에 직접 그림을 그리는 Texture Paint 워크스페이스도 있습니다 — 팔레트 방식에 익숙해진 뒤 무늬가 필요할 때 열어보세요.

09그 외 기능 한눈에 — 블렌더 전체 지도

"거의 모든 기능"을 약속했으니, 나머지 영역도 전부 소개합니다. 뱃지를 보고 지금 필요한 것만 골라 익히세요.

필수 타워디펜스 제작에 바로 필요  ·  알아두면 좋음 퀄리티가 올라감  ·  나중에 게임이 커지면 그때

알아두면 좋음

Sculpt Mode — 점토 조각

브러시로 표면을 밀고 당기는 모드. 바위, 지형, 유기체에 강합니다. 단, 폴리곤이 폭발하므로 로블록스용은 조각 후 Decimate나 리토폴로지가 필수.

알아두면 좋음

Texture Paint — 표면에 직접 그리기

3D 모델 위에 마우스로 바로 칠하는 워크스페이스. 팔레트 방식으로 부족할 때(낙서, 균열, 무늬) 여기서 PNG에 덧그립니다.

알아두면 좋음

Shader Editor — 재질을 노드로

재질을 레고 블록(노드)처럼 조립하는 화면. 우리가 쓴 Image Texture 연결도 사실 이 노드입니다. 로블록스는 복잡한 노드를 못 가져가니 Base Color에 꽂힌 텍스처 1장만 유지하세요.

나중에

Geometry Nodes — 절차적 모델링

"규칙"으로 모델을 자동 생성(돌담 흩뿌리기, 울타리 자동 배치). 강력하지만 학습 곡선이 가파릅니다. Array로 버틸 수 있을 때까진 보류.

알아두면 좋음

Armature & 애니메이션 — 뼈대와 움직임

뼈(본)를 심어 캐릭터를 움직이는 리깅 + 키프레임 애니메이션(I로 키 삽입, Dope Sheet/Graph Editor로 편집). 걸어다니는 적 유닛을 직접 애니메이팅하고 싶을 때 — 로블록스로는 본 포함 FBX를 가져가 Animation Editor와 연동합니다. 난이도 높음!

나중에

Physics & Simulation — 물리 시뮬

천, 강체, 유체, 파티클 시뮬레이션. 멋지지만 결과를 로블록스로 가져가긴 어렵습니다(굽기 필요). 게임 내 물리는 Studio가 담당.

알아두면 좋음

카메라 & 렌더 F12

씬을 사진으로 뽑는 기능. 게임 아이콘, 썸네일, 상점 이미지에 씁니다. Ctrl+Alt+0으로 현재 시점을 카메라로, Render Properties에서 EEVEE(빠름)/Cycles(예쁨) 선택.

나중에

Compositor — 렌더 후보정

렌더 결과에 글로우·색보정·블러를 노드로 얹는 단계. 썸네일 뽑을 때나 한 번 들여다보세요.

나중에

Video Sequence Editor — 영상 편집

블렌더 안의 미니 프리미어. 트레일러 영상을 자르고 붙일 수 있지만, 전용 편집 툴이 더 편한 경우가 많습니다.

나중에

Grease Pencil — 3D 공간에 2D 드로잉

3D 공간에 선을 그리는 2D 애니메이션/스케치 도구. 컨셉 낙서나 연출용. 에셋 제작과는 별개 세계.

알아두면 좋음

Add-ons — 숨은 공식 확장팩

Edit ▸ Preferences ▸ Add-ons에 공식 애드온이 잠들어 있어요. 체크만 하면 켜집니다. 추천: LoopTools(정리), Node Wrangler(노드 단축키), Bool Tool(불리언 빠르게).

알아두면 좋음

Asset Browser — 나만의 부품 창고

자주 쓰는 모델·재질을 에셋으로 등록(우클릭 ▸ Mark as Asset)해 드래그로 재사용. 타워 부품(포신, 지붕, 창문)을 모아두면 조립 속도가 비약적으로 빨라집니다.

🧭

길 잃지 않는 법: 위 기능들은 전부 상단 워크스페이스 탭(Layout, Modeling, Sculpting, UV Editing, Texture Paint, Shading, Animation, Rendering…)으로 연결됩니다. 탭은 "기능별로 미리 세팅된 화면 배치"일 뿐 — 눌러본다고 망가지는 건 없어요.

10로블록스로 내보내기 — FBX → Studio

여기가 블렌더와 로블록스가 만나는 다리입니다. 체크리스트 → FBX 설정 → Studio 임포트, 3단계면 끝.

① 내보내기 전 체크리스트

  1. 변환 굽기 — 오브젝트 모드에서 Ctrl+A ▸ All Transforms. 크기·회전이 1과 0으로 리셋되어 Studio에서 찌그러짐을 방지.
  2. 중복 점 청소 — 에디트 모드 A 전체 선택 → M ▸ By Distance.
  3. 노멀 정리 — Shift+N. 뒤집힌 면(투명하게 보이는 면) 제거.
  4. 트라이앵글 수 확인 — 뷰포트 우상단 Overlays(겹친 동그라미 2개) ▸ Statistics 체크 → 좌상단에 Tris 표시. MeshPart당 10,000 이하인지 확인.
  5. 이름 정리 — Outliner에서 더블클릭. TowerBase, TowerTurret처럼 영문으로 — Studio에서 그대로 파트 이름이 됩니다.
  6. 원점 확인 — 회전할 부품(터렛)의 원점이 회전축 위에 있는지 (§03 원점 다루기).

② FBX 내보내기 설정 File ▸ Export ▸ FBX (.fbx)

설정값왜?
Limit toSelected Objects ✓씬의 라이트·카메라까지 딸려가는 사고 방지. 내보낼 것만 선택 후 실행.
Object TypesMesh만 선택(애니메이션 없는 에셋 기준) 깔끔한 파일.
Apply Modifiers✓ (기본값)Mirror·Array·Bevel이 결과물에 구워져 나갑니다.
Path ModeCopy + 옆의 📦 아이콘 ON팔레트 PNG를 FBX 안에 내장 — 파일 하나만 옮기면 끝.
Forward / Up기본값 그대로Studio 임포터가 알아서 변환해 줍니다. 눕는다면 그때 조정.
💾

이 설정을 매번 만지기 귀찮다면 내보내기 창 상단 Operator Presets에서 +를 눌러 "Roblox" 프리셋으로 저장해 두세요. 다음부턴 두 클릭.

③ Roblox Studio에서 임포트

  1. Studio 상단 Home(또는 Avatar) 탭 ▸ Import 3D 클릭 → fbx 선택.
  2. 미리보기 창에서 경고(⚠) 확인 — 폴리곤 초과, 텍스처 누락이 여기서 다 잡힙니다.
  3. 크기가 이상하면 임포트 창의 Scale Unit/배율을 조정 (기본 Studs).
  4. Import → Workspace에 MeshPart(여러 개면 Model로 묶여서) 등장!
🎮
ROBLOX NOTE — 크기 감각

로블록스의 길이 단위는 스터드(stud), 1 스터드 ≈ 0.28 m. 기준은 캐릭터(R15) 키 ≈ 5 스터드입니다. 타워는 6~10 스터드, 적 유닛은 3~5 스터드 정도가 보기 좋아요. 블렌더에서 mm 단위로 고민하기보다, Studio에 캐릭터 옆에 놓고 Scale 도구로 맞추는 게 100배 빠릅니다.

⚠️

제한 요약: MeshPart당 트라이앵글 10,000개, 텍스처 1024×1024px 권장 최대. 그리고 임포트 후 MeshPart의 CollisionFidelity를 Box로 — 타워디펜스에서 정밀 충돌은 필요 없고, 성능은 크게 절약됩니다.

11실전: 타워디펜스 에셋 만들기

배운 것 총동원. 첫 타워를 단계별로 만들고, 적 유닛과 맵 타일까지 설계 기준을 잡아드립니다.

첫 타워 — 5단계 진화

  1. Shift+A ▸ Cylinder → F9에서 Vertices 12.
  2. Tab 에디트 모드 → Ctrl+R 가로 루프컷 2개 → 가운데 루프를 S로 살짝 줄여 허리 라인.
  3. 3 페이스 모드 → 윗면 선택 → I 인셋 → E로 머리(터렛 받침) 올리기.
  4. 머리 옆면 1개 선택 → E로 포신 돌출 → S로 끝을 살짝 좁히면 더 그럴듯.
  5. 모서리를 Ctrl+B로 한 번씩 베벨 → 로우폴리 특유의 단단한 느낌.
  6. 터렛 분리: 머리+포신 면 선택 → P ▸ Selection. 오브젝트 모드에서 터렛의 원점을 회전축에 — 3D 커서를 받침 중앙에 Shift+S ▸ Cursor to Selected → Object ▸ Set Origin ▸ Origin to 3D Cursor.
  7. 이름을 TowerBase / TowerTurret으로 → §08 팔레트로 색 → 둘 다 선택 후 FBX 내보내기.
🎮
ROBLOX NOTE — 터렛이 적을 바라보게

한 FBX로 가져오면 Studio에 Model 안에 두 MeshPart가 생깁니다. 터렛 회전은 스크립트에서 한 줄: turret.CFrame = CFrame.lookAt(turret.Position, enemyPos) — 원점(피벗)을 §03에서 제대로 잡았다면 제자리에서 깔끔하게 돕니다. 실전에선 Motor6D나 HingeConstraint로 부드럽게 보간하면 더 좋아요.

적 유닛 & 맵 — 설계 기준표

👾 적 유닛

300~800 트라이앵글. 좌우 대칭이니 Mirror 모디파이어로 절반만. 같은 몸통에 뿔·색만 바꿔 변종을 양산하세요(팔레트라 색 변경이 UV 이동 한 번).

🗺️ 맵 타일 — 모듈식

길 직선 / 코너 / T자 / 빈 땅 4종 타일을 같은 크기(예: 8×8 스터드)로 제작 → Studio에서 그리드 스냅으로 레고처럼 조립. 맵 100개도 타일 4개로.

📐 트라이앵글 예산

타워 800~2,000 · 적 300~800 · 소품(나무·돌) 50~300 · 맵 타일 100~500. 화면에 적이 100마리 떠도 버티는 숫자입니다.

♻️ 재사용이 곧 성능

Studio에서 같은 MeshPart를 복제하면 MeshId를 공유해 메모리를 아낍니다. "모델 가짓수는 적게, 배치는 많이"가 타워디펜스의 정석.

🏁

로드맵 제안: ① 위의 타워 1개 완성 → ② 적 1종 → ③ 타일 4종 → ④ 게임 루프에 연결 → ⑤ 그다음에야 디테일 욕심. 에셋 100개를 만들고 시작하는 게 아니라, 최소 세트로 게임을 먼저 돌리는 것이 완성의 비결입니다.
