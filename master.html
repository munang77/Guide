31OSL 셰이더 프로그래밍 초고급

Open Shading Language로 노드로는 만들 수 없는 재질을 코드로 직접 작성합니다. Cycles 전용.

OSL이란?

OSL(Open Shading Language)은 픽사, 소니픽처스 등 VFX 스튜디오에서 사용하는 셰이더 언어입니다. 블렌더 Cycles에서 Script 노드를 통해 사용할 수 있습니다. 노드로는 표현하기 어려운 복잡한 수학 기반 패턴이나 물리 모델을 코드로 직접 작성합니다.

사용 조건

  • 렌더 엔진: Cycles 전용. EEVEE에서는 작동하지 않습니다.
  • Render Properties > Sampling > Open Shading Language 켜기.
  • CPU 렌더링만 지원합니다(GPU OSL은 실험적).
  • Shader Editor에서 Script 노드 추가 > .osl 파일 연결.

OSL 기본 문법

// 체커보드 패턴 예시 OSL 셰이더
shader MyChecker(
    point UVW = P,        // 입력: 좌표
    float Scale = 4.0,   // 입력: 크기
    color Color1 = 0,    // 입력: 색상1 (검정)
    color Color2 = 1,    // 입력: 색상2 (흰색)
    output color Out = 0  // 출력: 결과 색상
) {
    // 좌표를 Scale로 나눈 나머지로 체커 계산
    point p = UVW * Scale;
    int xi = (int)floor(p[0]) % 2;
    int yi = (int)floor(p[1]) % 2;
    int zi = (int)floor(p[2]) % 2;

    if ((xi + yi + zi) % 2 == 0)
        Out = Color1;
    else
        Out = Color2;
}

OSL 주요 내장 함수

함수설명예시
noise(p)Perlin 노이즈 (0~1)float n = noise("perlin", P*4)
cellnoise(p)셀 노이즈. Voronoi 기반float c = cellnoise(P)
mix(a,b,t)두 값 사이 선형 보간mix(Color1, Color2, 0.5)
clamp(x,a,b)값을 a~b 범위로 고정clamp(n, 0, 1)
smoothstep(e0,e1,x)부드러운 0~1 전환smoothstep(0.4, 0.6, n)
transform(space,p)좌표계 변환transform("world","object",P)

diffuse(N)Lambertian 디퓨즈 클로저Ci = diffuse(N) * Color
reflection(N)완전 반사 클로저Ci = reflection(N)
refraction(N,eta)굴절 클로저Ci = refraction(N, 1.45)

OSL vs 노드 선택 기준

  • 노드 사용: 일반 PBR 재질, 표준 텍스처 조합, 팀원과 공유하는 재질.
  • OSL 사용: 수학적으로 복잡한 패턴, 절차적 기하학 패턴, 커스텀 BSDF 모델, 노드로 만들기 비효율적인 경우.

32커스텀 BSDF 설계 초고급

Principled BSDF 안에서 해결 안 되는 특수 재질을 기본 클로저를 조합해 직접 설계합니다.

BSDF란?

BSDF(Bidirectional Scattering Distribution Function)는 빛이 표면에서 어떻게 산란되는지를 수학적으로 정의한 함수입니다. Principled BSDF는 가장 일반적인 재질을 커버하지만, 특수한 재질(레이스, 벨벳 광섬유, 결정체 등)은 기본 클로저를 조합해 직접 만들어야 합니다.

기본 클로저 종류

클로저물리 의미재질
diffuse(N)완전 무광 산란 (Lambertian)분필, 무광 페인트
oren_nayar(N, sigma)거친 표면 산란천, 흙, 콘크리트
microfacet(dist, N, ag)미세면 반사 (GGX/Beckmann)금속, 광택면
translucent(N)반투과 (반대면 산란)종이, 얇은 천
transparent()완전 투과유리, 물
ashikhmin_shirley(N,T,nu,nv)비등방성(Anisotropic) 반사브러시 금속, 새틴
emission()발광LED, 화면, 용암
subsurface(N,r)내부 산란피부, 밀랍, 옥석

커스텀 벨벳 OSL 예시

// 벨벳: 시선과 수직인 방향이 밝아지는 특성
shader VelvetBSDF(
    color FiberColor = color(0.5, 0.05, 0.2),
    float Roughness = 0.8,
    output closure color BSDF = 0
) {
    normal Nf = faceforward(N, I, N);

    // 시선 방향과 노멀의 각도 계산
    float facing = 1.0 - fabs(dot(-I, Nf));

    // 외곽에서 밝아지는 산란 + 기본 디퓨즈
    closure color rim    = emission() * FiberColor * pow(facing, 2);
    closure color body   = diffuse(Nf) * FiberColor * (1 - facing * 0.7);
    closure color spec   = microfacet("ggx", Nf, Roughness, 1.0, 0);

    BSDF = body * 0.7 + rim * 0.2 + spec * 0.1;
}

33재질 라이브러리 구축 초고급

여러 프로젝트에서 재사용할 수 있는 재질 라이브러리를 체계적으로 관리합니다.

Asset Library 설정

  1. Edit > Preferences > File Paths > Asset Libraries > +.
  2. 라이브러리 폴더 경로 지정 (예: C:/BlenderAssets/).
  3. 재질을 만든 뒤 Shader Editor에서 재질 이름 옆 Mark as Asset 클릭.
  4. Asset Browser(Editor Type: Asset Browser)에서 드래그&드롭으로 오브젝트에 적용.

Link vs Append

방식특징언제 쓰나
Append (File > Append)복사해서 가져옴. 원본과 독립한 번만 쓸 때, 수정이 필요할 때
Link (File > Link)원본 파일과 연결 유지팀 작업, 원본 수정 시 자동 반영
Override LibraryLink로 가져온 것을 로컬에서 부분 수정팀 공용 캐릭터를 씬에서 포즈만 수정

34렌더팜 & 배치 렌더링 초고급

긴 애니메이션을 여러 컴퓨터에 분산하거나 자동화된 배치 작업으로 렌더링합니다.

Render Farm이란?

여러 대의 컴퓨터(노드)가 한 장면의 다른 프레임을 동시에 렌더링하는 시스템입니다. 100프레임짜리 애니메이션을 10대가 나눠 처리하면 이론상 10배 빨라집니다.

CLI 배치 렌더링 (명령줄)

# Windows PowerShell / macOS Terminal
# 프레임 1~100을 PNG로 렌더링
blender -b scene.blend -E CYCLES -o //output/frame_#### \
         -F PNG -s 1 -e 100 -a

# 옵션 설명
# -b  : 백그라운드 (GUI 없이)
# -E  : 렌더 엔진 (CYCLES / BLENDER_EEVEE)
# -o  : 출력 경로 (####은 프레임 번호)
# -F  : 파일 포맷
# -s / -e : 시작/끝 프레임
# -a  : 애니메이션 렌더

배치 렌더링 Python 스크립트

import bpy

scene = bpy.context.scene
scene.render.engine = 'CYCLES'
scene.cycles.samples = 256
scene.render.filepath = "//renders/frame_"
scene.render.image_settings.file_format = 'OPEN_EXR'
scene.frame_start = 1
scene.frame_end   = 240

bpy.ops.render.render(animation=True, write_still=True)

분산 렌더링 옵션

  • Flamenco (공식 무료): Blender Foundation에서 만든 오픈소스 렌더팜. blender.org/flamenco. 로컬 네트워크의 여러 컴퓨터를 묶을 수 있습니다.
  • SheepIt Render Farm: 무료 클라우드 렌더팜. 포인트 시스템. 자신도 다른 사람 렌더링을 도와주면 포인트를 얻습니다.
  • Google Colab / Cloud: Python 스크립트를 짜서 클라우드 VM에서 headless 렌더. 기술적 설정이 필요합니다.

EXR 렌더 + 후반 합성 워크플로우

현장에서는 각 패스(Diffuse, Specular, AO, Shadow, Emission 등)를 별도로 EXR에 저장한 뒤, DaVinci Resolve나 Nuke, 또는 블렌더 컴포지터에서 합성합니다. 한 번 렌더한 뒤 색보정이나 효과를 바꿔도 재렌더링이 필요 없습니다.

35VFX 파이프라인 초고급

모델링, 리깅, 애니메이션, 시뮬레이션, 렌더링, 합성까지 하나의 파이프라인으로 연결합니다.

VFX 파이프라인 개요

단계도구산출물
1. 에셋 제작블렌더 (모델링/텍스처)FBX / OBJ / USD 에셋
2. 리깅블렌더 (Armature / Rigify)뼈대 있는 FBX
3. 애니메이션블렌더 (Dope Sheet / NLA)Alembic (.abc) 캐시
4. 시뮬레이션블렌더 (Cloth / Fluid / Particles)Alembic 캐시
5. 렌더링블렌더 Cycles / EEVEEEXR 시퀀스 (패스 분리)
6. 합성블렌더 컴포지터 / Nuke / DaVinci최종 영상

에셋 버전 관리

  • 파일 명명: asset_name_v001.blend 식으로 버전 번호. 덮어쓰기 금지.
  • Git LFS: 대용량 .blend 파일도 Git으로 버전 관리 가능. Git LFS(Large File Storage) 설정 필요.
  • Kitsu / Zou: 오픈소스 프로덕션 트래커. 블렌더와 연동되는 전용 애드온이 있습니다.

Multi-Layer EXR 워크플로우

렌더 시 Output Properties > Output > File Format을 OpenEXR MultiLayer로 설정합니다. 단일 EXR 파일 안에 모든 렌더 패스가 레이어로 저장됩니다. 컴포지터나 Nuke에서 각 레이어를 독립적으로 불러올 수 있습니다.

36Alembic / USD 교환 초고급

다른 DCC 툴(Maya, Houdini, Cinema 4D 등)과 애니메이션/시뮬레이션 데이터를 교환합니다.

Alembic (.abc)

애니메이션된 메시, 카메라, 파티클 경로를 프레임별로 저장하는 캐시 포맷입니다. 뼈대 없이 메시 자체가 움직이는 정보(Vertex Cache)를 저장합니다.

  • 내보내기: File > Export > Alembic. 프레임 범위, XForm, Curves, Hair 포함 여부 선택.
  • 가져오기: File > Import > Alembic. 시뮬레이션 결과를 다른 프로그램에서 가져올 때.
  • 캐시 모디파이어: 오브젝트에 Mesh Sequence Cache 모디파이어 추가로 .abc 파일 연결. 실시간 스트리밍.

USD (Universal Scene Description)

픽사가 만든 씬 기술 포맷. 거대한 씬을 여러 파일로 분리해서 관리하고, 여러 팀이 동시에 작업하는 파이프라인에 적합합니다. 블렌더 4.x에서 USD 지원이 대폭 강화됐습니다.

  • Stage: 여러 Layer로 이루어진 씬 전체. 레이어 간 Override 지원.
  • Prim: 씬 계층 구조의 각 요소 (Mesh, Camera, Light 등).
  • 블렌더 지원: File > Import/Export > Universal Scene Description(.usd, .usda, .usdc, .usdz).
TIP

Houdini에서 유체/파이어 시뮬레이션을 한 뒤 Alembic으로 블렌더에서 렌더링하는 파이프라인이 인디 VFX에서 흔히 쓰입니다. 각 소프트웨어의 강점을 활용하는 방식입니다.

37프로덕션 리토폴로지 초고급

게임/애니메이션 프로덕션 기준의 정밀한 리토폴로지 기법.

프로덕션 토폴로지 규칙

  • 엣지 루프 흐름: 관절(어깨, 무릎, 손목, 입)을 따라 엣지 루프가 돌아야 합니다. 변형 시 메시가 찢어지지 않는 유일한 방법.
  • 폴 위치: 5방향 폴(5-pole)은 덜 움직이는 부위(정수리, 귀 뒤)에. 3방향 폴(3-pole)은 엣지를 줄여야 하는 전환점에.
  • 균일한 폴리곤 밀도: 얼굴처럼 디테일이 필요한 곳은 조밀하게, 배처럼 단순한 부분은 성기게. 하지만 인접한 두 영역의 밀도 차이가 4배를 넘지 않도록.
  • 삼각형 최소화: 변형 영역에는 절대 삼각형이 없어야 합니다. 외곽선, 움직이지 않는 딱딱한 부분에만 허용.

리토폴로지 워크플로우

  1. 스컬프트로 완성한 고해상도 메시를 레퍼런스로 유지.
  2. 새 오브젝트를 추가하고 Shrinkwrap 모디파이어로 고해상도 표면에 자동으로 달라붙게 설정.
  3. Mirror 모디파이어를 Shrinkwrap 위에 추가. 한쪽만 리토폴로지하면 됩니다.
  4. 에디트 모드에서 면을 하나씩 추가하며 루프 흐름을 따라 메시를 만들어 나갑니다.
  5. 완성 후 모디파이어 Apply. 고해상도에서 노멀맵 베이킹.

38NURBS / 서피스 초고급

산업 디자인, 자동차, 제품 디자인에서 쓰이는 수학 기반 곡면 모델링.

NURBS란?

NURBS(Non-Uniform Rational B-Spline)는 점이 아닌 수학 함수로 정의되는 곡선과 곡면입니다. 어떤 해상도로 확대해도 완벽하게 매끄럽습니다. 자동차 차체, 병, 안경, 항공기 동체 같은 산업 제품 설계에 사용됩니다.

블렌더에서 NURBS

Shift+A > Curve > NURBS Curve 또는 Surface > NURBS Surface.

  • 컨트롤 포인트: 곡선/곡면을 당기는 점. Bezier와 달리 점이 곡선 위에 있지 않습니다.
  • Order: 차수. 높을수록 부드럽지만 무거워짐. 3~5가 일반적.
  • Weight: 컨트롤 포인트의 영향력. 높을수록 곡선이 그 점에 더 가깝게 당겨짐.
  • Knot: 각 구간의 파라미터 값. Endpoint(시작/끝에 고정), Bezier, Cyclic.

NURBS 서피스 만들기

  1. NURBS Curve 두 개를 단면 모양으로 만들기.
  2. Loft: Surface > Surfaces 패널에서 두 커브 선택 후 Fill 또는 Skin 적용. 두 커브를 이어 붙이는 서피스가 생성.
  3. Birail: 두 경계 커브와 하나의 프로파일 커브로 서피스 생성.
  4. 완성 후 Object > Convert > Mesh로 변환해 일반 메시 작업 가능.
ADV

블렌더의 NURBS 지원은 Rhino, CATIA에 비해 제한적입니다. 산업 수준의 NURBS 모델링이 필요하다면 FreeCAD(무료)나 Rhino(유료)와 조합하고, STEP/IGES 포맷으로 교환 후 블렌더에서 렌더링하는 워크플로우를 쓰는 것이 현실적입니다.

39의상/천 시뮬레이션 마스터 초고급

프로덕션 수준의 의상 시뮬레이션. 여러 천 레이어, 충돌, 핀닝 고급 기법.

의상 시뮬레이션 순서

  1. 레스트 포즈 제작: 캐릭터를 A-포즈 또는 T-포즈 상태에서 의상 메시를 만듭니다. 이 형태가 시뮬레이션의 기준 상태가 됩니다.
  2. Collision 추가: 캐릭터 본체(피부 메시)에 Physics > Collision. Thickness Inner를 0.005 정도로 낮게 설정.
  3. Cloth 추가: 의상 메시에 Physics > Cloth. Presets에서 Cotton, Silk, Leather 등 시작점 선택 후 커스터마이즈.
  4. Pinning: 어깨, 허리 등 고정될 버텍스를 선택 > 버텍스 그룹 만들기 > Cloth > Shape > Pin Group에 연결.
  5. 시뮬 품질: Cloth > Quality Steps를 10~20으로. 기본 5보다 안정적.
  6. 캐시 베이크: Cloth 캐시 탭 > Bake. 전 프레임을 미리 계산해 저장.

여러 레이어 천 처리

티셔츠 위에 재킷을 입히는 것처럼 여러 겹의 천을 처리할 때는 안쪽 레이어가 먼저 시뮬되어야 합니다. 또는 안쪽 천을 Proxy(저해상도)로 시뮬한 뒤 그 결과를 바깥 천의 충돌면으로 쓰는 방식을 씁니다.

시뮬레이션 안정화 팁

  • 초기 프레임에서 폭발이 생기면 Shrink Factor를 0.05~0.1로 설정. 시뮬 시작 시 의상이 약간 수축해서 충돌면 안으로 들어가지 않습니다.
  • Self Collision은 성능을 많이 씁니다. 반드시 필요한 경우에만 켜고, Distance를 최소한으로.
  • Vertex Mass를 줄이면 가벼워지고 흔들림이 심해집니다. 무거운 의상은 높게.

40커스텀 애드온 제작 초고급

반복 작업을 버튼 하나로 만드는 커스텀 애드온. 팀에 배포하거나 오픈소스로 공유 가능.

애드온 최소 구조

# my_addon/__init__.py
import bpy

bl_info = {
    "name":        "My Addon",
    "author":      "내 이름",
    "version":     (1, 0, 0),
    "blender":     (4, 0, 0),
    "category":    "Mesh",
    "description": "내가 만든 도구",
}

# Operator: 실제로 실행되는 작업
class MESH_OT_my_tool(bpy.types.Operator):
    bl_idname      = "mesh.my_tool"
    bl_label       = "내 도구 실행"
    bl_description = "선택한 오브젝트를 처리합니다"
    bl_options     = {'REGISTER', 'UNDO'}

    # 속성 (UI에서 입력받는 값)
    scale: bpy.props.FloatProperty(name="Scale", default=2.0, min=0.1)

    def execute(self, context):
        for obj in context.selected_objects:
            obj.scale *= self.scale
        return {'FINISHED'}


# Panel: 사이드바(N 패널)에 버튼 추가
class VIEW3D_PT_my_panel(bpy.types.Panel):
    bl_label       = "내 애드온"
    bl_idname      = "VIEW3D_PT_my_panel"
    bl_space_type  = 'VIEW_3D'
    bl_region_type = 'UI'
    bl_category    = "My Tools"

    def draw(self, context):
        self.layout.operator("mesh.my_tool")


classes = [MESH_OT_my_tool, VIEW3D_PT_my_panel]

def register():
    for cls in classes:
        bpy.utils.register_class(cls)

def unregister():
    for cls in reversed(classes):
        bpy.utils.unregister_class(cls)

if __name__ == "__main__":
    register()
