21대형 맵 설계 전략 고급

맵이 커질수록 무작정 만들면 망합니다. 구역화, 재사용, 밀도 관리가 핵심.

구역(Zone) 단위로 쪼개기

  1. 맵 전체를 4~8개 구역으로 나눕니다 (숲 구역, 마을, 호수, 폐허...).
  2. Explorer에서 구역마다 Folder를 만들어 모든 파트/모델을 소속 구역 폴더에 넣습니다.
  3. 한 구역을 완성하고 다음 구역으로 — 전체를 동시에 만지면 영원히 안 끝납니다.
  4. 구역 사이는 길목(다리, 협곡, 문)으로 연결합니다. 길목은 시야를 한 번 끊어줘서 렌더 부담도 줄입니다.

밀도 배분

  • 핵심 구역(플레이어가 오래 머무는 곳): 디테일 소품, SurfaceAppearance, 개별 조명을 아낌없이.
  • 이동 구역(지나가는 길): 중간 밀도. 시선 끝에 볼거리 하나씩.
  • 배경 구역(못 가는 곳): 로우폴리 단색 덩어리. 어차피 멀리서만 봅니다.
  • 전체를 균일한 밀도로 채우면 성능은 망하고 인상은 밋밋해집니다. 강약 조절이 곧 실력.

재사용 모듈 만들기

집 3종, 나무 4종, 바위 5종, 울타리 2종 — 이런 기본 모듈 세트를 먼저 만들고, 크기/회전/색 변주로 조합하면 대형 맵도 일관된 스타일로 빠르게 채워집니다. 모듈은 ReplicatedStorage나 별도 보관 폴더에 원본을 두고 복제해서 씁니다.

TIP

유명 게임 맵도 뜯어보면 같은 모델 수십 개의 변주입니다. 모듈 10개를 잘 만드는 것이 소품 100개를 대충 만드는 것보다 낫습니다.

22StreamingEnabled 고급

큰 맵의 필수 기능 — 플레이어 주변만 로드해서 메모리와 로딩 시간을 극적으로 줄입니다.

켜는 법과 핵심 속성

  1. Explorer에서 Workspace 선택 > Properties.
  2. StreamingEnabled 체크.
  3. StreamingMinRadius: 플레이어 주변 항상 로드되는 반경. 256~512 권장.
  4. StreamingTargetRadius: 여유 있을 때 로드를 시도하는 반경. 1024+ .
  5. StreamOutBehavior: 범위 밖 처리. LowMemory면 메모리 부족할 때만 내림 — 보통 이걸로.

ModelStreamingMode

중요한 Model은 개별로 스트리밍 방식을 정할 수 있습니다 (Model 선택 > ModelStreamingMode):

모드동작쓰는 곳
Default일반 스트리밍대부분의 맵 구조물
Atomic모델이 통째로 로드/언로드반쪽만 보이면 이상한 것 (건물, 다리)
Persistent항상 로드 유지스폰 지점, 핵심 랜드마크, 게임에 늘 필요한 것
!!

스트리밍을 켜면 멀리 있는 파트는 클라이언트에 존재하지 않습니다. 멀리서 봐야 하는 랜드마크(거대한 탑, 산)는 Persistent로 지정하거나, 별도의 단순한 원거리용 모형을 두세요. 안 그러면 지평선이 텅 비어 보입니다.

23파트 수 최적화 고급

파트 수는 맵 성능의 제1 변수. 측정하고, 합치고, 줄입니다.

측정부터

  • Play 중 Ctrl+Shift+F2 (Stats) — Instance 수, 렌더 시간 확인.
  • Play 중 Ctrl+F6 (MicroProfiler) — 프레임별로 무엇이 느린지.
  • View > Performance — Memory, Recv, FPS 요약.

파트 수 가이드라인

상태총 파트 수대응
쾌적~5,000걱정 없음
주의5,000~20,000스트리밍 켜기, 합치기 시작
위험20,000+모바일에서 문제 발생 가능. 적극 정리

줄이는 방법 (효과 순)

  1. 대형 파트로 교체: 바닥 타일 100개 → 큰 파트 1개. 가장 효과 큽니다.
  2. 블렌더 메시로 통합: 파트 30개로 조립한 집 → 블렌더에서 하나의 메시로 다시 만들어 임포트. MeshPart 1개가 됩니다. 디테일 많은 구조물일수록 이득.
  3. Union (소극적으로): 같은 재질 인접 파트 합치기. 단, 복잡한 Union은 오히려 무거워질 수 있어 단순 형태에만.
  4. 안 보이는 파트 삭제: 벽 속에 파묻힌 파트, 지형 아래 깔린 파트 — 보이지도 않는데 렌더와 물리를 먹습니다.
  5. 중복 텍스처 정리: 같은 그림인데 ID가 다른 텍스처들을 하나로 통일 — 메모리 절약.

24그림자와 렌더 최적화 고급

파트 수 다음은 렌더링. 그림자, 머티리얼, 메시 품질을 다이얼처럼 조절합니다.

그림자

  • 작은 소품 전부 CastShadow: false — 풀, 돌멩이, 장식. 그림자 계산이 통째로 빠집니다.
  • GlobalShadows(Lighting)를 끄면 모든 그림자 제거 — 로우폴리 밝은 스타일이면 고려할 만합니다.
  • 개별 조명(PointLight 등)의 Shadows 속성은 꼭 필요한 조명만 true로. 그림자 만드는 조명은 비쌉니다.

MeshPart 렌더 품질

  • RenderFidelity: Automatic이 기본 — 멀면 자동으로 단순화(LOD). 배경 소품은 Performance로 강제 단순화. Precise는 가까이서 보는 핵심 모델만.
  • DoubleSided: 양면 렌더. 필요한 메시(잎사귀 평면)만 켜고 기본은 끄기 — 렌더 비용 2배 차이.

머티리얼/이펙트 비용

요소비용권장
SmoothPlastic최저제한 없음
일반 머티리얼 (Brick, Wood...)낮음제한 없음
Glass / ForceField중간투명 겹침 줄이기
SurfaceAppearance중간 (텍스처 메모리)핵심 모델만, 에셋 공유
ParticleEmitter높음 (Rate 비례)Rate 합계 관리, 멀면 안 보이게

25충돌과 물리 설정 고급

보이는 것과 부딪히는 것을 분리하면 성능과 플레이 감각이 동시에 좋아집니다.

파트별 충돌 속성 총정리

속성의미끄면 좋은 것
CanCollide물리 충돌풀, 장식, 통과해도 되는 것
CanQueryRaycast 등 감지 대상장식 전부 — 감지 연산에서 제외
CanTouchTouched 감지 대상장식 전부
Anchored물리 고정맵 구조물은 전부 true — false인 파트는 물리 계산 대상
!!

맵에서 Anchored가 풀린 파트 하나가 끼어서 덜덜 떨고 있으면 그것만으로 물리 부하가 생깁니다. 모든 맵 파트는 Anchored true가 기본입니다.

충돌 단순화 — 보이지 않는 충돌체

모양이 복잡한 메시(바위 더미, 부서진 담장)는 정밀 충돌을 켜는 대신, 투명한 단순 파트를 충돌용으로 따로 깔고 메시 자체는 CanCollide false로 둡니다:

  1. 복잡한 메시 위에 박스/웨지 파트 몇 개로 대략의 충돌 형태를 만듭니다.
  2. 충돌 파트: CanCollide: true Transparency: 1
  3. 메시: CanCollide: false + CollisionFidelity는 Box로.
  4. 계단도 마찬가지 — 보이는 건 계단, 충돌은 경사 Wedge 하나. 캐릭터가 덜컹거리지 않고 미끄러지듯 올라갑니다. 대형 게임들의 표준 기법입니다.

CollisionFidelity — 메시 충돌 정밀도

MeshPart는 충돌 형태를 4단계로 정할 수 있습니다(Properties > CollisionFidelity). 정밀할수록 무겁습니다.

모드정밀도성능쓰임
Box가장 단순 (직육면체)가장 가벼움장식, 멀리 있는 것
Hull외곽만 감싸기가벼움대충 막으면 되는 바위
Default중간보통일반 구조물
PreciseConvexDecomposition가장 정밀무거움구멍 통과 등 정밀 필요 시만
TIP

대부분의 맵 메시는 Box나 Hull로 충분합니다. 정밀 모드는 "캐릭터가 그 구멍을 정확히 통과해야 한다" 같은 특수한 경우에만. 무심코 Precise로 둔 메시 수백 개가 렉의 원인인 경우가 많습니다.

충돌 확인하고 디버깅하기

  • 충돌 형태 보기: Studio 설정에서 Visualization > Show Decomposition Geometry를 켜면 실제 충돌 형태가 색으로 보입니다. 보이는 모양과 다르면 여기서 확인.
  • 벽에 끼임: 캐릭터가 벽에 박히거나 떨면 충돌 형태가 너무 복잡하거나 겹친 것 — 단순 충돌체로 교체.
  • 바닥 통과(떨어짐): CanCollide가 꺼져 있거나, 얇은 파트(두께 1 미만)는 빠른 속도에서 통과할 수 있음 — 두께를 늘리거나 아래 보강 파트.
  • 안 닿아야 할 게 닿음: 장식인데 길을 막으면 CanCollide false.
  • 투명벽으로 경계: 맵 밖으로 못 나가게 Transparency 1 + CanCollide true 파트로 보이지 않는 벽을 두르세요.

26패키지와 에셋 관리 고급

맵이 커지면 모델 관리가 일이 됩니다. 패키지로 한 번에 갱신하고, 에셋을 체계적으로 보관합니다.

패키지 (Package)

  1. 잘 만든 Model 우클릭 > Convert to Package.
  2. 맵 곳곳에 이 패키지를 복제해서 배치합니다.
  3. 원본을 수정하고 Publish changes하면 — 맵에 깔린 모든 복제본이 한 번에 업데이트됩니다.
  4. 가로등 50개의 디자인을 바꿔야 할 때, 패키지면 1번, 아니면 50번 작업입니다.

대형 맵일수록 플러그인이 한 번에 수천 개 파트를 생성/변경할 수 있습니다. 큰 작업 전에는 반드시 버전을 저장하고, 무거운 플러그인 작업은 별도 테스트 플레이스에서 먼저 시험하세요.

TIP

스크립트까지 다룰 줄 안다면(Luau 초고급 40) Rojo 같은 외부 도구 체인으로 넘어가는 길도 있습니다. 다만 순수 맵 제작에서는 위 플러그인들로 충분히 프로 수준의 생산성을 낼 수 있습니다.

  2. Atmosphere 추가: Lighting에 Atmosphere가 없으면 추가 > Density·Haze·Color 입력. 원근감과 분위기가 살아납니다.
  3. 머티리얼 통일: 바닥·벽·주요 구조물을 테마 머티리얼로. (숲=Grass/Wood, 던전=Slate, 화산=Basalt)
  4. 포인트 조명: 횃불·네온·빛기둥 같은 국소 광원으로 시선 유도와 디테일.
  5. 소품과 파티클: 마지막에 표의 소품(버섯, 백골, 불씨)과 파티클로 생기를.

분위기를 끌어올리는 연출 디테일

  • 빛기둥(God Rays): 반투명(Transparency 0.9) Neon 원기둥을 창문·천장 구멍 아래 비스듬히 — 신전·던전에 극적인 빛내림.
  • 젖은 바닥: 바닥 파트의 Reflectance를 0.1~0.3 — 밤 도시·비 온 뒤 느낌. 네온이 반사돼 화려해집니다.
  • 안개로 깊이감: Atmosphere Haze를 올리면 먼 곳이 뿌예져 공간이 넓어 보이고 먼 파트의 디테일 부담도 줄어듭니다.
  • 색 대비: 따뜻한 주광(횃불) + 차가운 그림자(달빛 Ambient)를 섞으면 입체감이 확 삽니다.
  • 움직이는 요소: 떠다니는 불씨·먼지·잎사귀 파티클을 약하게 — 정적인 맵이 살아 있는 느낌이 됩니다.
  • 소리도 테마: 환경음(새소리/바람/용암 끓는 소리)이 더해지면 분위기가 두 배 (중급 19 참고).

같은 맵, 다른 시간대

하나의 맵을 낮/밤/노을로 바꾸면 완전히 다른 분위기를 공짜로 얻습니다. ClockTime과 Ambient만 바꾸면 됩니다.

시간대ClockTime느낌
맑은 아침7~9상쾌, 시작 지점
한낮12~13밝고 또렷, 그림자 짧음
노을17~18주황빛, 감성적, 클라이맥스
밤0~2긴장, 조명 연출이 핵심

28이펙트 메시 배치 고급

블렌더에서 만든 검기/스킬 이펙트 메시(블렌더 고급 EX챕터)를 맵 장식으로 활용하는 방법. 코드 없이 배치만으로.

맵 장식으로 쓰는 이펙트 메시

  • 마법진: 신전 바닥, 보스방 입구, 포탈 아래에 깔기. Neon + 보라/하늘색.
  • 에너지 구체: 크리스탈 꼭대기, 제단 위에 띄우기. 2겹(코어+글로우)으로.
  • 빔: 하늘로 솟는 빛기둥 — 랜드마크나 목표 지점 표시로. 멀리서도 보여 길찾기에 좋습니다.
  • 충격파 링: 포탈 주변에 크기 다르게 2~3개 겹쳐 두면 정지 상태로도 에너지장 느낌.
  • 회오리: 마법 분수, 저주받은 탑 꼭대기.

배치 시 속성

  1. 모든 이펙트 메시: Anchored: true CanCollide: false CastShadow: false CanQuery: false
  2. Material Neon + Lighting에 BloomEffect — 발광이 살아납니다.
  3. Transparency 0.2~0.5로 살짝 비치게 하면 에너지 느낌이 더 납니다.
  4. 겹칠 땐 안쪽 밝은 색(흰/연한색), 바깥쪽 테마색.

움직임이 필요하면

회전하는 마법진, 맥동하는 구체 같은 움직임은 스크립트(TweenService)가 필요합니다 — 이 가이드는 맵 제작 범위라 다루지 않지만, 정지 상태 배치만으로도 충분히 효과적입니다. 움직임 없이도 Bloom과 반투명 겹침이 살아있는 느낌을 만들어 줍니다.

RBX

이펙트 메시를 만드는 법(호 메시, UV 그라데이션, 구체/빔/링/마법진/회오리)은 블렌더 고급 강좌의 "게임 스킬 이펙트 메시" 챕터에 있습니다.

29모바일 대응 고급

Roblox 유저 대부분은 모바일입니다. PC에서만 잘 돌아가는 맵은 절반의 맵입니다.

모바일에서 테스트하기

  1. Test 탭 > Device — 화면을 휴대폰/태블릿 크기로 에뮬레이션.
  2. 저사양 그래픽 확인: Studio Settings > Rendering > EditQualityLevel을 Level 1~3으로 낮춰 보기 — 저사양 기기가 보는 화면입니다.
  3. 이 상태로 맵을 돌아다니며: 길이 구분되는지, 중요한 것이 보이는지 확인.

저사양에서 무너지는 것들

  • 그림자와 조명: 저품질에선 그림자가 거의 사라집니다. "그림자가 있어야만" 구분되는 지형/길은 위험 — 색과 머티리얼로도 구분되게.
  • Future 라이팅: 저사양에선 자동으로 다운그레이드됩니다. Future 전제의 연출(정밀한 SpotLight 그림자 등)은 대체 수단을 함께.
  • 파티클: 저사양에서 수가 줄어듭니다. 파티클로만 표시하는 중요한 표지는 금물.
  • 먼 디테일: RenderFidelity Automatic이 더 공격적으로 단순화합니다 — 멀리서도 형태가 유지되게 실루엣을 단순하게.

모바일 친화 맵 수칙

  • 총 파트/인스턴스 수를 PC 기준보다 30~50% 빡빡하게 관리.
  • StreamingEnabled 필수.
  • 핵심 동선은 넓게 — 터치 조작은 정밀한 이동이 어렵습니다. 좁은 외길 점프맵 요소는 모바일 이탈 1순위.
  • 중요 정보는 화면 중앙부에 — 작은 화면 가장자리는 손가락에 가려집니다.

30맵 점검 체크리스트 고급

퍼블리시 전 마지막 점검. 위에서부터 순서대로 확인하세요.

구조

  • Explorer가 구역 폴더로 정리되어 있고, 이름 없는 "Part" 가 남아있지 않음
  • 맵 밖으로 나갈 수 있는 구멍/틈 없음 (벽 모서리, 지형과 파트 경계 특히)
  • 끼이는 곳 없음 — 좁은 틈, 소품 사이를 직접 걸어서 확인
  • 스폰 위치가 안전하고 (떨어지거나 끼지 않음) 방향이 맵 중심을 향함

성능

  • 모든 맵 파트 Anchored: true
  • 장식 소품 CanCollide / CanQuery / CanTouch / CastShadow: false
  • StreamingEnabled 켜짐, 랜드마크는 Persistent
  • 안 보이는 파트, 중복 텍스처, 미사용 에셋 삭제
  • Stats(Ctrl+Shift+F2)에서 파트 수 확인 — 목표 범위 내
  • 저품질(EditQualityLevel 1~3) + 모바일 에뮬레이션에서 FPS와 가독성 확인

완성도

  • 맵 경계가 자연스럽게 처리됨 (절벽/바다/산 — 갑자기 끊기는 투명벽 없음)
  • 어두운 구역에서도 길이 보임 (Ambient 또는 광원 확인)
  • BGM과 구역 환경음 동작, 볼륨 밸런스
  • 스카이박스와 먼 배경이 비어 보이지 않음
  • 다른 사람에게 목적지를 말하지 않고 걷게 해보기 — 길을 잃으면 시선 유도 보강
TIP

완성 후 하루 묵혔다가 다시 보세요. 만들 때는 안 보이던 어색한 부분이 선명하게 보입니다. 그리고 가장 좋은 점검은 친구를 초대해 말없이 지켜보는 것입니다.
