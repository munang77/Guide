Studio 중급 — 본격 맵 제작

맵 설계부터 지형, 건축, 임포트, 조명, 사운드까지. 진짜 게임처럼 보이는 맵을 만드는 모든 기술.

11맵 설계와 레이아웃 중급

바로 만들기 시작하면 중간에 다 갈아엎게 됩니다. 종이 한 장 분량의 설계가 며칠을 아낍니다.

만들기 전에 정할 것

  1. 맵의 용도: 탐험 맵인지, 전투 맵인지, 로비인지. 용도가 동선을 결정합니다.
  2. 크기: 플레이어가 걸어서 끝에서 끝까지 몇 초 걸릴지. 로비는 20~30초, 오픈월드는 2~3분 이상.
  3. 테마: 숲, 사막, 도시, 던전, 설원 등. 테마가 머티리얼과 색을 결정합니다.
  4. 랜드마크: 멀리서도 보이는 큰 구조물 1~2개 (탑, 거목, 산). 플레이어가 길을 잃지 않게 하는 기준점.

탑뷰 스케치

종이나 그림판에 위에서 본 맵을 대충 그립니다. 주요 구역(스폰, 중심 광장, 구역별 테마), 길의 연결만 표시하면 됩니다. 그 다음 Studio에서 색만 다른 큰 파트들을 깔아 구역을 잡는 것이 그레이박싱(grayboxing)입니다 — 디테일 없이 덩어리로 먼저 배치하고 걸어다녀 보며 크기를 검증합니다.

TIP

그레이박스 상태에서 F5로 직접 걸어보세요. 화면으로 볼 때와 캐릭터로 걸을 때의 크기 감각이 완전히 다릅니다. 대부분 처음 만든 맵은 생각보다 너무 큽니다.

동선 설계 — 어디로 흐르게 할 것인가

플레이어가 자연스럽게 가야 할 곳으로 향하게 만드는 게 좋은 맵입니다. 강제하지 않고 유도합니다.

  • 넓은 길 = 주 동선: 가야 할 길은 넓고 밝게, 곁길은 좁고 어둡게. 플레이어는 자연스럽게 넓은 쪽으로 갑니다.
  • 랜드마크로 끌기: 멀리 보이는 탑·빛기둥을 향해 길을 내면 "저기로 가야겠다"는 목표가 생깁니다.
  • 막다른 길엔 보상: 곁길 끝에 아이템·전망·비밀을 두면 탐험이 헛걸음이 안 됩니다.
  • 병목으로 페이싱 조절: 좁은 다리·문을 지나면 넓은 공간이 펼쳐지게 — 첫인상이 극대화됩니다.
  • 루프 만들기: 끝까지 갔다가 지름길로 시작점에 돌아오는 고리형이 막다른 왕복보다 훨씬 쾌적합니다.

구역 나누기와 색 구분

  • 구역마다 다른 색온도: 안전지대는 따뜻한 색, 위험지대는 차가운 색 — 플레이어가 색으로 분위기를 학습합니다.
  • 전이 구역 두기: 숲에서 사막으로 갑자기 바뀌지 말고, 사이에 메마른 풀밭 같은 중간 지대를 — 자연스러운 연결.
  • Explorer 폴더로 구역 관리: Zone_Forest, Zone_Town처럼 폴더로 나눠두면 작업·숨기기가 편합니다.

12지형 에디터 심화 중급

초급에서 배운 브러시를 넘어 — Generate, Sea Level, 자연스러운 지형을 빚는 테크닉.

Generate로 시작하기

  1. Terrain Editor > Generate 탭.
  2. Biome 선택: Mountains(산), Canyons(협곡), Dunes(사막), Arctic(설원), Lavaflow(용암) 등 조합 가능.
  3. Size를 맵 크기에 맞게, Seed를 바꿔가며 마음에 드는 형태가 나올 때까지 재생성.
  4. 생성된 지형을 기반으로 브러시로 다듬는 게 맨바닥부터 그리는 것보다 훨씬 빠릅니다.

자연스러운 지형 브러시 테크닉

  • 큰 브러시 → 작은 브러시: 큰 브러시로 산의 덩어리를 잡고, 점점 작은 브러시로 능선과 굴곡을 추가합니다. 처음부터 작은 브러시로 그리면 울퉁불퉁 지저분해집니다.
  • Strength는 낮게: 0.3 이하로 여러 번 쌓는 것이 한 번에 솟구치게 하는 것보다 자연스럽습니다.
  • Smooth는 마지막에: 형태를 다 잡은 뒤 Smooth로 정리. 중간중간 Smooth하면 디테일이 다 뭉개집니다.
  • Erode(침식): 산비탈에 살짝 쓰면 빗물에 깎인 듯한 골이 생깁니다. 진짜 산처럼 보이게 하는 비밀.
  • 재질 경계 흐리기: Paint로 잔디와 바위 경계를 칠할 때 일직선이 아니라 들쭉날쭉하게. 자연엔 직선이 없습니다.

Sea Level로 물 채우기

  1. Terrain Editor > Sea Level.
  2. 영역과 높이를 지정하고 Create — 그 높이 아래 빈 공간이 전부 물로 채워집니다.
  3. 호수를 만들려면 먼저 Subtract로 웅덩이를 파고 Sea Level을 적용.

지형 물 속성 (Terrain 선택 시 Properties)

속성역할추천
WaterColor물 색열대: 청록 / 늪: 탁한 녹색 / 깊은 바다: 진한 남색
WaterTransparency물 투명도0.7~0.9 (맑은 물)

13파트 건축 기법 중급

파트 몇 개로 건물, 계단, 아치, 다리를 만드는 실전 건축 기술.

Union과 Negate 복습 + 실전

  • Union (Ctrl+Shift+G): 선택한 파트들을 하나로 합치기.
  • Negate (Ctrl+Shift+N): 파트를 "빼는 모양"으로 전환. 빨갛게 변한 Negate 파트와 일반 파트를 함께 Union하면 그 모양대로 구멍이 뚫립니다.
  • Separate (Ctrl+Shift+U): Union을 다시 분해.
  1. 창문 뚫기: 벽 파트 + 창문 크기의 블록을 벽에 겹치게 배치 > 블록 Negate > 둘 다 선택 > Union.
  2. 아치문: 벽 + Negate한 원기둥(눕혀서) + Negate한 블록(아래 부분) > Union.
  3. 터널: 산 모양 파트에 Negate 원기둥을 관통시켜 Union.
!!

Union을 너무 많이 겹쳐 쓰면(Union의 Union의 Union...) 성능이 나빠지고 가끔 모양이 깨집니다. 복잡한 형태는 블렌더에서 만들어 임포트하는 게 낫습니다.

계단 만들기 3가지

  • 블록 쌓기: 높이 1, 깊이 2짜리 블록을 계단식으로 복제(Ctrl+D 후 이동 반복). 가장 확실.
  • WedgePart (경사면): 파트 추가할 때 Wedge 선택. 계단 대신 경사로 — 캐릭터가 부드럽게 올라갑니다. 계단처럼 보이게 하려면 위에 얇은 블록들을 장식으로 얹기.
  • 나선 계단: 한 칸 만들고 복제하면서 위치 이동 + 회전(15도씩)을 반복. 중심축 파트를 기준으로 돌리면 정확합니다.

건물 빠르게 짓는 순서

  1. 바닥 한 장 > 벽 4면 (두께 1) > 창문/문 Negate로 뚫기.
  2. 지붕: WedgePart 두 개를 맞대면 삼각 지붕.
  3. 벽 머티리얼(Brick/Wood)과 지붕 색을 먼저 정하고 시작 — 나중에 일괄 변경은 귀찮습니다.
  4. 완성되면 전체 선택 > Ctrl+G로 Model로 묶고 이름 붙이기.
  5. 같은 건물이 여러 채 필요하면 Model째 복제 후 색/소품만 바꿔 변화 주기.

높이·정렬 맞추기 — 빠르고 정확하게

맵이 지저분해 보이는 가장 큰 원인은 "삐뚤빼뚤 안 맞은 파트"입니다. 아래 방법으로 칼같이 정렬하세요.

  • 같은 높이로 맞추기: 기준 파트의 Position Y값을 Properties에서 확인 > 나머지 파트들을 선택하고 같은 Y값 입력. 여러 개를 한 번에 선택하면 값이 동시에 적용됩니다.
  • 바닥에 딱 붙이기: 파트 높이(Size Y)의 절반이 바닥 위로 오게 — 예를 들어 Size Y가 4면 바닥(Y=0) 위에 놓으려면 Position Y = 2. 또는 아래 Move 스냅을 쓰면 자동으로 격자에 맞습니다.
  • 일렬로 세우기: 한 축(예: X) 값만 모두 같게 입력하면 일직선이 됩니다.
  • 등간격 배치: 첫 파트 만들고 Ctrl+D 복제 후 정확한 간격(예: X로 8)만큼 이동 > 다시 Ctrl+D를 반복하면 같은 간격이 유지됩니다(직전 변형을 기억).

스냅과 그리드 — 격자에 착착

상단 Model 탭의 Snap to Grid에서 이동(Move)·회전(Rotate) 스냅 단위를 정합니다. 이것만 잘 써도 정렬이 저절로 됩니다.

14외부 에셋 임포트 중급

블렌더에서 만든 모델을 맵에 들여오는 방법과 자주 터지는 문제들.

Import 3D

  1. Home 또는 Avatar 탭 > Import 3D.
  2. FBX 파일 선택 > 임포트 미리보기 창에서 크기와 형태 확인.
  3. 블렌더 1m = 약 3.57 studs가 기본 환산입니다. 크기가 안 맞으면 미리보기 창에서 Scale 조절.
  4. 임포트하면 Workspace에 MeshPart(또는 Model)로 들어옵니다.

CollisionFidelity 선택

  • Box: 박스 한 덩어리로 충돌. 가장 가볍습니다. 장식 소품에 권장.
  • Default (Hull): 대략적인 외곽 충돌. 일반 용도.
  • PreciseConvexDecomposition: 모양 그대로 정밀 충돌. 무겁습니다. 플레이어가 밟고 다니는 구조물에만.
RBX

텍스처 이미지는 Creator Hub(create.roblox.com) > Development Items > Decals에서 업로드합니다. 승인 후 받은 ID를 rbxassetid://ID 형태로 사용합니다.

15SurfaceAppearance 재질 중급

MeshPart에 노멀/러프니스 맵을 입혀 진짜 같은 표면을 만드는 기능. 맵 퀄리티가 한 단계 올라갑니다.

적용 방법

  1. MeshPart 선택 > Explorer에서 + > SurfaceAppearance 추가.
  2. ColorMap: 기본 색 텍스처.
  3. NormalMap: 표면 요철 정보. 블렌더에서 베이킹한 노멀 맵 (블렌더 중급 15챕터 참고).
  4. RoughnessMap: 어디가 매끈하고 어디가 거친지.
  5. MetalnessMap: 금속 부분 지정.

언제 쓰고 언제 안 쓰나

  • 쓰면 좋은 곳: 플레이어가 가까이서 보는 핵심 구조물 — 바위, 나무 줄기, 건물 벽, 무기 거치대.
  • 안 써도 되는 곳: 멀리 있는 배경, 작은 소품, 로우폴리 스타일 맵 (단색이 컨셉이면 오히려 방해).
  • 텍스처 4장씩 들어가므로 메모리를 먹습니다. 같은 SurfaceAppearance를 여러 메시가 공유하게 하면 효율적입니다.
TIP

AlphaMode를 Transparency로 바꾸면 ColorMap의 알파 채널로 구멍 뚫린 표면(철망, 나뭇잎)을 표현할 수 있습니다. 잎사귀 한 장 한 장을 모델링할 필요가 없어집니다.

16소품 배치와 디테일 중급

텅 빈 맵과 살아있는 맵의 차이는 소품입니다. 어디에 무엇을 어떻게 놓는가.

배치 원칙

  • 군집 배치: 나무 한 그루씩 띄엄띄엄보다, 3~5그루 모인 군집 + 빈 공간. 자연은 뭉쳐서 자랍니다.
  • 크기/회전 변주: 같은 소품도 Scale 0.9~1.2배, Y축 회전을 제각각. 복붙 티가 사라집니다.
  • 스토리텔링: 부서진 수레 + 흩어진 상자 + 바퀴 자국 = 사연 있는 장면. 소품 조합으로 이야기를 암시하면 맵이 살아납니다.
  • 동선 방해 금지: 플레이어가 자주 다니는 길엔 소품을 적게. 걸려서 끼는 순간 스트레스.
  • 시선 유도: 가야 할 방향에 밝은 색, 빛, 큰 소품 배치. 플레이어는 말 안 해줘도 그쪽으로 갑니다.

빠른 배치 도구

  • Alt+드래그 — 이동하면서 복제. 같은 소품을 빠르게 흩뿌릴 때.
  • Ctrl+L — 이동 축을 로컬/월드 전환. 기울어진 표면에 맞춰 놓을 때.
  • Model 탭 > Snap to Grid 해제 — 소품은 그리드에 안 맞춰야 자연스럽습니다 (건축은 켜고, 소품은 끄고).

17조명과 Atmosphere 중급

같은 맵도 조명에 따라 천국과 폐허가 됩니다. Lighting 서비스 완전 정리.

Lighting 핵심 속성

속성역할분위기별 추천
ClockTime시간 (0~24). 태양 위치와 하늘색낮 14 / 황혼 17.5 / 밤 22 / 새벽 5.5
Brightness태양광 세기낮 2~3 / 밤 0.3~0.8
Ambient그림자 속 최소 밝기실내·동굴 맵에서 중요. 너무 검으면 안 보임
OutdoorAmbient야외 환경광 색황혼이면 주황 기, 설원이면 푸른 기
ColorShift_Top위에서 내리쬐는 빛의 색조노을 연출에 주황~분홍
EnvironmentDiffuseScale하늘이 사물을 물들이는 정도0.5~1 — 켜면 색이 풍부해짐
Technology렌더 방식Future(최고 화질) / ShadowMap(균형)

Atmosphere

Lighting 안에 Atmosphere 인스턴스를 추가하면 공기 자체를 연출합니다:

  • Density (0~1): 공기 농도. 0.3 산뜻 / 0.45 보통 / 0.6+ 안개 낀 느낌.
  • Haze: 지평선의 뿌연 정도. 2~4면 원근감이 생겨 맵이 넓어 보입니다.
  • Color / Decay: 대기 색과 소실 색. 노을엔 주황/보라 조합.
  • Glare: 태양 번짐. 0.5~1.

후처리 이펙트 (Lighting에 추가)

  • BloomEffect: 밝은 것이 빛나 보임. Neon과 세트. Intensity 0.5~1, Threshold 1~2.
  • ColorCorrectionEffect: 전체 색감 보정. Saturation -0.1이면 차분, +0.15면 쨍한 만화풍. TintColor로 맵 전체에 색조.
  • DepthOfFieldEffect: 멀리 흐림. 로비처럼 좁은 공간에서 분위기용. 게임플레이 맵에선 끄는 게 보통.
  • SunRaysEffect: 태양 광선 줄기. Intensity 0.05~0.15만 — 과하면 눈아픔.

개별 조명

  • PointLight: 전구처럼 사방으로. 횃불, 램프 안에. Range 8~16, Brightness 1~3.
  • SpotLight: 원뿔 방향광. 가로등, 무대 조명. Face로 방향, Angle로 퍼짐 조절.
  • SurfaceLight: 면 전체에서 빛. 모니터, 창문, 간판 발광에 자연스럽습니다.
TIP

밤 맵의 공식: ClockTime 22 + Brightness 0.5 + 따뜻한 색(255,180,110) PointLight를 길 따라 배치. 어두운 배경과 따뜻한 불빛의 대비가 분위기를 만듭니다.

18물과 특수 지형 중급

강, 폭포, 용암, 동굴 — 평범한 맵을 특별하게 만드는 지형 요소들.

강 만들기

  1. Subtract 브러시로 강줄기 골을 팝니다. 일직선이 아니라 구불구불하게.
  2. Sea Level로 물 채우기 — 또는 강 구간만 Fill 도구에서 Water 재질로 채우기.
  3. 강가에 모래(Sand)를 Paint로 칠해 물가 느낌. 바위 소품을 강 중간중간에.

폭포

  1. 절벽 위 물과 아래 물을 만들어 두고, 떨어지는 부분은 파트로 표현합니다 (Terrain 물은 수직으로 못 섭니다).
  2. 흰색~하늘색 반투명 파트(Transparency 0.4, Glass 또는 SmoothPlastic)를 절벽에 세우기.
  3. 떨어지는 지점에 흰색 반투명 파트를 겹쳐 물보라 표현. ParticleEmitter(흰 연기 텍스처)를 추가하면 물안개까지.

용암

  • Terrain 재질 CrackedLava로 칠하면 갈라진 틈에서 빛나는 용암 지형이 됩니다.
  • 흐르는 용암 호수는 주황 Neon 파트 + 위에 검붉은 반투명 파트를 겹쳐 표면 막 표현.
  • 주변 Lighting Ambient에 붉은 기 + 주황 PointLight 여러 개 = 열기 연출.

동굴

  1. 산을 만들고 Subtract 브러시로 안을 파냅니다. 입구는 좁고 안은 넓게.
  2. 동굴 안은 햇빛이 안 드므로 Ambient만으로는 깜깜합니다 — 발광 버섯(Neon), 크리스탈, 횃불 PointLight를 광원으로 배치.
  3. 천장에서 떨어지는 종유석: 블렌더 원뿔 메시 or Wedge 조합.
!!

Terrain의 Water는 진짜 수영 가능한 물입니다. 파트로 만든 가짜 물(반투명 파란 파트)은 수영이 안 되고 그냥 통과하거나 막힙니다. 들어갈 수 있는 물은 반드시 Terrain Water로.

19사운드 환경 중급

눈 감고도 숲인지 동굴인지 알 수 있어야 좋은 맵입니다. 소리는 코드 없이 배치만으로 됩니다.

환경음 (Ambient) 배치

  1. 소리 낼 위치의 파트(또는 Attachment)에 Sound 인스턴스 추가.
  2. SoundId에 사운드 에셋 ID. Toolbox > Audio 탭에서 무료 사운드 검색 가능 (새소리, 물소리, 바람).
  3. Looped: true, Playing: true 체크 — 코드 없이 게임 시작부터 무한 재생됩니다.
  4. RollOffMaxDistance: 소리가 들리는 최대 거리. 모닥불 30, 폭포 150.
  5. RollOffMode: InverseTapered가 자연스럽게 멀어질수록 작아집니다.

구역별 사운드 설계

구역소리Volume
숲새소리 + 바람 잎사귀0.3~0.5
강/폭포물 흐름 (폭포는 가까울수록 큼)0.4~0.8
동굴물방울 + 낮은 울림0.3
마을웅성거림 + 대장간 망치질0.2~0.4
전체 BGMSoundService에 Sound 추가 (위치 무관 재생)0.15~0.3
TIP

BGM은 작게, 환경음은 위치별로. 플레이어가 이동하면서 소리가 자연스럽게 교차되면 맵이 진짜 공간처럼 느껴집니다.

20스카이박스와 배경 중급

맵의 끝은 어디인가 — 하늘과 먼 배경으로 세계가 이어지는 착각을 만듭니다.

Skybox 교체

  1. Lighting에 Sky 인스턴스 추가.
  2. SkyboxBk/Dn/Ft/Lf/Rt/Up 6면에 하늘 이미지 ID 입력. Toolbox에서 "skybox" 검색하면 완성된 세트를 구할 수 있습니다.
  3. SunAngularSize / MoonAngularSize로 해와 달 크기, StarCount로 밤하늘 별 개수 조절.

먼 배경 산 (스카이라인)

  • 맵 가장자리 너머에 큰 산 모양 파트나 로우폴리 산 메시를 배치합니다. 단색에 가깝게, 안개 색과 비슷한 톤으로.
  • Atmosphere의 Haze 덕분에 멀리 있는 것은 자동으로 뿌옇게 — 배경 산은 대충 만들어도 그럴듯해집니다.
  • 배경 파트는 CanCollide false, CastShadow false, CanQuery false.

맵 경계 처리

  • 자연 경계: 절벽, 바다, 산맥으로 못 가게 막기 — 보이지 않는 벽보다 납득됩니다.
  • 투명 벽: 어쩔 수 없을 땐 Transparency 1, CanCollide true 파트. 단, 보이는 길이 벽으로 막혀 있으면 플레이어가 답답해합니다 — 경계는 시각적으로도 끝나 보이게.
TIP

중급을 마쳤으면 작은 테마 맵 하나를 처음부터 끝까지 완성해 보세요 — 설계, 지형, 건물 두세 채, 조명, 소리까지. 다음 단계인 고급에서 이 맵을 최적화하고 완성도를 끌어올립니다.
